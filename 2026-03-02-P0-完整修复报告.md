## 三、完整修复方案

### 修复 1：后端 AI 调用超时保护 ✅

**文件**: `wechat_backend/nxm_concurrent_engine_v3.py`  
**修改位置**: 第 174-240 行

**修复内容**:

```python
async def _execute_single_task(
    self,
    question: str,
    model_name: str,
    q_idx: int,
    model_idx: int
) -> Dict[str, Any]:
    """
    执行单个任务（问题×模型，带超时保护）
    """
    async with self._semaphore:
        task_start = time.time()

        try:
            # ... 初始化代码 ...

            # 2. 获取超时配置（默认 60 秒）
            timeout_manager = get_timeout_manager()
            timeout = timeout_manager.get_timeout(model_name) or 60  # 默认 60 秒

            # 4. 执行 AI 调用（使用单模型执行器，带超时保护）
            # 【P0 关键修复 - 2026-03-02】使用 asyncio.wait_for 添加超时保护
            try:
                ai_result, actual_model = await asyncio.wait_for(
                    get_single_model_executor(timeout=timeout).execute(
                        prompt=prompt,
                        model_name=model_name,
                        execution_id=self.execution_id,
                        q_idx=q_idx
                    ),
                    timeout=timeout
                )
            except asyncio.TimeoutError:
                # 【P0 关键修复】AI 调用超时，返回错误结果但不阻塞其他任务
                task_elapsed = time.time() - task_start
                api_logger.error(
                    f"[NxM-Parallel] ⏰ 任务超时：Q{q_idx + 1}×{model_name} "
                    f"({task_elapsed:.2f}秒 > {timeout}秒)"
                )
                
                return {
                    'success': False,
                    'error': f'AI 调用超时（{timeout}秒）',
                    'question': question,
                    'model': model_name,
                    'timeout': True,
                    'latency': task_elapsed
                }

            # 5. 处理结果
            if ai_result.success:
                # ... 成功处理 ...
```

**修复效果**:
- 每个 AI 调用最多等待 60 秒
- 超时后返回错误结果，不阻塞其他任务
- 即使部分失败，也能继续后续流程

max_retries = 3
for attempt in range(max_retries):
    saved_results = result_repo.get_by_execution_id(execution_id)
