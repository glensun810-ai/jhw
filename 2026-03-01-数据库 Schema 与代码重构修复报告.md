## 问题 3：索引优化跳过（中等）

### 问题描述
**报错日志**:
```
WARNING - query_optimizer.py:274 - create_index() - 跳过索引创建：表 users/test_records/brand_test_results 不存在
```

### 根因分析
查询优化器在启动时尝试创建索引，但此时数据库表可能还未完全创建。虽然 `init_db()` 已执行，但索引优化器连接的是不同的数据库路径，或者表创建逻辑在索引逻辑之后。

### 修复方案
在 `query_optimizer.py` 的 `init_recommended_indexes()` 函数中添加重试机制：

```python
def init_recommended_indexes():
    """初始化推荐索引（增强版）"""
    import time
    
    index_manager = get_index_manager()

    # 【修复 P3-003】等待表完全创建后再创建索引
    max_retries = 3
    retry_delay = 0.5  # 秒
    
    for attempt in range(max_retries):
        tables = index_manager.list_tables()
        required_tables = ['users', 'test_records', 'brand_test_results', 'audit_logs', 'sync_results', 'cache_entries']
        missing_tables = [t for t in required_tables if t not in tables]
        
        if not missing_tables:
            api_logger.info(f"所有必需表已存在，开始创建索引")
            break
        else:
            if attempt < max_retries - 1:
                api_logger.warning(f"等待表创建：缺少 {missing_tables}，{retry_delay}s 后重试...")
                time.sleep(retry_delay)
            else:
                api_logger.warning(f"表不存在，跳过索引创建：{missing_tables}")
```

### 修复效果
- 自动等待表创建完成
- 最多重试 3 次，每次间隔 0.5 秒
- 即使表不全，也会尝试创建已存在表的索引

✅ **初始化顺序已优化**
