```
2026-03-03 23:18:51,495 - wechat_backend.api - ERROR - diagnosis_views.py:556 - run_orchestrated_diagnosis() - [Orchestrator] ❌ 未预期异常 - execution_id=a7f23258-0b71-4c82-b991-c1cdbbd9c85d, thread_id=13063548928: 'DiagnosisErrorCode' object has no attribute 'code'
```

### 1.2 问题根因

**主要问题**: `AttributeError: 'DiagnosisErrorCode' object has no attribute 'code'`

- **错误链路**:
  1. 数据库连接超时 → `ValueError: 结果保存失败：Database connection timeout after 5.0s`
  2. 编排器捕获异常并调用 `_determine_error_code()` → 返回 `DiagnosisErrorCode.DIAGNOSIS_TIMEOUT`
  3. 调用 `error_logger.log_error()` → 尝试访问 `error_code.code`
  4. **失败**: `DiagnosisErrorCode` 是 Enum 成员，不是 `ErrorCodeDefinition` 对象

- **技术原因**:
  - `_determine_error_code()` 返回 `DiagnosisErrorCode` 枚举成员
  - `error_logger.log_error()` 期望 `ErrorCodeDefinition` 对象
  - Enum 成员需要通过 `.value` 访问内部的 `ErrorCodeDefinition`

**次要问题**: 数据库连接池超时

- **原因**: 并行 AI 调用完成后，阶段 3 结果保存时并发写入数据库
- **配置**: 连接池大小 10，超时 5 秒
- **瓶颈**: 高并发场景下连接数不足

## 二、修复内容

### 2.1 修复 `error_logger.py` - Enum 类型支持

**文件**: `backend_python/wechat_backend/error_logger.py`

**修复代码**:

```python
def log_error(
    self,
    error: Exception,
    error_code,  # 可以是 ErrorCodeDefinition 或 Enum（如 DiagnosisErrorCode）
    execution_id: Optional[str] = None,
    # ... 其他参数
) -> str:
    from enum import Enum
    
    # 【修复 - 2026-03-03】支持 Enum 类型的错误码（如 DiagnosisErrorCode）
    # 如果是 Enum，通过 .value 获取 ErrorCodeDefinition
    error_code_def = error_code.value if isinstance(error_code, Enum) else error_code

    # 构建错误上下文
    context = ErrorContext(
        error_code=error_code_def.code,      # 使用 .value.code
        error_message=str(error),
        severity=error_code_def.severity.value,
        # ...
    )
    
    # 日志级别判断
    if error_code_def.severity == ErrorSeverity.CRITICAL:
        log_level = logging.CRITICAL
    # ...
    
    # 日志消息
    log_message = f"[{error_code_def.code}] {context.error_message}\n..."
```

**修复点**:
1. 参数类型改为通用类型（支持 Enum 和 ErrorCodeDefinition）
2. 添加 `isinstance(error_code, Enum)` 检查
3. 通过 `.value` 访问 `ErrorCodeDefinition` 对象
4. 后续所有 `error_code.*` 改为 `error_code_def.*`
