## 一、问题背景

根据最新运行日志分析，系统存在以下核心问题：

### 1.1 核心问题：认证拦截导致 403/401 级联错误

**现象**:
- 前端轮询 `/test/status/<id>` 接口时被后端 `auth_enhanced` 中间件拦截
- 日志中出现高频权限警告：`WARNING - auth_enhanced.py:66 - [Auth Middleware] 未认证访问敏感端点`
- 前端收到错误响应后触发 `ERR_CONNECTION_REFUSED`
- 前端不断重试（每隔不到 1 秒），导致日志瞬间被刷屏

**根因**:
- `auth_enhanced.py` 将 `/test/status/` 列为**严格认证端点**
- `diagnosis_views.py` 的装饰器设置为 `require_auth=False`
- **中间件优先级高于装饰器**，导致 401/403 错误

**后果**:
- 前端无法获取任务完成状态，形成死循环
- 后端认证中间件和日志系统面临巨大无效负载
- 掩盖真实问题，增加排查难度

### 1.2 潜在 Bug：AI 响应格式不兼容

**现象**:
- GeoParser 组件频繁触发降级逻辑：`WARNING - geo_parser.py:146 - [GeoParser] JSON 解析失败，启用语义分析降级`
- DeepSeek 返回带有 Markdown 格式的自然语言文本，而非纯 JSON

**影响**:
- 降级处理比直接解析 JSON 更耗时且不可控
- 语义分析失败时，前端可能拿不到任何结构化结果

### 1.3 性能隐患：AI 响应延迟与并发压力

**数据**:
- DeepSeek 单次 API 调用耗时高达 **29.98 秒**
- 在 30 秒等待时间内，前端发起超过 **30 次** 无效轮询请求
- 多用户同时使用时，后端压力呈指数级增长

## 二、优化方案与实施状态

### P0 修复（紧急）

#### ✅ P0-1: 修复认证配置冲突

**问题**: `/test/status` 端点的认证配置不一致导致 401/403 级联错误

**修复方案**:
- 从严格认证列表中移除 `/test/status/`
- 理由：该端点仅返回任务进度，不包含敏感用户数据
- 允许匿名访问符合业务需求（用户无需登录即可查看诊断进度）

**实施文件**:
- `backend_python/wechat_backend/security/auth_enhanced.py`
  - 修改 `is_strict_auth_endpoint()` 函数
  - 修改 `STRICT_AUTH_ENDPOINTS` 常量

**状态**: ✅ **已完成** (2026-03-02)

**预期效果**:
- 前端轮询不再被认证中间件拦截
- 401/403 错误日志消失
- 前端熔断机制正常触发

---

### P1 修复（高优先级）

#### ✅ P1-1: 增强 AI 响应 JSON 解析容错性

**问题**: AI 返回非标准 JSON 格式导致解析失败

**修复方案**:
1. 新增 `_repair_json_string()` 函数，修复常见 JSON 问题：
   - 未闭合的括号
   - 转义字符问题
   - 引号不匹配
   - Markdown 残留
   - 末尾逗号 (trailing commas)
   - BOM 或不可见字符

2. 增强 `_try_parse_json()` 函数，在解析前自动修复 JSON

**实施文件**:
- `backend_python/wechat_backend/ai_adapters/geo_parser.py`
  - 新增 `_repair_json_string()` 函数
  - 修改 `_try_parse_json()` 函数

**状态**: ✅ **已完成** (2026-03-02)

**预期效果**:
- JSON 解析成功率提升 30%+
- 减少语义分析降级触发频率
- 提高数据提取稳定性

---

#### ✅ P1-2: 优化 Prompt 工程强制 JSON 输出

**问题**: AI 模型返回 Markdown 包裹的 JSON 或纯文本

**修复方案**:
1. 优化 `GEO_PROMPT_TEMPLATE`，强化 JSON 格式要求
2. 添加明确的禁止指令：
   - ⚠️ **必须输出纯 JSON，不要包含在 Markdown 代码块中**
   - ⚠️ **不要使用 ```json 或 ``` 包裹 JSON 内容**
   - ⚠️ **不要有任何额外说明文字在 JSON 之后**
   - ⚠️ **JSON 必须是最后一行，后面不要有任何内容**

3. 添加自检指令和输出示例

**实施文件**:
- `backend_python/wechat_backend/ai_adapters/base_adapter.py`
  - 修改 `GEO_PROMPT_TEMPLATE` 常量

**状态**: ✅ **已完成** (2026-03-02)

**预期效果**:
- AI 返回纯 JSON 格式的比例提升
- 减少 Markdown 解析开销
- 提高解析成功率

## 四、架构建议（中长期）

### 4.1 认证架构优化

**建议**: 引入认证策略配置文件

```yaml
# config/auth_config.yaml
endpoints:
  /test/status:
    auth: optional
    rate_limit: 60/minute
  /api/test-history:
    auth: required
    rate_limit: 30/minute
```

**优势**:
- 配置与代码分离
- 便于动态调整
- 减少配置冲突

---

### 4.2 AI 响应处理优化

**建议**: 引入 AI 响应标准化层

```
AI 平台 → 适配器 → 标准化响应 → JSON 解析 → 业务逻辑
```

**优势**:
- 统一不同 AI 平台的输出格式
- 降低解析复杂度
- 提高可维护性

---

### 4.3 轮询机制优化

**建议**: 使用 WebSocket 替代 HTTP 轮询

**优势**:
- 实时推送，减少无效请求
- 降低服务器压力
- 改善用户体验
