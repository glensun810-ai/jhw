# P0-7 修复：数据格式统一化方案

## 问题描述

项目中多处存在 `snake_case` 与 `camelCase` 转换逻辑，可能导致前端解析失败：
- 多个文件定义了各自的转换函数
- 转换逻辑不一致导致字段名混乱
- 前端期望 camelCase 但后端返回 snake_case

## 解决方案

### 1. 统一转换模块

创建统一的字段转换模块：`backend_python/utils/field_converter.py`

**核心函数**：
```python
# 将 snake_case 转换为 camelCase
def to_camel_case(snake_str: str) -> str

# 递归转换字典键名
def convert_response_to_camel(data: Any) -> Any

# 将 camelCase 转换为 snake_case
def to_snake_case(camel_str: str) -> str

# 递归转换请求数据
def convert_request_to_snake(data: Any) -> Any
```

### 2. 使用规范

#### 2.1 后端 API 响应

**所有 API 响应必须使用 `convert_response_to_camel` 转换**：

```python
# ✅ 正确用法
from utils.field_converter import convert_response_to_camel

@diagnosis_bp.route('/api/diagnosis/report/<execution_id>')
def get_full_report(execution_id):
    report = service.get_full_report(execution_id)
    return jsonify(convert_response_to_camel(report)), 200
```

**已应用的文件**：
- `wechat_backend/views/diagnosis_api.py` ✅
- `wechat_backend/views/sync_views.py` ✅
- `wechat_backend/state_manager.py` ✅
- `wechat_backend/diagnosis_report_service.py` ✅

#### 2.2 前端请求处理

**前端传入的请求数据应转换为 snake_case**：

```python
# ✅ 正确用法
from utils.field_converter import convert_request_to_snake

@diagnosis_bp.route('/api/diagnosis/start', methods=['POST'])
def start_diagnosis():
    # 将前端的 camelCase 转换为后端的 snake_case
    data = convert_request_to_snake(request.get_json())
    brand_name = data.get('brand_name')
    competitor_brands = data.get('competitor_brands', [])
```

### 3. 命名约定

| 层级 | 命名风格 | 示例 |
|------|---------|------|
| 数据库字段 | snake_case | `execution_id`, `brand_name` |
| Python 变量 | snake_case | `report_id`, `user_id` |
| API 响应 | camelCase | `executionId`, `brandName` |
| 前端请求 | camelCase | `executionId`, `brandName` |
| 前端变量 | camelCase | `executionId`, `reportData` |

### 7. 常见问题

#### Q1: 为什么 API 响应要用 camelCase？

**答**：前端 JavaScript 使用 camelCase 命名约定，保持前后端各自的语言习惯：
- 后端 Python：snake_case（PEP 8 规范）
- 前端 JavaScript：camelCase（JavaScript 惯例）

#### Q2: 如果前端需要访问 snake_case 字段怎么办？

**答**：不应该。所有前端可访问的字段都应该通过 API 响应提供，并自动转换为 camelCase。

#### Q3: 数据库字段可以改为 camelCase 吗？

**答**：不建议。数据库字段保持 snake_case：
- SQL 标准推荐使用下划线分隔
- 现有数据库已使用 snake_case
- 转换层会处理格式差异
