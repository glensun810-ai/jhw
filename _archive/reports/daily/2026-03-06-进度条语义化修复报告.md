## 📋 问题描述

### 用户体验断裂

当前进度条存在严重的语义不一致问题：

```
状态：status='failed'
进度：progress=100%
文案：诊断中... 100%
颜色：蓝色（正常进行色）
```

**问题本质**: 进度条仅由数值驱动，而非业务状态驱动。用户看到 100% 进度和蓝色进度条，但实际诊断已失败。

---

## 🎯 修复目标

### 架构师建议

> 进度条应由业务逻辑状态驱动，而非单纯的数值。若 `save_diagnosis_report` 校验失败（expected_count != actual_count），进度条应立即变为警示色（如珊瑚红），下方文字显示"报告聚合异常"。

### 修复原则

1. **状态驱动**: 进度条颜色、文案由 `status` 字段决定
2. **语义明确**: 失败状态使用明确的业务文案
3. **视觉警示**: 失败时使用珊瑚红警示色
4. **进度保持**: 失败时保留当前进度值，不重置为 0

```javascript
  switch(status) {
    case 'failed':
      return '#FF6B6B'; // 珊瑚红 - 警示色
    case 'completed':
      return '#00F5A0'; // 绿色 - 成功色
    case 'ai_fetching':
    case 'intelligence_analyzing':
    case 'competition_analyzing':
    case 'report_aggregating':
      return '#00A9FF'; // 蓝色 - 进行中
    case 'init':
      return '#8c8c8c'; // 灰色 - 初始化
    default:
      return '#8c8c8c'; // 灰色 - 未知状态
  }
}
```

```javascript
// 真正的失败（进度<100% 且没有完成标志）

parsed.statusText = '诊断中断';

parsed.stage = TASK_STAGES.FAILED;
parsed.is_completed = false;
```

## 🎨 设计规范

### 颜色语义

| 状态 | 颜色 | Hex 代码 | 用途 |
|------|------|----------|------|
| 成功 | 绿色 | #00F5A0 | completed 状态 |
| 进行中 | 蓝色 | #00A9FF | ai_fetching, analyzing 等 |
| 失败 | 珊瑚红 | #FF6B6B | failed 状态 |
| 初始化 | 灰色 | #8c8c8c | init 状态 |

### 动画规范

| 动画名称 | 用途 | 时长 | 效果 |
|----------|------|------|------|
| pulse-warning | 状态文字 | 2s | 透明度 1 ↔ 0.7 |
| shimmer-warning | 进度条 | 3s | 透明度 1 ↔ 0.8 |

## 🔧 技术实现

### 状态驱动架构

```
业务状态 (status)
    ↓
颜色映射 (getStatusColor)
    ↓
样式类名 (status-failed)
    ↓
视觉效果 (颜色 + 动画)
```

### CSS 类名层级

```
.analysis-card
  └─ .status-failed (卡片整体)
  └─ ::before (顶部渐变条)
  └─ .status-text
      └─ .status-text-failed (文字颜色 + 动画)
  └─ .progress-container
      └─ .progress-bar-container
          └─ .progress-bar-failed (容器背景)
          └─ .progress-bar
              └─ .progress-bar-failed (进度条颜色 + 动画)
      └─ .progress-text
          └─ .progress-text-failed (文字颜色)
```
