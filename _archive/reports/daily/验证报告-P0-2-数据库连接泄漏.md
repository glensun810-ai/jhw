### 1. 检查上下文管理器实现

#### 1.1 `DiagnosisReportRepository.get_connection()` 方法

**代码位置**: `diagnosis_report_repository.py:96-116`

```python
@contextmanager
def get_connection(self):
    """
    获取数据库连接（P0 修复 - 2026-03-05：修复上下文管理器）

    【P0 关键修复】确保上下文管理器在任何情况下都正确 yield 和清理
    """
    conn = get_db_pool().get_connection()
    try:
        yield conn  # 先 yield 连接
        conn.commit()  # 成功后提交
    except Exception as e:
        conn.rollback()  # 失败时回滚
        db_logger.error(f"数据库操作失败：{e}")
        raise  # 重新抛出异常
    finally:
        # 无论成功还是失败，都归还连接
        try:
            get_db_pool().return_connection(conn)
        except Exception as return_err:
            db_logger.error(f"归还连接失败：{return_err}")
```

**分析**:
- ✅ 使用了 `@contextmanager` 装饰器
- ✅ 在 `finally` 块中归还连接
- ✅ 异常时先回滚再归还
- ✅ 归还操作有异常处理

**潜在问题**:
1. 如果 `get_db_pool().get_connection()` 抛出异常，`conn` 变量可能未定义
2. `finally` 块中访问 `conn` 可能导致 `UnboundLocalError`

## 根因分析

### 真正根因

**不是** "上下文管理器实现错误"，而是：

1. **代码不一致**: 多处代码绕过连接池，直接使用 `sqlite3.connect()`
2. **手动管理连接**: 部分代码使用 `conn = get_db_pool().get_connection()` 而非 `with` 语句
3. **泄漏检测阈值过长**: 30 秒对于短事务来说太长，可能导致小泄漏未被及时发现
4. **缺乏实时监控**: 连接池指标存在，但无主动监控和告警

### 潜在风险场景

#### 场景 1: 异常导致连接未归还

```python
conn = get_db_pool().get_connection()
try:
    # 如果这里发生异常，且异常发生在 conn 赋值之后
    do_something(conn)
finally:
    # finally 总是会执行，所以连接会被归还 ✅
    get_db_pool().return_connection(conn)
```

**结论**: 有 `finally` 块保护，风险低

#### 场景 2: 重复归还

```python
# 代码 bug 导致重复调用 return_connection
get_db_pool().return_connection(conn)  # 第一次
# ... 某些逻辑 ...
get_db_pool().return_connection(conn)  # 第二次 ⚠️

# return_connection 方法会检测到并关闭连接
if conn_id not in self._in_use:
    db_logger.warning("连接池检测到重复归还")
    conn.close()  # ✅ 安全处理
```

**结论**: 有重复归还检测，风险低

#### 场景 3: 长事务持有连接

```python
with repo.get_connection() as conn:
    # 执行网络 IO（AI 调用）⚠️
    ai_result = call_ai_api()  # 可能耗时 10-30 秒
    # 连接在此期间被占用
```

**结论**: 可能导致连接泄漏警告（>30 秒），但连接最终会归还

## 修复建议

### 方案 A: 修复上下文管理器中的变量作用域问题（推荐）

```python
@contextmanager
def get_connection(self):
    """获取数据库连接（增强版）"""
    conn = None  # ✅ 初始化为 None
    try:
        conn = get_db_pool().get_connection()
        yield conn
        conn.commit()
    except Exception as e:
        if conn:  # ✅ 检查 conn 是否存在
            conn.rollback()
        db_logger.error(f"数据库操作失败：{e}")
        raise
    finally:
        if conn:  # ✅ 检查 conn 是否存在
            try:
                get_db_pool().return_connection(conn)
            except Exception as return_err:
                db_logger.error(f"归还连接失败：{return_err}")
```

### 方案 B: 统一使用 `with` 语句

将所有直接调用改为：

```python
# ❌ 旧代码
conn = get_db_pool().get_connection()
try:
    # 操作
finally:
    get_db_pool().return_connection(conn)

# ✅ 新代码
with repo.get_connection() as conn:
    # 操作
```

### 方案 C: 降低泄漏检测阈值

```python
# 从 30 秒降低到 10 秒
if use_duration > 10:  # 更激进的检测
    self._leak_detection_count += 1
    db_logger.warning(
        f"⚠️ 连接池检测到潜在连接泄漏："
        f"id={conn_id}, 使用时长={use_duration:.1f}秒"
    )
```

### 方案 D: 添加连接池监控和告警

```python
# 定期检查连接池状态
def check_pool_health():
```
