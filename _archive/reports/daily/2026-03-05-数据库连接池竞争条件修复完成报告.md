## 一、问题描述

### 1.1 原问题

**修复前代码问题**:

```python
# 问题 1: 使用 Lock 可能导致死锁
self._lock = threading.Lock()

# 问题 2: 无健康检查，unhealthy 连接被回收
if self._pool:
    conn = self._pool.pop()
    self._in_use.add(id(conn))  # 直接加入使用集合
    return conn

# 问题 3: 异常时连接可能泄漏
conn = sqlite3.connect(DB_PATH, timeout=30.0)
conn.execute('PRAGMA journal_mode=WAL')
# 如果这里发生异常，conn 已创建但未加入 _in_use

```

**问题表现**:
- 高并发下连接泄漏
- unhealthy 连接被重复使用
- 频繁超时（2 秒太短）
- 异常时连接未正确清理

**影响范围**:
- 所有数据库操作
- 高并发场景尤为严重
- 可能导致系统雪崩

## 二、修复方案

### 2.1 使用 RLock 避免死锁

**文件**: `database_connection_pool.py:61`

```python
# 修复前
self._lock = threading.Lock()

# 修复后
self._lock = threading.RLock()  # 可重入锁
```

**优势**:
- 同一线程可多次获取锁
- 避免嵌套调用死锁
- 更安全的方法间调用

### 2.3 改进连接获取逻辑

**文件**: `database_connection_pool.py:74-182`

```python
def get_connection(self, timeout: Optional[float] = None) -> sqlite3.Connection:
    """获取数据库连接（P0 增强版）"""
    if timeout is None:
        timeout = self.default_timeout  # 5 秒
    
    start_time = time.time()
    conn = None
    
    while True:
        try:
            with self._lock:
                # 从池中获取时健康检查
                while self._pool:
                    conn = self._pool.pop()
                    if self._is_connection_healthy(conn):
                        self._in_use.add(id(conn))
                        self._connection_use_time[id(conn)] = time.time()
                        return conn
                    else:
                        # unhealthy 连接关闭
                        conn.close()
                        conn = None
                        self._created_count -= 1
                
                # 创建新连接
                if self._created_count < self.max_connections:
                    self._created_count += 1
                    try:
                        conn = sqlite3.connect(DB_PATH, timeout=30.0)
                        conn.execute('PRAGMA journal_mode=WAL')
                        conn.execute('PRAGMA synchronous=NORMAL')
                        
                        # 验证连接可用
                        conn.execute('SELECT 1').fetchone()
                        
                        self._in_use.add(id(conn))
                        self._connection_use_time[id(conn)] = time.time()
                        return conn
                    except Exception as e:
                        # 创建失败回滚计数
                        self._created_count -= 1
                        if conn:
                            conn.close()
                        conn = None
                        raise
                
                # 超时检查
                if (time.time() - start_time) >= timeout:
                    raise TimeoutError(...)
            
            time.sleep(0.1)  # 锁外等待
            
        except TimeoutError:
            raise
        except Exception as e:
            # 确保异常时清理连接
            if conn:
                conn.close()
            raise
```

**关键改进**:
1. 健康检查后再使用
2. 创建失败正确回滚
3. 异常时确保清理
4. 超时时间增加到 5 秒

### 2.4 改进连接归还逻辑

**文件**: `database_connection_pool.py:202-265`

```python
def return_connection(self, conn: sqlite3.Connection):
    """归还数据库连接（P0 增强版）"""
    with self._lock:
        conn_id = id(conn)
        
        # 检测重复归还
        if conn_id not in self._in_use:
            db_logger.warning(f"连接池检测到重复归还：id={conn_id}")
            conn.close()
            return
        
        # 检查使用时长（检测泄漏）
        use_start_time = self._connection_use_time.get(conn_id, time.time())
        use_duration = time.time() - use_start_time
        
        if use_duration > 60:
            self._leak_detection_count += 1
            db_logger.warning(
                f"⚠️ 连接池检测到潜在连接泄漏："
                f"id={conn_id}, 使用时长={use_duration:.1f}秒"
            )
        
        # 清理使用时间记录
        del self._connection_use_time[conn_id]
        self._in_use.remove(conn_id)
        
        # 健康检查后再回收
        if self._is_connection_healthy(conn):
            self._pool.append(conn)
        else:
            conn.close()
            self._created_count -= 1
```

**关键改进**:
1. 检测重复归还
2. 检查使用时长（>60 秒告警）
3. 健康检查后再回收
4. unhealthy 连接不回收

### 2.5 实现连接泄漏检测

**文件**: `database_connection_pool.py:292-315`

```python
def detect_leaks(self) -> Dict[int, Dict[str, Any]]:
    """检测可能的连接泄漏"""
    leaks = {}
    current_time = time.time()
    
    with self._lock:
        for conn_id, start_time in list(self._connection_use_time.items()):
            duration = current_time - start_time
            if duration > 30:  # 超过 30 秒视为潜在泄漏
                leaks[conn_id] = {
                    'duration': duration,
                    'in_use': conn_id in self._in_use
                }
    
    if leaks:
        db_logger.warning(f"🚨 连接池检测到 {len(leaks)} 个潜在连接泄漏")
    
    return leaks
```

**功能**:
- 实时检测超过 30 秒未归还的连接
- 返回详细泄漏信息
- 用于监控和告警

### 2.6 增强监控指标

**文件**: `database_connection_pool.py:341-402`

```python
def get_pool_status(self) -> Dict[str, Any]:
    """获取连接池详细状态（P0 增强版）"""
    with self._lock:
        leaks = self.detect_leaks()
        
        return {
            'max_connections': self.max_connections,
            'active_connections': len(self._in_use),
            'available_connections': len(self._pool),
            'total_created': self._created_count,
            'timeout_count': self._timeout_count,
            'leak_detection_count': self._leak_detection_count,
            'potential_leaks': len(leaks),
            'leak_details': leaks,  # 详细泄漏信息
            'avg_wait_time_ms': ...,
            'health_status': ...,
            'default_timeout': self.default_timeout
        }
```

**新增指标**:
- `leak_detection_count`: 泄漏检测次数
- `potential_leaks`: 当前潜在泄漏数
- `leak_details`: 详细泄漏信息
- `default_timeout`: 默认超时时间
