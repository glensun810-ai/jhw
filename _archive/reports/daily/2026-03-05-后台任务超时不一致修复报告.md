# 2026-03-05-后台任务超时不一致修复报告

**修复日期**: 2026-03-05  
**修复类型**: P0 关键问题修复  
**影响范围**: 后台任务管理、诊断编排器、报告生成  

---

## 一、问题描述

### 1.1 问题现象

1. **报告不完整**: 用户收到的诊断报告缺少品牌分析或竞争分析数据
2. **轮询卡死**: 前端轮询任务状态时，长时间显示"处理中"但实际后台已超时
3. **超时不一致**: 不同位置的超时配置不一致，导致不可预测的行为

### 1.2 问题影响

- **用户体验**: 报告数据缺失，影响用户决策
- **系统资源**: 编排器无效等待，浪费服务器资源
- **可维护性**: 超时配置分散，难以排查问题

---

## 二、根因分析

### 2.1 超时配置不一致

| 位置 | 代码 | 默认超时 | 问题 |
|------|------|----------|------|
| `AnalysisTask` 数据类 | `timeout_seconds: int = 300` | 300 秒 | ❌ 与提交方法不一致 |
| `submit_analysis_task` | `timeout_seconds: int = 180` | 180 秒 | ✅ 基准值 |
| `effective_timeout` | `max(180, min(timeout_seconds, 300))` | 180-300 秒 | ⚠️ 范围模糊 |
| `_wait_for_analysis_complete` | `timeout_seconds: int = 180` | 180 秒 | ⚠️ 无余量 |

### 2.2 analysis_complete 判断逻辑缺陷

**修复前代码**:
```python
all_completed = all(task.status == TaskStatus.COMPLETED for task in tasks)
return {
    'analysis_complete': all_completed,  # ❌ 只有全部 COMPLETED 才为 True
    ...
}
```

**问题**:
- 如果有任务失败 (FAILED) 或超时 (TIMEOUT)，`analysis_complete` 始终为 `False`
- 编排器会一直等待，直到 180 秒超时
- 即使后台任务已经终止（失败/超时），编排器也不知道应该继续

### 2.3 超时/失败任务 result 可能为 None

**修复前代码**:
```python
except TimeoutError as e:
    task.status = TaskStatus.TIMEOUT
    task.error = str(e)
    task.completed_at = datetime.now()
    # ❌ task.result 保持为 None
```

**问题**: 结果聚合时，`if task.result` 判断为 False，导致数据丢失

## 三、修复方案

### 3.1 修复清单

| 优先级 | 修复项 | 修改文件 | 状态 |
|--------|--------|----------|------|
| P0 | 统一 `AnalysisTask` 默认超时为 180 秒 | `background_service_manager.py` | ✅ |
| P0 | 改进 `analysis_complete` 判断逻辑 | `background_service_manager.py` | ✅ |
| P1 | 编排器等待超时增加到 210 秒 | `diagnosis_orchestrator.py` | ✅ |
| P1 | 确保超时/失败任务 result 不为 None | `background_service_manager.py` | ✅ |

### 3.2 详细修复

#### 修复 1: 统一 AnalysisTask 默认超时

**文件**: `backend_python/wechat_backend/services/background_service_manager.py`

**修改前**:
```python
@dataclass
class AnalysisTask:
    timeout_seconds: int = 300  # 默认 5 分钟超时
```

**修改后**:
```python
@dataclass
class AnalysisTask:
    timeout_seconds: int = 180  # 【P0 修复】默认 3 分钟超时，与 submit_analysis_task 默认值一致
```

#### 修复 2: 改进 analysis_complete 判断逻辑

**文件**: `backend_python/wechat_backend/services/background_service_manager.py`

**修改前**:
```python
all_completed = all(task.status == TaskStatus.COMPLETED for task in tasks)
return {
    'analysis_complete': all_completed,
    ...
}
```

**修改后**:
```python
# 【P0 修复】检查是否所有任务都已终止（完成/失败/超时）
all_terminated = all(
    task.status in [TaskStatus.COMPLETED, TaskStatus.FAILED, TaskStatus.TIMEOUT]
    for task in tasks
)

# 【P0 修复】检查是否至少有一个任务成功完成
has_completed = any(
    task.status == TaskStatus.COMPLETED
    for task in tasks
)

# 【P0 修复】分析结束条件：所有任务终止 + 至少一个完成
analysis_complete = all_terminated and has_completed

return {
    'analysis_complete': analysis_complete,
    'all_terminated': all_terminated,  # 新增字段
    'has_completed': has_completed,     # 新增字段
    ...
}
```

#### 修复 3: 编排器等待超时增加余量

**文件**: `backend_python/wechat_backend/services/diagnosis_orchestrator.py`

**修改前**:
```python
analysis_results = await self._wait_for_analysis_complete(timeout_seconds=180)
```

**修改后**:
```python
# 【P1 修复】等待后台分析完成（最多等待 210 秒）
# 后台任务超时 180 秒 + 30 秒余量（处理状态更新延迟）
analysis_results = await self._wait_for_analysis_complete(timeout_seconds=210)
```

#### 修复 4: 确保超时/失败任务 result 不为 None

**文件**: `backend_python/wechat_backend/services/background_service_manager.py`

**修改前**:
```python
except TimeoutError as e:
    task.status = TaskStatus.TIMEOUT
    task.error = str(e)
    task.completed_at = datetime.now()
```

**修改后**:
```python
except TimeoutError as e:
    task.status = TaskStatus.TIMEOUT
    task.error = str(e)
    task.completed_at = datetime.now()
    # 【P1 修复】确保 result 不为 None，即使超时也有部分数据
    if task.result is None:
        task.result = {'_timeout': True, '_error': str(e)}

except Exception as e:
    task.status = TaskStatus.FAILED
    task.error = str(e)
    task.completed_at = datetime.now()
    # 【P1 修复】确保 result 不为 None，即使失败也有错误信息
    if task.result is None:
        task.result = {'_failed': True, '_error': str(e)}
```
