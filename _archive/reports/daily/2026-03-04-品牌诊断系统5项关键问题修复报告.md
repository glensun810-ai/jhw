### 问题 5: WS 状态管理

**问题描述**: 前端在 connecting 状态时仍可发起新连接，导致资源浪费

**修复内容**:
- ✅ 添加 `isConnecting` 状态锁标志
- ✅ 在 `connect()` 方法开始时检查状态锁
- ✅ 连接成功/失败/超时后正确释放锁

**修改文件**: `miniprogram/services/webSocketClient.js`

**修改内容**:

1. **构造函数初始化**:
```javascript
constructor() {
  // ...
  this.state = ConnectionState.DISCONNECTED;
  // 【P0 修复 - 2026-03-04】添加连接锁标志
  this.isConnecting = false;
  // ...
}
```

2. **connect 方法增强**:
```javascript
connect(executionId, callbacks = {}) {
  // 【P0 修复 - 2026-03-04】添加状态锁，防止重复连接
  if (this.isConnecting) {
    console.log('[WebSocket] ⚠️ 正在连接中，拒绝重复连接请求');
    return true;
  }
  
  // ...
  
  try {
    // 【P0 修复 - 2026-03-04】设置连接锁
    this.isConnecting = true;
    console.log('[WebSocket] 🔒 设置连接锁，开始连接');
    
    // ... 创建连接
    
    this.socket = wx.connectSocket({
      // ...
      fail: (error) => {
        console.error('[WebSocket] 连接失败:', error);
        // 【P0 修复】释放连接锁
        this.isConnecting = false;
        this._handleConnectionFailed(error);
      }
    });
    
    // 连接超时也释放锁
    this.connectionTimeout = setTimeout(() => {
      if (this.state !== ConnectionState.CONNECTED) {
        // 【P0 修复】释放连接锁
        this.isConnecting = false;
        this._handleConnectionFailed({ code: 'TIMEOUT', message: '连接超时' });
      }
    }, CONFIG.CONNECTION_TIMEOUT);
  } catch (error) {
    console.error('[WebSocket] 创建连接失败:', error);
    // 【P0 修复】释放连接锁
    this.isConnecting = false;
    this._handleConnectionFailed(error);
    return false;
  }
}
```

3. **连接成功释放锁**:
```javascript
this.socket.onOpen(() => {
  console.log('[WebSocket] 连接已建立');
  clearTimeout(this.connectionTimeout);
  // 【P0 修复 - 2026-03-04】连接成功，释放连接锁
  this.isConnecting = false;
  this._setState(ConnectionState.CONNECTED);
  // ...
});
```

**修复效果**:
- ✅ 防止重复连接请求
- ✅ 状态锁在各种情况下正确释放
- ✅ 减少 WebSocket 连接资源浪费
