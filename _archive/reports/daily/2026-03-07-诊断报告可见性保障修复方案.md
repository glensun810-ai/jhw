# 诊断报告可见性保障修复方案

## 问题分析

### 当前问题
1. **后端严格失败判定**: 当所有 AI 调用失败时，返回 `success: False` 和空结果数组
2. **前端阻断式体验**: 
   - `index.js`: 检测到 `results.length === 0` 时弹出"无有效结果"模态框，阻止用户查看报告
   - `results.js`: 同样逻辑阻止用户查看任何诊断信息
3. **用户体验极差**: 即使有诊断配置、执行日志等元数据，用户也无法查看

### 核心原则
**任何情况下，都必须让用户看到尽可能多的信息**

即使诊断完全失败，用户也应该看到：
- ✅ 诊断配置（使用的 AI 模型、诊断问题）
- ✅ 执行日志（发生了什么、哪里失败了）
- ✅ 错误详情（具体错误信息）
- ✅ 恢复建议（如何解决问题）

## 修复方案

### 1. 后端修复：优雅降级

#### 1.1 修改 `nxm_execution_engine.py`
即使完全失败，也返回执行元数据：

```python
# 原代码（完全失败时）
return {
    'success': False,
    'execution_id': execution_id,
    'error': '所有 AI 调用均失败，未获取任何有效结果',
    'results': [],  # 空结果
}

# 修复后（即使失败也返回元数据）
return {
    'success': False,  # 保持失败状态
    'execution_id': execution_id,
    'status': 'failed',  # 明确状态
    'error': '所有 AI 调用均失败，未获取任何有效结果',
    'error_details': {  # 新增：详细错误信息
        'type': 'all_ai_calls_failed',
        'message': '所有 AI 调用均失败',
        'suggestion': '请检查 AI 平台配置或网络设置'
    },
    'results': [],  # 结果为空
    # 新增：返回执行元数据
    'execution_metadata': {
        'selected_models': selected_models,  # 使用的 AI 模型
        'custom_questions': raw_questions,   # 诊断问题
        'total_tasks': total_tasks,          # 总任务数
        'completed_tasks': 0,                # 完成任务数
        'completion_rate': 0,                # 完成率
        'execution_log': scheduler.get_execution_log()  # 执行日志
    },
    'formula': f"{len(raw_questions)} 问题 × {len(selected_models)} 模型 = {total_tasks} 次请求"
}
```

### 2. 前端修复：降级展示

#### 2.1 修改 `pages/index/index.js`
移除阻断式弹窗，允许查看已有信息：

```javascript
// 原代码（阻断式）
if (!Array.isArray(resultsToCheck) || resultsToCheck.length === 0) {
  wx.showModal({
    title: '无有效结果',  // ❌ 阻断用户
    content: '未获取到任何诊断结果...',
    showCancel: false
  });
  return;  // ❌ 阻止查看报告
}

// 修复后（降级展示）
if (!Array.isArray(resultsToCheck) || resultsToCheck.length === 0) {
  // ✅ 仍然跳转到报告页，展示可用信息
  this.navigateToReportPage(executionId, {
    hasResults: false,
    showExecutionLog: true,  // 展示执行日志
    showConfigReview: true   // 展示配置回顾
  });
  return;
}
```

#### 2.2 修改 `pages/results/results.js`
即使无结果也展示诊断信息：

```javascript
// 原代码（阻断式）
if (!Array.isArray(results) || results.length === 0) {
  this.showEmptyState({
    status: 'failed',
    error: { status: 'no_results', ... }
  });
  return;  // ❌ 只展示空状态
}

// 修复后（降级展示）
if (!Array.isArray(results) || results.length === 0) {
  // ✅ 展示失败信息 + 执行日志 + 配置回顾
  this.showFailedStateWithDetails({
    status: 'failed',
    error: resultData.error,
    executionMetadata: resultData.execution_metadata,
    showExecutionLog: true,
    showConfigReview: true
  });
  return;
}
```

### 3. 新增功能：诊断过程追溯

#### 3.1 执行日志展示组件
在报告页新增"执行日志"区域，展示：
- 每个任务的执行状态（成功/失败）
- 每个 AI 调用的响应时间
- 具体错误信息
- 配额用尽警告

#### 3.2 配置回顾区域
展示诊断配置：
- 使用的 AI 模型列表
- 诊断问题列表
- 品牌列表
- 执行时间和完成率

## 预期效果

| 场景 | 当前体验 | 修复后体验 |
|------|---------|-----------|
| 诊断完全失败 | ❌ "无有效结果"弹窗，无法查看任何信息 | ✅ 展示执行日志 + 配置回顾 + 错误详情 |
| 诊断部分失败 | ❌ 可能无法查看报告 | ✅ 展示已有结果 + 失败任务详情 |
| 诊断成功但有警告 | ⚠️ 可能显示警告弹窗 | ✅ 展示完整报告 + 警告提示 |

4. 新增配置回顾区域
