# 2026-03-02-日志系统修复报告

**问题**: TypeError: StructuredLogger.warning() got multiple values for argument 'event_type'

**根因分析**: 系统存在多套日志系统，可能导致混用

---

## 一、问题诊断

### 1.1 日志系统架构

当前系统存在**三套日志系统**：

| 日志系统 | 位置 | 类型 | 调用方式 |
|---------|------|------|---------|
| **标准 logging** | logging_config.py | logging.Logger | `api_logger.warning("msg")` |
| **UnifiedLogger** | unified_logging/entry.py | UnifiedLogger | `logger.warning("msg")` |
| **StructuredLogger** | monitoring/logging_enhancements.py | StructuredLogger | `logger.warning(event_type, "msg")` |

### 1.2 潜在问题

**问题 1**: StructuredLogger 签名不兼容

```python
# StructuredLogger.warning 签名
def warning(self, event_type: LogEventType, message: str, **kwargs):
    # 第一个参数是 event_type，第二个是 message

# 但如果这样调用会出错：
logger.warning("消息内容", event_type="EVENT_TYPE")
# ❌ "消息内容" 被赋值给 event_type 参数
# ❌ 然后又通过关键字参数传递 event_type
# ❌ 导致：got multiple values for argument 'event_type'
```

**问题 2**: 日志器可能混用

```python
# logging_config.py 导入
from wechat_backend.logging_config import api_logger

# monitoring/logging_enhancements.py 也定义了
api_logger = logging.getLogger("api")

# 如果某个地方导入了错误的 api_logger，可能导致问题
```

---

## 二、修复方案

### 2.1 统一日志调用规范

**原则**: 所有日志调用统一使用标准 logging 接口

```python
# ✅ 正确用法
api_logger.warning("警告消息")
api_logger.error("错误消息")
api_logger.info("信息消息")

# ❌ 错误用法（可能导致问题）
api_logger.warning("消息", event_type="类型")
api_logger.warning(event_type, "消息")
```

### 2.2 修复诊断 API 中的日志调用

**文件**: `diagnosis_api.py`

**修复内容**: 确保所有日志调用只传递 message 参数

```python
# 修复前（潜在问题）
api_logger.warning(f"验证服务失败：{validation_err}", event_type="VALIDATION_ERROR")

# 修复后
api_logger.warning(f"验证服务失败：{validation_err}")
```

### 2.3 添加日志调用检查

创建检查脚本，确保所有日志调用正确：

```python
# scripts/check_logger_usage.py
import re
import sys
from pathlib import Path

def check_logger_usage():
    """检查日志调用是否正确"""
    errors = []
    
    for py_file in Path('backend_python/wechat_backend').rglob('*.py'):
        content = py_file.read_text()
        
        # 检查是否有 .warning(.*,.*event_type 的调用
        if re.search(r'\.warning\([^)]*,[^)]*event_type\s*=', content):
            errors.append(f"{py_file}: 发现可疑的 warning 调用")
        
        # 检查是否有 .error(.*,.*event_type 的调用
        if re.search(r'\.error\([^)]*,[^)]*event_type\s*=', content):
            errors.append(f"{py_file}: 发现可疑的 error 调用")
    
    if errors:
        print("发现以下问题:")
        for error in errors:
            print(f"  - {error}")
        return 1
    else:
        print("✅ 日志调用检查通过")
        return 0

if __name__ == '__main__':
    sys.exit(check_logger_usage())
```
