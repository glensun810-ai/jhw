## 根本原因分析

### 问题 1: 快照表查询错误

**症状**: 验证脚本报错 `no such table: diagnosis_report_snapshots`

**原因**: 
- 验证脚本查询的表名错误：`diagnosis_report_snapshots`
- 实际表名是：`diagnosis_snapshots`

**影响**: 无法正确显示快照状态，但不影响核心功能

### 问题 2: 快照创建无异常处理

**症状**: 如果快照表不存在，`complete_report` 方法会抛出异常

**原因**: `create_snapshot` 方法没有表存在性检查和异常处理

**影响**: 快照失败会导致整个报告完成流程中断

### 问题 3: 数据完整性验证缺失

**症状**: 无法检测品牌名称为空、结果数量过少等问题

**原因**: 验证脚本缺少数据质量检查逻辑

**影响**: 问题数据无法被及时发现和修复

### 问题 4: 空品牌名称数据（历史遗留）

**症状**: 数据库中执行 ID `4aae8fab-175d-43af-a33a-548fc9e6a17b` 的品牌名称为空

**原因**: 
- 前端提交时可能未正确传递 brand_list
- 或者 DiagnosisOrchestrator 在保存时 config 参数为空

**影响**: 报告无法正确显示品牌信息，影响用户体验

```python
def calculate_data_quality_score(report, results, analysis):
    """计算数据质量评分 (0-100)"""
    score = 100
    
    # 品牌名称缺失 (-30)
    if not report.get('brand_name'):
        score -= 30
    
    # 结果数量不足 (-20)
    if len(results) < 10:
        score -= 20
    elif len(results) < 30:
        score -= 10
    
    # 分析数据缺失 (-30)
    if len(analysis) == 0:
        score -= 30
    elif len(analysis) < 5:
        score -= 15
    
    # 快照缺失 (-10，非关键)
    # 快照是可选功能，不扣分或只扣少量分数
    
    return max(0, score)
```
