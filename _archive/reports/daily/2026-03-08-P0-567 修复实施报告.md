## P0-5: 状态同步修复

### 问题描述

报告状态字段（status）与 stage 字段可能不同步，导致前端显示错误的状态信息。

**根本原因**：
1. 状态更新分散在多个函数中
2. 没有强制 status 和 stage 同时更新
3. 异常处理时状态设置不一致
4. 缺少状态验证机制

### 修复方案

#### 1. 新增状态验证方法

在 `diagnosis_report_repository.py` 中添加：

```python
def validate_state_consistency(self, execution_id: str) -> Dict[str, Any]:
    """
    验证状态一致性
    
    返回:
        {
            'is_consistent': bool,  # 状态是否一致
            'status': str,  # 当前 status
            'stage': str,  # 当前 stage
            'expected_stage': str,  # 期望的 stage
            'issues': list  # 发现的问题列表
        }
    """
```

#### 2. 新增状态修复方法

```python
def fix_inconsistent_state(self, execution_id: str) -> bool:
    """
    修复不一致的历史数据
    
    优先信任 status 字段，根据 status 修正 stage 和 is_completed
    """
```

#### 3. 统一状态更新方法

增强 `update_status_sync` 方法，确保自动推导 stage 和 progress：

```python
def update_status_sync(self, execution_id: str, status: str, 
                      progress: int = None, is_completed: bool = False) -> bool:
    """
    统一状态更新函数（确保 status 和 stage 同步）
    
    自动根据 status 推导 stage 和 progress
    """
    # 状态映射表
    status_stage_map = {
        'initializing': 'init',
        'ai_fetching': 'ai_fetching',
        'analyzing': 'analyzing',
        'completed': 'completed',
        'failed': 'failed',
        'partial_completed': 'completed'
    }
    
    # 自动推导 stage
    stage = status_stage_map.get(status, status)
    
    # 自动推导 progress
    if progress is None:
        progress_map = {
            'initializing': 0,
            'ai_fetching': 50,
            'analyzing': 80,
            'completed': 100,
            'failed': 0
        }
        progress = progress_map.get(status, 0)
    
    return self.update_status(execution_id, status, progress, stage, is_completed)
```

### 状态映射表

| status | stage | progress | is_completed |
|--------|-------|----------|-------------|
| initializing | init | 0 | false |
| ai_fetching | ai_fetching | 50 | false |
| analyzing | analyzing | 80 | false |
| completed | completed | 100 | true |
| failed | failed | 0 | false |
| partial_completed | completed | 100 | true |

## P0-6: API 配置修复

### 问题描述

`getDiagnosisReport/index.js` 第 16 行 API_BASE_URL 配置可能导致开发环境无法连接生产后端。

**根本原因**：
1. 硬编码开发环境地址 `http://localhost:5001`
2. 没有环境变量支持
3. 缺少配置验证
4. 开发/生产环境配置混用

### 修复方案

#### 1. 使用环境变量配置

```javascript
// 配置优先级：环境变量 > 云函数配置 > 默认值
const API_BASE_URL = (function() {
  // 1. 优先使用环境变量（云函数控制台配置）
  if (process.env.API_BASE_URL) {
    console.log('[配置] 使用环境变量 API_BASE_URL:', process.env.API_BASE_URL);
    return process.env.API_BASE_URL;
  }
  
  // 2. 生产环境配置
  const PRODUCTION_API_URL = 'https://api.your-domain.com';
  
  // 3. 开发环境配置
  const DEVELOPMENT_API_URL = 'http://localhost:5001';
  
  // 根据云函数环境判断
  const env = process.env.WX_CONTEXT_ENV_ID || 'production';
  if (env.includes('dev') || env.includes('test')) {
    return DEVELOPMENT_API_URL;
  }
  
  return PRODUCTION_API_URL;
})();
```

#### 2. 添加配置验证

```javascript
function validateApiConfig() {
  if (!API_BASE_URL) {
    console.error('[配置错误] API_BASE_URL 未配置');
    return false;
  }
  
  if (API_BASE_URL === 'YOUR_API_URL_HERE' || 
      API_BASE_URL === 'https://api.example.com' ||
      API_BASE_URL === 'http://localhost:5001') {
    console.warn('[配置警告] API_BASE_URL 使用默认值，可能无法连接');
  }
  
  // 验证 URL 格式
  try {
    new URL(API_BASE_URL);
    return true;
  } catch (e) {
    console.error('[配置错误] API_BASE_URL 格式不正确:', API_BASE_URL);
    return false;
  }
}

// 在模块加载时验证配置
const isConfigValid = validateApiConfig();
```

## P0-7: 数据格式统一化修复

### 问题描述

多处存在 snake_case 与 camelCase 转换逻辑，可能导致前端解析失败。

**根本原因**：
1. 多个文件定义了各自的转换函数
2. 转换逻辑不一致
3. 没有统一的转换模块
4. 缺少命名规范文档

### 修复方案

#### 1. 统一转换模块

使用已有的 `backend_python/utils/field_converter.py` 作为唯一转换模块：

```python
from utils.field_converter import (
    convert_response_to_camel,  # 后端响应转 camelCase
    convert_request_to_snake,   # 前端请求转 snake_case
    to_camel_case,              # 单个字段转 camelCase
    to_snake_case               # 单个字段转 snake_case
)
```

#### 2. 核心函数

**to_camel_case**: 将 snake_case 转换为 camelCase
```python
@lru_cache(maxsize=256)
def to_camel_case(snake_str: str) -> str:
    """
    Convert snake_case to camelCase
    
    Examples:
        >>> to_camel_case('execution_id')
        'executionId'
        >>> to_camel_case('brand_name')
        'brandName'
    """
```

**convert_response_to_camel**: 递归转换字典
```python
def convert_response_to_camel(data: Any) -> Any:
    """
    Recursively convert dictionary keys from snake_case to camelCase
    
    Examples:
        >>> convert_response_to_camel({'execution_id': '123'})
        {'executionId': '123'}
        
        >>> convert_response_to_camel({
        ...     'brand_name': 'Test',
        ...     'selected_models': [{'model_name': 'a'}]
        ... })
        {'brandName': 'Test', 'selectedModels': [{'modelName': 'a'}]}
    """
```

#### 3. 命名约定

| 层级 | 命名风格 | 示例 |
|------|---------|------|
| 数据库字段 | snake_case | `execution_id`, `brand_name` |
| Python 变量 | snake_case | `report_id`, `user_id` |
| API 响应 | camelCase | `executionId`, `brandName` |
| 前端请求 | camelCase | `executionId`, `brandName` |
| 前端变量 | camelCase | `executionId`, `reportData` |

### P0-5 回滚

如果状态同步修复导致问题，可以：

1. 使用旧版 `update_status()` 方法
2. 手动修复不一致数据：

```sql
UPDATE diagnosis_reports 
SET stage = status, 
    is_completed = CASE WHEN status = 'completed' THEN 1 ELSE 0 END
WHERE status != stage;
```
