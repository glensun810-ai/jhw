## P2 任务：优化同步状态字段返回逻辑

### 问题描述

**修复前**:
```python
# 始终返回 cache_sync_status 字段
'cache_sync_status': cache_sync_status,
```

**问题**:
- 前端始终收到 `cache_sync_status` 字段
- 即使同步正常也返回 `"synced"` 状态
- 可能引起前端误解和不必要的担忧

### 修复内容

**修复文件**: `backend_python/wechat_backend/views/diagnosis_views.py`

**修复代码** (第 2854 行):
```python
# 【P2 优化 - 2026-03-03】只在异常时返回同步状态字段，减少前端误解
**({'cache_sync_status': cache_sync_status} if cache_sync_status == 'out_of_sync' else {}),
```

**修复逻辑**:
| 同步状态 | 修复前 | 修复后 |
|---------|--------|--------|
| `out_of_sync` | 返回字段 | ✅ 返回字段 |
| `synced` | 返回字段 | ❌ 不返回 |
| `cache_miss` | 返回字段 | ❌ 不返回 |
| `unknown` | 返回字段 | ❌ 不返回 |

### 验证结果

```bash
# 语法检查
✅ 语法检查通过

# 优化验证
✅ P2 优化已应用：cache_sync_status 条件返回
```

### 影响分析

**正面影响**:
✅ 减少响应数据大小（正常情况下减少 1 个字段）
✅ 避免前端误解（只在异常时返回）
✅ 符合"静默成功"设计原则

**兼容性**:
⚠️ 前端需要处理字段不存在的情况
