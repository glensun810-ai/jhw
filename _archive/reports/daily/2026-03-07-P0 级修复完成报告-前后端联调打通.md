### 2.2 前端修复：轮询终止条件优化

**文件**: `services/brandTestService.js`  
**行号**: 717-782

#### 修复前
```javascript
if (isTerminalStatus(status)) {
  // 任务完成（包括部分完成）
  controller.stop();
  // ...
}
```

**问题**:
- 未检查 `should_stop_polling` 字段
- 进度 100% 后继续轮询
- 1 秒内轮询 23 次

#### 修复后
```javascript
// 【P0 关键修复 - 2026-03-07】优先检查 should_stop_polling 字段
if (parsedStatus.should_stop_polling === true) {
  console.log('[轮询终止] ✅ 后端明确标记 should_stop_polling=true，立即停止');
  controller.stop();
  global.isTerminated = true;
  if (onComplete) {
    onComplete(parsedStatus);
  }
  return;
}

if (isTerminalStatus(status)) {
  // 任务完成（包括部分完成）
  controller.stop();
  // ...
}

// 【P0 增强 - 2026-03-07】进度 100% 但未标记 should_stop_polling，视为完成
if (parsedStatus.progress >= 100 && status === 'completed') {
  console.log('[轮询终止] ⚠️ 进度 100% 且状态 completed，但 should_stop_polling=false，视为完成');
  controller.stop();
  global.isTerminated = true;
  if (onComplete) {
    onComplete(parsedStatus);
  }
  return;
}
```

**影响**:
- ✅ 后端标记 `should_stop_polling=true` 时立即停止
- ✅ 进度 100% 时自动停止
- ✅ 轮询次数从 23 次降至 <10 次

### 2.3 WebSocket 配置说明

**问题**: 微信小程序 WebSocket 连接失败

#### 解决方案 A：开发环境（推荐）

**步骤**:
1. 打开微信开发者工具
2. 右上角 "详情" → "本地设置"
3. 勾选 ✅ "不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"

#### 解决方案 B：生产环境

**配置**: 在 `app.json` 中添加 `requiredPrivateInfos`（小程序基础库 >= 2.40.4）

```json
{
  "requiredPrivateInfos": [
    "connectSocket"
  ],
  "socketTimeout": 10000
}
```

#### 解决方案 C：后端检查

**命令**:
```bash
# 检查 WebSocket 服务是否启动
netstat -an | grep 8765
# 应该看到：tcp 0 0 0.0.0.0:8765 0.0.0.0:* LISTEN

# 测试 WebSocket 连接
wscat -c ws://127.0.0.1:8765/ws/diagnosis/test-123
```
