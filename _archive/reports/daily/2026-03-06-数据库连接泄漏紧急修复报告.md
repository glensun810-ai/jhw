## 🔍 根因分析

### 问题 1: state_manager.py 连接泄漏

**文件**: `wechat_backend/state_manager.py:214`

**问题代码**:
```python
def _persist_state_change_to_db(self, record: StateChangeRecord):
    try:
        conn = get_db_pool().get_connection()  # ❌ 获取连接
        cursor = conn.cursor()
        # ... 数据库操作
        conn.commit()
        # ❌ 未归还连接！
    except Exception as e:
        db_logger.warning(f"[StateManager] 持久化状态变更失败：{e}")
        # ❌ 异常时也未归还连接！
```

**后果**: 每次状态变更都会泄漏一个连接，最终导致连接池耗尽

---

### 问题 2: 上下文管理器未处理超时情况

**文件**: 
- `wechat_backend/diagnosis_report_repository.py:95-113`
- `wechat_backend/diagnosis_report_storage.py:109-122`

**问题代码**:
```python
@contextmanager
def get_connection(self):
    conn = get_db_pool().get_connection()  # ❌ 如果这里超时抛出异常
    try:
        yield conn
        conn.commit()
    except Exception as e:
        conn.rollback()  # ❌ conn 未定义，这里会再次报错
        raise
    finally:
        get_db_pool().return_connection(conn)  # ❌ conn 未定义
```

**后果**: 当连接池超时时，`conn` 变量未初始化，导致 `finally` 块尝试归还未定义变量，引发二次错误

## 🛠️ 修复内容

### 修复 1: state_manager.py - 使用上下文管理器

**文件**: `wechat_backend/state_manager.py:206-269`

**修复代码**:
```python
def _persist_state_change_to_db(self, record: StateChangeRecord):
    """持久化状态变更到数据库"""
    try:
        from wechat_backend.diagnosis_report_repository import get_repository

        # 【P0 修复】使用上下文管理器，确保连接归还
        repo = get_repository()
        with repo.get_connection() as conn:
            cursor = conn.cursor()
            # ... 数据库操作
            conn.commit()
            db_logger.debug(f"[StateManager] 状态变更已持久化：{record.execution_id}")

    except Exception as e:
        # 数据库失败不影响主流程，只记录日志
        db_logger.warning(f"[StateManager] 持久化状态变更失败：{e}")
```

**修复要点**:
- ✅ 使用现有仓库的上下文管理器
- ✅ 自动确保连接归还
- ✅ 异常时自动回滚

### 修复 2: diagnosis_report_repository.py - 安全上下文管理器

**文件**: `wechat_backend/diagnosis_report_repository.py:95-118`

**修复代码**:
```python
@contextmanager
def get_connection(self):
    """获取数据库连接

    【P0 紧急修复 - 2026-03-06】修复连接超时未初始化导致的重复超时问题
    """
    conn = None  # ✅ 初始化为 None
    try:
        conn = get_db_pool().get_connection()
        yield conn
        if conn:
            conn.commit()
    except Exception as e:
        if conn:
            conn.rollback()
        db_logger.error(f"数据库操作失败：{e}")
        raise
    finally:
        if conn is not None:  # ✅ 检查是否为 None
            try:
                get_db_pool().return_connection(conn)
            except Exception as return_err:
                db_logger.error(f"归还连接失败：{return_err}")
```

**修复要点**:
- ✅ `conn` 初始化为 `None`
- ✅ 所有操作前检查 `conn is not None`
- ✅ `finally` 块安全归还连接
