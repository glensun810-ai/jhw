## 二、详细修复方案

### 2.1 修复 1: 数据库路径不一致 (P0-DB-INIT-004)

#### 问题描述

日志显示：
```
init_db() 已完成表结构初始化
但 query_optimizer.py 报错：表创建超时，检测到当前存在的表：[]
```

**根因分析**:

不同模块计算数据库路径的方式不同，导致路径偏差：

```python
# database_core.py (正确)
# 文件位置：/.../backend_python/wechat_backend/database_core.py
DB_PATH = Path(__file__).parent.parent / 'database.db'
# 结果：/.../backend_python/database.db ✅

# query_optimizer.py (错误 - 修复前)
# 文件位置：/.../backend_python/wechat_backend/database/query_optimizer.py
DATABASE_PATH = Path(__file__).parent.parent / 'database.db'
# 结果：/.../backend_python/wechat_backend/database.db ❌
# 问题：多了一层 wechat_backend 目录！

# database_connection_pool.py (错误 - 修复前)
# 文件位置：/.../backend_python/wechat_backend/database_connection_pool.py
DB_PATH = Path(__file__).parent.parent / 'database.db'
# 结果：/.../backend_python/wechat_backend/database.db ❌
```

**路径偏差示意**:

```
正确的数据库路径:
/backend_python/database.db
          ↑
          └── database_core.py 创建表到这里

错误的数据库路径 (query_optimizer):
/backend_python/wechat_backend/database.db
          ↑                    ↑
          └── query_optimizer.py 在这里找表
                                 (找不到！)
```

#### 修复方案

**文件 1**: `wechat_backend/database/query_optimizer.py`

```python
# P0-DB-INIT-004: 直接计算 backend_python 目录，避免相对路径错误
# 此文件路径：/.../backend_python/wechat_backend/database/query_optimizer.py
# 需要上溯 3 层到 backend_python 目录
backend_root = Path(__file__).resolve().parent.parent.parent
DATABASE_PATH = backend_root / 'database.db'
```

**文件 2**: `wechat_backend/database_connection_pool.py`

```python
# P0-DB-INIT-004: 修复数据库路径计算，确保与 database_core.py 一致
# 此文件路径：/.../backend_python/wechat_backend/database_connection_pool.py
# 需要上溯 2 层到 backend_python 目录
DB_PATH = Path(__file__).resolve().parent.parent / 'database.db'
```

**关键改进**:
- 使用 `.resolve()` 确保绝对路径
- 明确注释需要上溯的层数
- 统一指向 `backend_python/database.db`

### 2.2 修复 2: Redis 连接错误处理优化 (P0-DB-INIT-005)

#### 问题描述

```
[Redis] 连接失败：Error 61 connecting to localhost:6379. Connection refused.
```

#### 修复方案

**文件**: `wechat_backend/cache/redis_cache.py`

```python
except Exception as e:
    error_msg = str(e)
    
    # P0-DB-INIT-005: 针对常见错误提供友好提示
    if "Connection refused" in error_msg or "Error 61" in error_msg:
        api_logger.warning(
            f"[Redis] 连接被拒绝 (Error 61)：{self.config.HOST}:{self.config.PORT} - "
            f"Redis 服务可能未启动。使用内存缓存降级方案。"
            f"如需启用 Redis，请运行：brew services start redis"
        )
    elif "No such file or directory" in error_msg:
        api_logger.warning(
            f"[Redis] 无法连接到 Unix socket - "
            f"Redis 可能配置为使用 socket 连接。使用内存缓存降级方案。"
        )
    else:
        api_logger.warning(
            f"[Redis] 连接失败：{e}，使用内存缓存降级方案"
        )
    
    self._client = None
    self._stats['errors'] += 1
```

**改进**:
- ✅ 区分不同错误类型，提供针对性提示
- ✅ 降低日志级别为 WARNING（非致命错误）
- ✅ 提供明确的修复命令（`brew services start redis`）
- ✅ 自动降级到内存缓存，不影响系统运行
