# BUG-002: 后端任务卡在 initializing 状态 - 修复报告

**修复日期**: 2026-03-02  
**修复优先级**: P0  
**修复状态**: ✅ 已完成  

---

## 一、问题描述

### 1.1 现象
- 诊断任务创建后，状态一直保持在 `initializing`
- 进度始终为 0%
- 前端轮询持续进行但不进入下一阶段
- 后端日志无报错，任务似乎"卡住"

### 1.2 影响范围
- 所有诊断任务
- 用户无法完成诊断流程
- 前端轮询无限持续（直到达到 500 次上限）

---

## 二、根因分析

### 2.1 问题定位

通过分析日志和代码，发现以下问题：

#### 问题 1: 线程未设置守护线程
```python
# ❌ 修复前
thread = Thread(target=run_async_test)
thread.start()
```

**风险**: 线程异常时可能导致进程挂起

#### 问题 2: 缺少线程级异常捕获
```python
# ❌ 修复前
def run_async_test():
    try:
        # ... 业务逻辑
    except Exception as e:
        # 只捕获了部分异常
```

**风险**: 未预期的异常导致线程卡死，状态无法更新

#### 问题 3: 缺少执行进度日志
```python
# ❌ 修复前
for q_idx, question in enumerate(raw_questions):
    for model_info in selected_models:
        # 无日志记录
```

**风险**: 无法定位卡点，调试困难

#### 问题 4: run_execution 异常未捕获
```python
# ❌ 修复前
execution_result = run_execution()  # 无异常保护
```

**风险**: 异常时静默失败，前端一直等待

---

## 三、修复方案

### 3.1 修复 diagnosis_views.py

#### 修复 1: 添加线程级异常捕获
```python
def run_async_test():
    """
    重构后的 NxM 执行逻辑
    【P0 修复 - 2026-03-02】增强异常处理和状态更新
    """
    thread_id = None
    try:
        # 获取线程 ID 用于调试
        import threading
        thread_id = threading.current_thread().ident
        api_logger.info(f"[AsyncTest] 异步线程启动 - execution_id={execution_id}, thread_id={thread_id}")
        
        # ... 业务逻辑
        
    except Exception as e:
        # 【P0 修复】捕获所有未预期的异常，防止线程卡死
        import traceback
        error_traceback = traceback.format_exc()
        api_logger.error(f"[AsyncTest] 未预期异常 - execution_id={execution_id}, thread_id={thread_id}: {e}")
        api_logger.error(f"[AsyncTest] 堆栈跟踪：{error_traceback}")
        
        # 更新状态为失败
        if execution_id in execution_store:
            execution_store[execution_id].update({
                'status': 'failed',
                'stage': 'failed',
                'progress': 100,
                'is_completed': True,
                'should_stop_polling': True,
                'error': f'异步任务执行失败：{str(e)}',
                'traceback': error_traceback
            })
        
        # 更新数据库状态
        try:
            from wechat_backend.state_manager import get_state_manager
            state_manager = get_state_manager(execution_store)
            state_manager.update_state(
                execution_id=execution_id,
                status='failed',
                stage='failed',
                progress=100,
                is_completed=True,
                error_message=f'异步任务执行失败：{str(e)}',
                write_to_db=True,
                ...
            )
        except Exception as state_err:
            api_logger.error(f"[AsyncTest] ⚠️ 数据库状态更新失败：{state_err}")
```

#### 修复 2: 设置守护线程
```python
# ✅ 修复后
thread = Thread(target=run_async_test, daemon=True)
thread.name = f"DiagnosisThread-{execution_id[:8]}"
thread.start()

api_logger.info(f"[P0 修复] ✅ 异步线程已启动 - execution_id={execution_id}, thread_name={thread.name}")
```

### 3.2 修复 nxm_execution_engine.py

#### 修复 3: 添加执行进度日志
```python
def run_execution():
    try:
        results = []
        completed = 0

        # P0 修复：添加详细日志
        api_logger.info(f"[NxM] 执行开始 - execution_id={execution_id}, 问题数={len(raw_questions)}, AI 平台数={len(selected_models)}")
        
        # 添加执行进度日志
        total_iterations = len(raw_questions) * len(selected_models)
        api_logger.info(f"[NxM] 预计总迭代次数：{total_iterations}")

        # 外层循环：遍历问题
        for q_idx, question in enumerate(raw_questions):
            # 添加问题进度日志
            api_logger.info(f"[NxM] 开始处理问题 {q_idx + 1}/{len(raw_questions)}: {question[:50]}...")
            
            # 内层循环：遍历模型
            for model_info in selected_models:
                model_name = model_info.get('name', '')
                
                # 添加模型进度日志
                api_logger.info(f"[NxM] 开始处理模型 {model_name}, Q{q_idx + 1}")
                
                # ... 处理逻辑
```

#### 修复 4: 捕获 run_execution 异常
```python
# 启动执行（同步方式，由上层调度器管理超时）
# 【P0 修复 - 2026-03-02】添加执行日志和异常捕获
api_logger.info(f"[NxM] 开始执行 run_execution - execution_id={execution_id}")
try:
    execution_result = run_execution()
    api_logger.info(f"[NxM] 执行完成 - execution_id={execution_id}, success={execution_result.get('success', False)}")
except Exception as e:
    # 【P0 修复】捕获 run_execution 的异常
    api_logger.error(f"[NxM] 执行异常 - execution_id={execution_id}, error={e}")
    import traceback
    execution_result = {
        'success': False,
        'execution_id': execution_id,
        'error': f'执行异常：{str(e)}',
        'traceback': traceback.format_exc()
    }
```
