## 一、问题分析

### 1.1 原重试机制问题

**修复前代码问题**:

```python
# 问题 1: 硬编码的重试逻辑
max_retries = 3
for attempt in range(max_retries):
    saved_results = result_repo.get_by_execution_id(execution_id)
    actual_count = len(saved_results)
    
    if actual_count >= expected_count:
        break
    
    # 问题 2: 简单的线性等待
    if attempt < max_retries - 1:
        wait_time = 0.1 * (attempt + 1)  # 仅 100ms, 200ms, 300ms
        time.sleep(wait_time)
```

**问题表现**:
- 重试策略单一，无法适应不同场景
- 延迟时间固定，无法应对网络波动
- 无错误分类，所有错误同等对待
- 无指标收集，无法监控重试效果
- 代码复用性差，无法在其他地方使用

### 1.2 需求分析

1. **指数退避** - 重试延迟应呈指数增长，避免过度请求
2. **抖动机制** - 添加随机抖动，避免雷群效应
3. **错误分类** - 区分可重试和不可重试错误
4. **指标收集** - 记录重试次数、延迟、成功率等指标
5. **可配置性** - 支持自定义重试参数
6. **超时控制** - 设置总超时时间，避免无限重试

### 2.3 延迟计算优化

#### _calculate_delay

**文件**: `result_validator.py:298-319`

```python
def _calculate_delay(self, attempt: int) -> float:
    """计算重试延迟时间（指数退避 + 抖动）"""
    # 指数退避
    delay = self.retry_config.base_delay * (self.retry_config.exponential_base ** attempt)
    
    # 限制最大延迟
    delay = min(delay, self.retry_config.max_delay)
    
    # 添加抖动（确保不超过最大延迟）
    if self.retry_config.jitter > 0:
        jitter_range = delay * self.retry_config.jitter
        delay += random.uniform(-jitter_range, jitter_range)
        delay = max(0.01, min(delay, self.retry_config.max_delay))
    
    return delay
```

**延迟序列示例** (base_delay=0.1s, exponential_base=2.0):
- Attempt 0: 0.1s ± 10%
- Attempt 1: 0.2s ± 10%
- Attempt 2: 0.4s ± 10%
- Attempt 3: 0.8s ± 10% (max 2.0s)

### 2.4 错误识别优化

#### _is_retryable_error

**文件**: `result_validator.py:321-361`

```python
def _is_retryable_error(
    self,
    error: Exception,
    custom_checker: Optional[Callable[[Exception], bool]] = None
) -> bool:
    """判断错误是否可重试"""
    # VisibilityDelayError 总是可重试
    if isinstance(error, VisibilityDelayError):
        return True
    
    # 检查错误消息模式
    error_str = str(error).lower()
    retryable_patterns = [
        'database is locked', 'locked', 'timeout', 'timed out',
        'connection reset', 'connection refused', 'network',
        'busy', 'temporary', 'visibility', '数据可见性',
    ]
    
    for pattern in retryable_patterns:
        if pattern in error_str:
            return True
    
    return False
```

**可重试错误类型**:
- 数据库锁定 (database is locked, locked, busy)
- 超时错误 (timeout, timed out)
- 连接错误 (connection reset, connection refused, network)
- 临时错误 (temporary)
- 可见性延迟 (visibility, 数据可见性)

**不可重试错误类型**:
- 数据验证错误 (ValueError, TypeError)
- 约束违反
- SQL 语法错误

### 2.5 Validate 方法集成

**文件**: `result_validator.py:397-495`

```python
def validate(
    self,
    execution_id: str,
    expected_results: List[Dict[str, Any]],
    validate_quality: bool = True,
    validate_completeness: bool = True
) -> Tuple[ValidationResult, List[Dict[str, Any]]]:
    """执行完整验证流程（优化重试机制版本）"""
    
    def fetch_results():
        """获取结果的闭包函数"""
        saved_results = result_repo.get_by_execution_id(execution_id)
        actual_count = len(saved_results)
        
        if actual_count < expected_count:
            raise VisibilityDelayError(
                f"数据可见性延迟：expected={expected_count}, actual={actual_count}"
            )
        
        return saved_results
    
    # 执行带重试的获取操作
    saved_results, retry_metrics = self._retry_with_exponential_backoff(
        operation=fetch_results,
        operation_name="数据可见性检查",
        execution_id=execution_id
    )
    
    # 存储重试指标
    self._retry_metrics[execution_id] = retry_metrics
```

**返回值变化**:
- **旧版本**: `ValidationResult`
- **新版本**: `Tuple[ValidationResult, List[Dict[str, Any]]]`
  - 第一项：验证结果
  - 第二项：数据库中的实际结果列表
