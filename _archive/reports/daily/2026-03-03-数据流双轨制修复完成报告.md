## 🔍 问题根因分析

### 核心问题：数据流双轨制

在诊断流程中，存在**两条独立的数据流轨道**：

| 轨道 | 数据源 | 使用环节 | 风险 |
|------|--------|----------|------|
| **内存轨道** | AI 调用返回的 `results` | 验证、分析、聚合 | 数据可能与数据库不一致 |
| **数据库轨道** | 保存到数据库的结果 | 仅保存环节 | 保存后不再被读取 |

### 问题示意图

```
AI 调用完成
    ↓
results (内存数据) ──┬──→ 验证环节 ←── saved_results (数据库数据) 【❌ 验证后仍使用内存数据】
                     ├──→ 后台分析 ←── results (内存数据) 【❌ 不使用数据库数据】
                     └──→ 报告聚合 ←── results (内存数据) 【❌ 不使用数据库数据】
```

### 潜在风险

1. **数据不一致**: 如果数据库保存失败或部分保存，后续环节无法发现
2. **验证失效**: 验证通过的是数据库数据，但后续使用的是内存数据
3. **分析错误**: 后台分析基于可能不完整的内存数据
4. **报告偏差**: 报告聚合使用与分析相同的问题数据

### 阶段三：优化后台分析等待逻辑 (P1)

**目标**: 增加超时时间，优化轮询频率，确保分析完成

#### 修改 5: `diagnosis_orchestrator.py` - 后台分析等待

**修改内容**:
```python
async def _wait_for_analysis_complete(
    self,
    timeout_seconds: int = 120
) -> Dict[str, Any]:
    """
    等待后台分析完成
    
    【P1 关键修复 - 2026-03-03】
    1. 超时时间从 60 秒增加到 120 秒
    2. 轮询间隔从 2 秒改为 1 秒（更快响应）
    3. 添加详细日志，便于问题排查
    4. 增加错误状态检测，提前退出
    """
    start_time = datetime.now()
    check_interval = 1  # 【P1 修复】轮询间隔从 2 秒改为 1 秒
    
    api_logger.info(
        f"[Orchestrator] 开始等待后台分析：{self.execution_id}, "
        f"超时={timeout_seconds}秒，轮询间隔={check_interval}秒"
    )
    
    while (datetime.now() - start_time).total_seconds() < timeout_seconds:
        try:
            manager = get_background_service_manager()
            status = manager.get_task_status(self.execution_id)
            
            if status:
                if status.get('analysis_complete'):
                    api_logger.info(
                        f"[Orchestrator] ✅ 后台分析完成：{self.execution_id}"
                    )
                    return status.get('analysis_results', {})
                elif status.get('error'):
                    # 【P1 新增】后台分析出错，提前退出
                    api_logger.warning(
                        f"[Orchestrator] ⚠️ 后台分析出错：{self.execution_id}"
                    )
                    return {}
        
        except Exception as e:
            api_logger.debug(f"[Orchestrator] 检查后台分析状态失败：{e}")
        
        await asyncio.sleep(check_interval)
    
    # 超时
    api_logger.warning(
        f"[Orchestrator] ⏰ 后台分析超时：{self.execution_id}, "
        f"超时={timeout_seconds}秒"
    )
    return {}
```
