## 六、问题根因分析

### 6.1 核心问题：结果未保存

**可能原因**:

1. **AI 调用执行失败**
   - 豆包 API 密钥格式错误 → 401 认证失败
   - 文心一言未配置 → 跳过调用
   - 其他平台可能也存在配置问题

2. **结果保存逻辑缺陷**
   - NxM 执行引擎可能未正确调用保存方法
   - 保存逻辑可能存在异常被捕获但未记录
   - 事务可能回滚

3. **异步执行问题**
   - 异步任务可能未完成就被终止
   - 后台执行线程可能崩溃
   - 超时保护可能过早触发

### 6.2 诊断流程断点

**正常流程**:
```
1. 接收请求 → ✅ 成功
2. 创建任务记录 → ✅ 成功 (diagnosis_reports 有记录)
3. 调用 AI 平台 → ❓ 未知 (日志缺失)
4. 保存 AI 响应 → ❌ 失败 (diagnosis_results 为空)
5. 更新任务状态 → ✅ 执行了 (但多为 failed/timeout)
6. 返回结果给前端 → ❓ 未知
```

**断点位置**: **步骤 3-4 之间**
