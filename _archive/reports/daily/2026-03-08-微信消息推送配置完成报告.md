## 📝 四、技术实现说明

### 4.1 微信消息加密原理

微信使用 AES 加密算法对消息进行加密，具体流程：

1. **密钥处理**: EncodingAESKey 经过 Base64 解码后作为 AES 密钥
2. **加密模式**: AES-CBC 模式，使用前 16 字节作为 IV
3. **填充方式**: PKCS7 填充
4. **消息格式**: 16 字节随机前缀 + 原始消息 + 填充

### 4.2 签名验证算法

**GET 请求验证** (服务器配置验证):
```
signature = SHA1(Token + Timestamp + Nonce)  // 字典序排序后拼接
```

**POST 请求验证** (消息加密验证):
```
msg_signature = SHA1(EncodingAESKey + Timestamp + Nonce + Encrypt)  // 字典序排序后拼接
```

### 4.3 消息处理流程

```
微信服务器 → ngrok 隧道 → Flask 应用 → 消息验证 → 消息解密 → 业务处理
```

## 📌 六、常见问题排查

### 6.1 签名验证失败

**现象**: 微信公众平台提示「验证失败」

**排查步骤**:
1. 确认 Token 配置一致
2. 检查服务器时间是否同步
3. 查看日志中的签名计算结果

### 6.2 消息解密失败

**现象**: 收到消息但无法解密

**排查步骤**:
1. 确认 EncodingAESKey 配置正确（43 位字符）
2. 检查 pycryptodome 库是否安装
3. 验证 msg_signature 签名

### 6.3 ngrok 连接问题

**现象**: 微信服务器无法访问

**排查步骤**:
1. 确认 ngrok 隧道已启动
2. 检查 URL 是否匹配
3. 验证网络连接
