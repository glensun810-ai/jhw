## 🛠️ 故障排查

### 问题 1: Redis 连接失败

**症状**:
```
❌ [Redis] 连接被拒绝 (Error 61)：localhost:6379
```

**解决方案**:
```bash
# 1. 检查 Redis 服务状态
brew services list | grep redis

# 2. 如果未启动，启动 Redis
brew services start redis

# 3. 如果已启动，重启 Redis
brew services restart redis

# 4. 检查端口占用
lsof -i :6379

# 5. 查看 Redis 日志
tail -f /opt/homebrew/var/log/redis/redis.log
```

---

### 问题 2: Flask 双重初始化

**症状**:
- 日志中出现两次数据库初始化
- 调度器启动两次
- 缓存预热执行两次

**解决方案**:
```bash
# 1. 检查 app.py 配置
grep "use_reloader" backend_python/wechat_backend/app.py

# 应该看到：use_reloader=False

# 2. 如果配置正确但依然双重初始化，检查进程
ps aux | grep python

# 3. 杀死所有相关进程，重新启动
pkill -f "python.*app.py"
python backend_python/wechat_backend/app.py
```

---

### 问题 3: 调度器重复启动警告

**症状**:
```
[调度器] 已启动，跳过重复启动
[BackgroundService] 服务已初始化，跳过重复初始化
```

**说明**: 这是**正常防护机制**在起作用，表明 Flask Debug 模式导致双重初始化。

**解决方案**:
1. 确认 `use_reloader=False` 已配置
2. 重启应用
3. 如果警告消失，说明防护机制正常工作

---

### 问题 4: 数据库锁定错误

**症状**:
```
sqlite3.OperationalError: database is locked
```

**解决方案**:
```bash
# 1. 检查是否有多个进程访问数据库
lsof | grep database.db

# 2. 关闭应用，清理 WAL 文件
rm -f /tmp/nxm_wal/*.pkl

# 3. 检查数据库文件权限
ls -la database.db

# 4. 如果问题持续，重启应用
```
