## 问题一：WebSocket 连接完全失效

### 1.1 用户报告的现象

```
日志显示 WebSocket 始终处于"重连 - 失败 - 再重连"的死循环中。

错误现象：connectSocket:fail invalid url "wxs://your-dev-ws-server.com/..."

原因分析：
- 占位符未替换：your-dev-ws-server.com 显然是一个开发占位符
- 协议不支持：小程序环境需要 wss:// 协议，但使用 wxs://
```

### 1.2 当前代码分析

**文件**: `miniprogram/services/webSocketClient.js`

**问题代码** (第 30-48 行):
```javascript
function getWebSocketBaseUrl() {
  if (WS_SERVER_BASE) {
    return WS_SERVER_BASE;
  }

  const envVersion = __wxConfig?.envVersion || 'release';
  const envId = __wxConfig?.envId || 'your-env-id';

  // 微信小程序 WebSocket 地址格式
  if (envVersion === 'release') {
    WS_SERVER_BASE = `wxs://${envId}.ws.tencentcloudapi.com`;  // ❌ 问题 1
  } else if (envVersion === 'trial') {
    WS_SERVER_BASE = `wxs://${envId}.ws.tencentcloudapi.com`;  // ❌ 问题 1
  } else {
    // 开发版可以使用 IP 或域名
    WS_SERVER_BASE = 'wxs://your-dev-ws-server.com';  // ❌ 问题 2
  }

  return WS_SERVER_BASE;
}
```

### 1.3 问题分析

#### ❌ 问题 1：协议错误
- **现状**: 代码使用 `wxs://` 协议
- **正确做法**: 微信小程序应使用 `wss://` (加密) 或 `ws://` (本地开发)
- **影响**: 小程序环境无法建立 WebSocket 连接

#### ❌ 问题 2：占位符未替换
- **现状**: 开发环境配置为 `wxs://your-dev-ws-server.com`
- **影响**: 开发环境下 WebSocket 连接必然失败

#### ❌ 问题 3：EnvId 占位符
- **现状**: `envId` 默认值为 `your-env-id`
- **影响**: 即使协议正确，域名也是无效的

### 1.4 后端状态检查

**日志验证**:

**结论**: 
- ✅ **后端 WebSocket 服务已正常启动** (P0 修复已生效)
- ❌ **前端配置错误导致无法连接**
