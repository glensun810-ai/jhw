## 二、问题识别

### 2.1 P0 关键问题

#### 问题 1：数据不完整 - 平均每报告仅 3 条结果

**现象**：
- 数据库统计显示平均每报告 2.97 条结果
- 预期应为 12 条（4 模型 × 3 问题）
- 最新诊断（2026-03）每报告仅 1 条结果

**根因分析**：
```sql
-- 最近诊断配置检查
SELECT execution_id, selected_models, custom_questions 
FROM diagnosis_reports 
WHERE created_at >= '2026-03-15' 
LIMIT 5;

-- 结果：
-- selected_models: [{"name": "deepseek", "checked": true}]  ← 只选 1 个模型
-- custom_questions: ["深圳新能源汽车改装门店推荐？"]  ← 只选 1 个问题
-- 结果：1 模型 × 1 问题 = 1 条结果
```

**影响**：
- 用户体验：报告内容单薄，价值感低
- 商业价值：无法充分体现多模型对比优势
- 数据分析：样本量不足，统计意义弱

**修复方案**：
1. 前端默认勾选全部 4 个模型
2. 前端默认填充 3 个标准问题
3. 后端验证最少模型/问题数

---

#### 问题 2：历史数据与实时数据割裂

**现象**：
- 2026-02 期间数据：50 条（同步脚本恢复）
- 2026-03 期间数据：117 条（实际诊断）

**影响**：

- 搜索功能无法覆盖全部历史

**修复方案**：

#### 任务 2：实现核心指标计算

**目标**：填充核心指标卡数据

**实施步骤**：
1. 定义指标计算公式
   ```python
   # 声量份额 (Share of Voice)
   sov = (品牌提及数 / 总提及数) × 100
   
   # 情感得分 (Sentiment Score)
   sentiment = (正面数 - 负面数) / 总提及数
   
   # 物理排名 (Physical Rank)
   rank = 按提及数排序的品牌位次
   
   # 影响力得分 (Influence Score)
   influence = sov × 0.4 + sentiment × 100 × 0.3 + (1/rank) × 100 × 0.3
   ```

2. 实现计算服务
   ```python
   # services/metrics_calculator.py
   class MetricsCalculator:
       def calculate_sov(self, brand, all_brands):
           ...
       
       def calculate_sentiment(self, results):
           ...
   ```

3. 前端数据绑定
   ```javascript
   // history-detail.js
   this.setData({
     sovShare: report.metrics?.sov || 0,
     sentimentScore: report.metrics?.sentiment || 0,
     physicalRank: report.metrics?.rank || 0,
     influenceScore: report.metrics?.influence || 0
   })
   ```

**验收标准**：
- 核心指标卡显示真实数据
- 数据与数据库一致

```
当前架构:
用户请求 → API → 执行引擎 → AI 调用 → 数据库
```
