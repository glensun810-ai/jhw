# P0 紧急修复：任务卡死在 initializing 状态

**问题发现时间**: 2026-03-02 02:29  
**影响范围**: 所有诊断任务  
**严重等级**: 🔴 P0 (紧急)

---

## 一、问题现象

### 1.1 前端日志证据

```javascript
// 任务创建时间：2026-03-02T02:28:35.183493
// 当前时间：2026-03-02 02:29:45
```

### 1.2 核心问题

| 问题 | 期望值 | 实际值 | 影响 |
|------|--------|--------|------|
| **任务状态** | `processing` → `ai_fetching` | 一直 `initializing` | 前端无法进入下一阶段 |
| **进度** | 逐步增长 | 始终 0% | 轮询间隔无法动态调整 |
| **is_completed** | `false` (布尔) | `0` (数字) | 状态判断可能失效 |
| **轮询次数** | < 50 次 | 221 次+ | 无效请求堆积 |

## 二、根因分析

### 2.1 后端异步任务执行流程

```
/api/perform-brand-test (POST)
    ↓
创建 execution_store 记录 (status: initializing)
    ↓
启动异步线程 run_async_test()
    ↓
初始化 QuestionManager ✅
    ↓
验证自定义问题 ✅
    ↓
调用 execute_nxm_test() ❌ 可能在此卡住
    ↓
[卡死] 未更新状态，未抛出异常
```

### 2.2 可能原因

#### 原因 1: AI API 调用阻塞 ⭐⭐⭐⭐⭐

**位置**: `nxm_execution_engine.py:320`

```python
# M002 改造：使用 FaultTolerantExecutor 统一包裹 AI 调用
ai_executor = FaultTolerantExecutor(timeout_seconds=timeout)

# 如果 FaultTolerantExecutor 内部异常未捕获，可能导致：
# 1. 线程阻塞
# 2. 状态未更新
# 3. 异常未抛出
```

**证据**:
- 任务卡在 `initializing` 状态
- execution_store 未更新为 `failed`
- 无错误日志

#### 原因 2: 状态更新逻辑缺失 ⭐⭐⭐⭐

**位置**: `diagnosis_views.py:502`

```python
# 发现无效响应时的处理
if results and len(invalid_results) == len(results):
    # 更新内存状态
    if execution_id in execution_store:
        execution_store[execution_id].update({
            'status': 'failed',
            'stage': 'failed',
            'progress': 100,
            'is_completed': True,
            'should_stop_polling': True,  # ✅ 有 should_stop_polling
            'error': error_message
        })
```

**问题**:
- 只检查"所有结果都无效"的情况
- 如果 `results` 为空列表，不会触发失败逻辑
- 异步线程可能已退出，但状态未更新

#### 原因 3: 数据类型不一致 ⭐⭐⭐

**位置**: `diagnosis_views.py:3335`

```python
# 后端返回
{
    "is_completed": 0,  // ❌ 数字 0
    "should_stop_polling": false  // ✅ 布尔值
}
```

**影响**:
- 前端 `taskStatusService.js:215` 检查 `typeof statusData.is_completed === 'boolean'`
- 数字 `0` 不会被视为完成状态

## 三、修复方案

### 3.1 P0 紧急修复：添加异步线程异常捕获

**文件**: `backend_python/wechat_backend/views/diagnosis_views.py`

**修改位置**: `run_async_test()` 函数的 try-catch 块

```python
def run_async_test():
    """异步执行诊断任务"""
    try:
        # ... 现有代码 ...
        
        # 调用 NxM 执行函数
        result = execute_nxm_test(
            execution_id=execution_id,
            main_brand=main_brand,
            competitor_brands=competitor_brands,
            selected_models=selected_models,
            raw_questions=raw_questions,
            user_id=user_id or "anonymous",
            user_level=user_level.value,
            execution_store=execution_store
        )
        
        # ... 处理结果 ...
        
    except Exception as e:
        # 【P0 修复】捕获所有未预期异常
        error_message = f"异步任务执行失败：{str(e)}"
        api_logger.error(f"[AsyncTest] 未预期异常：{error_message}", exc_info=True)
        
        # 更新内存状态
        if execution_id in execution_store:
            execution_store[execution_id].update({
                'status': 'failed',
                'stage': 'failed',
                'progress': 100,
                'is_completed': True,
                'should_stop_polling': True,  # ✅ 关键：标记停止轮询
                'error': error_message
            })
        
        # 更新数据库状态
        try:
            from wechat_backend.state_manager import get_state_manager
            state_manager = get_state_manager(execution_store)
            state_manager.update_state(
                execution_id=execution_id,
                status='failed',
                stage='failed',
                progress=100,
                is_completed=True,
                error_message=error_message,
                write_to_db=True,
                user_id=user_id or "anonymous",
                brand_name=main_brand,
                competitor_brands=competitor_brands,
                selected_models=selected_models,
                custom_questions=raw_questions
            )
        except Exception as state_err:
            api_logger.error(f"[AsyncTest] 状态更新失败：{state_err}")
    
    finally:
        # 【P0 修复】确保线程清理
        api_logger.info(f"[AsyncTest] 线程结束：{execution_id}")
```

### 3.2 P0 紧急修复：添加执行超时保护

**文件**: `backend_python/wechat_backend/nxm_execution_engine.py`

**修改位置**: `execute_nxm_test()` 函数

```python
def execute_nxm_test(..., timeout_seconds: int = 300):
    """执行 NxM 测试（增强超时保护）"""
    
    # 创建调度器
    scheduler = create_scheduler(execution_id, execution_store)
    total_tasks = len(raw_questions) * len(selected_models)
    scheduler.initialize_execution(total_tasks)
    
    # 【P0 新增】启动超时定时器
    def on_timeout():
        """超时回调"""
        error_message = f"执行超时 ({timeout_seconds}秒)"
        api_logger.error(f"[NxM] {error_message}: {execution_id}")
        
        # 更新状态
        if execution_id in execution_store:
            execution_store[execution_id].update({
                'status': 'failed',
                'stage': 'failed',
                'progress': 100,
                'is_completed': True,
                'should_stop_polling': True,  # ✅ 关键
                'error': error_message
            })
    
    scheduler.start_timeout_timer(timeout_seconds, on_timeout)
    
    # 在后台线程中执行
    def run_execution():
        start_time = time.time()
        
        try:
            # ... 现有执行逻辑 ...
            
            # 【P0 新增】每次循环检查超时
            elapsed = time.time() - start_time
            if elapsed > timeout_seconds * 0.9:  # 90% 阈值警告
                api_logger.warning(f"[NxM] 执行接近超时：{elapsed:.1f}s / {timeout_seconds}s")
            
        except Exception as e:
            api_logger.error(f"[NxM] 执行异常：{e}", exc_info=True)
            on_timeout()  # 触发超时处理
            raise
    
    # 启动后台线程
    thread = threading.Thread(target=run_execution)
    thread.daemon = True
    thread.start()
    
    return {'success': True, 'execution_id': execution_id}
```
