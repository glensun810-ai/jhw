### 问题 3: 过滤重复请求

**问题描述**: 同一 executionId 在同一毫秒发出多个状态查询请求

**修复内容**:

#### 修复：home.js 添加请求状态锁

**文件**: `api/home.js`

**新增代码**:
```javascript
// 【P0 关键修复 - 2026-03-04】添加请求状态锁，防止重复请求
const pendingRequests = new Map();

const getTaskStatusApi = (executionId) => {
  const requestKey = `status_${executionId}`;
  
  if (pendingRequests.has(requestKey)) {
    console.warn(`[API] ⚠️ 重复请求被拦截：${requestKey}`);
    return pendingRequests.get(requestKey);
  }

  const requestPromise = get(apiUrl)
    .then(result => {
      pendingRequests.delete(requestKey);
      return result;
    })
    .catch(error => {
      pendingRequests.delete(requestKey);
      throw error;
    });

  pendingRequests.set(requestKey, requestPromise);
  return requestPromise;
};
```

**修复效果**:
- ✅ 同一 executionId 的并发请求被拦截
- ✅ 返回已存在的 Promise
- ✅ 请求完成/失败后自动清理锁

**状态**: ✅ 已修复

### 问题 7: 事务提交确认

**问题描述**: AI 调用阶段被包裹在大事务中，中间进度无法写入

**验证结果**:

#### 检查 DiagnosisTransaction

**文件**: `backend_python/wechat_backend/services/diagnosis_transaction.py`

**事务流程**:
```python
def __enter__(self):
    """进入事务上下文"""
    self.status = "active"
    self.started_at = datetime.now()
    self._init_dependencies()
    db_logger.info(f"[Transaction] 事务开始：{self.execution_id}")
    return self

def __exit__(self, exc_type, exc_val, exc_tb):
    """退出事务上下文"""
    self.completed_at = datetime.now()
    
    if exc_type is not None:
        # 发生异常，自动回滚
        self.error = str(exc_val)
        if self.auto_rollback:
            self.rollback()
        self.status = "failed"
        return False
    else:
        # 执行成功，提交
        self.status = "committed"
        db_logger.info(f"[Transaction] 事务提交：{self.execution_id}")
        return False
```

**关键发现**:
1. ✅ 事务使用 `auto_rollback=True`
2. ✅ 异常时自动回滚
3. ✅ 成功时自动提交
4. ⚠️ **问题**: Phase 1/2 不写入数据库，Phase 3 才开始写入

**DiagnosisOrchestrator 事务使用**:

**结论**:
