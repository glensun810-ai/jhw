## 🔍 根因分析

### 1. 数据库连接池配置过小

```python
# 修复前
max_connections = 50          # 太小
timeout = 5.0 秒              # 太短
min_connections = 10          # 太小
```

### 2. 连接泄漏检测不灵敏

```python
# 修复前
if use_duration > 60:  # 60 秒才检测，太慢
```

### 3. 事务持有连接时间过长

- 结果保存使用单一长事务
- 批量保存时连接一直被占用
- 没有分批提交机制

### 4. 连接归还不彻底

- 归还逻辑没有检查连接健康状态
- 不健康的连接被放回池中
- 导致可用连接逐渐减少
