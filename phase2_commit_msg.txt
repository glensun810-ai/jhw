🔧 Phase 2: 优化轮询机制

## 优化内容

### 1. 动态轮询间隔调整
- 进度<20%: 4 秒轮询 (减少服务器压力)
- 进度 20-80%: 3 秒轮询 (标准间隔)
- 进度>80%: 2 秒轮询 (用户期待完成，加快反馈)

### 2. 进度停滞检测
- 检测连续 3 次进度无变化
- 自动显示等待提示
- 重置停滞计数器

### 3. 错误处理优化
- 统一轮询错误处理
- 网络异常自动重试
- 友好的错误提示

### 4. 代码结构优化
- 移除 insightPulseTimer (Phase 1 已移除相关知识显示)
- 提取 completePolling 函数
- 提取 handlePollingError 函数
- 提取 detectStalledProgress 函数

## 配置参数
pollingConfig: {
  baseInterval: 3000,        // 基础间隔 3 秒
  fastProgress: { threshold: 80, interval: 2000 },
  slowProgress: { threshold: 20, interval: 4000 },
  stalledRetry: { maxStalledCount: 3, retryInterval: 5000 }
}

## 用户体验提升
- 进度快时更快反馈 (2 秒 vs 6 秒)
- 进度停滞时主动提示
- 网络异常时自动重试
- 减少不必要的服务器请求
