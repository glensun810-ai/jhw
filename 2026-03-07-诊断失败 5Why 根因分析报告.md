## 一、问题描述

### 1.1 问题现象

用户反馈：**诊断系统无法成功输出完整详细的品牌报告**

**具体症状**:
1. 诊断任务显示"失败"或"完成"但无结果
2. `diagnosis_results` 表记录数量不足（应该有多条，实际只有 1 条或 0 条）
3. `brand` 字段全部为空字符串（应该有品牌名）
4. `tokens_used` 全部为 0（应该有实际值）
5. 前端显示"诊断失败"或空白报告

### 1.2 错误日志

```
[状态同步 - 失败处理 3/3] ✅ 错误报告已保存：37dfe794-1631-46a9-b5f5-51b8607140ab
```

**注意**: 这条日志不是错误，而是正常的错误处理流程记录！

## 二、5Why 根因分析

### 🔴 问题 1: 为什么诊断任务显示失败？

**Why 1**: 因为 NxM 执行引擎返回失败状态

**证据**:
```python
# diagnosis_views.py:824
else:
    error_message = result.get('error', '执行失败')
    api_logger.error(f"NxM execution failed for '{execution_id}': {error_message}")
    # ↓ 这里标记为失败
    await _update_diagnosis_status(
        execution_id=execution_id,
        status='failed',
        stage='failed',
        progress=100,
        error_message=error_message
    )
```

**结论**: NxM 执行引擎执行失败 → 诊断状态标记为 failed

---

### 🔴 问题 2: 为什么 NxM 执行引擎会失败？

**Why 2**: 因为 AI 调用超时或返回错误

**证据**:
```python
# nxm_concurrent_engine_v3.py:261-286
if ai_result.success:
    # AI 调用成功
    return {
        'success': True,
        'data': {
            'brand': main_brand,  # ✅ 有 brand 字段
            'question': question,
            'model': actual_model,
            'response': {...},
            'geo_data': self._parse_geo_data(ai_result.content),
            'error': None,
            'error_type': None,
            'is_objective': True
        }
    }
else:
    # AI 调用失败
    return {
        'success': False,  # ❌ 失败
        'error': ai_result.error_message,
        'brand': main_brand,
        'question': question,
        'model': model_name,
        'timeout': True,
        'latency': task_elapsed
    }
```

**可能的失败原因**:
1. **API Key 未配置或无效**
2. **API 配额用尽（429 限流）**
3. **网络连接超时**
4. **模型服务不可用**
5. **AI 响应格式错误**

### 🔴 问题 5: 为什么诊断结果数量不足？

**Why 5**: 因为以下原因之一：

#### 5.1 AI 调用部分失败

**场景**:
- 主品牌调用成功，竞品调用失败
- 部分模型调用成功，部分失败

**证据**:
```python

```

#### 5.2 结果保存失败

**检查点**:
```python
# diagnosis_transaction.py:277
def add_results_batch(self, report_id, results, batch_size=10):
    # 批量保存结果
```

## 三、根因总结

### 3.1 直接原因

| 原因 | 说明 | 严重性 |
|------|------|--------|
| AI 调用失败 | API Key 问题、限流、超时等 | 🔴 高 |
| brand 字段缺失 | 结果字典缺少 brand 字段 | 🔴 高 (已修复) |
| 结果数量不足 | 部分 AI 调用失败 | 🟠 中 |

### 3.2 根本原因

1. **API Key 配置管理不当**
   - 缺少配置验证
   - 缺少配置健康检查

2. **错误处理机制不完善**
   - 部分失败时缺少降级策略
   - 缺少重试机制

3. **数据完整性验证缺失**
   - 保存前未验证必填字段
   - 缺少端到端测试

4. **监控告警不足**
   - 缺少 API 调用失败率监控
   - 缺少数据完整性告警

```python
def validate_api_keys():
    """验证 API Key 配置"""
    required_keys = ['DOUBAO_API_KEY', 'QWEN_API_KEY', 'DEEPSEEK_API_KEY']
    missing = []
    
    for key in required_keys:
        if not os.getenv(key):
            missing.append(key)
    
    if missing:
        raise AIConfigError(f"缺少必要的 API Key 配置：{missing}")
    
    return True
```

```python
def validate_result(result: dict) -> tuple:
    """验证结果字典完整性"""
    required_fields = ['brand', 'question', 'model', 'response']
    missing = [f for f in required_fields if f not in result]
    
    if missing:
        return False, f"缺少必要字段：{missing}"
    
    # 验证 brand 不为空
    if not result.get('brand'):
        return False, "brand 字段不能为空"
    
    return True, "验证通过"
```
