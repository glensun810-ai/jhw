## 🔍 连接泄漏分析

### 泄漏特征

**日志证据**:
```log
[连接泄漏] 连接超时未归还：id=6402286080, 年龄=37.3 秒，thread=8387316736
[连接泄漏] 强制归还连接：id=6402286080
```

**分析**:
- **线程 ID**: 8387316736（固定线程）
- **泄漏频率**: 每 3-4 秒一次
- **连接年龄**: 37+ 秒（超过 30 秒阈值）
- **自动回收**: 连接池已自动强制归还

### 可能原因

1. **健康检查线程**
   - 可能是连接池健康检查线程持有连接
   - 检查后未正确归还

2. **后台任务线程**
   - 可能是缓存维护、导出清理等后台任务
   - 任务执行时间过长

3. **监控线程**
   - 连接池监控线程可能持有连接
   - 用于采集指标数据

### 排查建议

1. **启用 DEBUG 日志**
   ```bash
   # 在 .env 中设置
   LOG_CONSOLE_LEVEL=DEBUG
   LOG_FILE_LEVEL=DEBUG
   ```

2. **查看线程名称**
   ```log
   [DB] 连接获取：thread=8387316736(ThreadName), ...
   ```

3. **识别泄漏源**
   - 根据线程名称定位具体任务
   - 检查该任务的连接使用逻辑
