## 一、修复背景

### 1.1 问题现象

前端日志显示任务卡死在 `initializing` 状态：
- 任务创建时间：`02:28:35`
- 当前时间：`02:29:45`
- **已耗时 70 秒**，状态一直是 `initializing`
- 前端已轮询 **221 次**，进度始终 0%
- `is_completed: 0` (数字) 而非 `false` (布尔值)

### 1.2 核心问题

| 问题 | 期望值 | 实际值 | 影响 |
|------|--------|--------|------|
| **任务状态** | `processing` → `ai_fetching` | 一直 `initializing` | 前端无法进入下一阶段 |
| **进度** | 逐步增长 | 始终 0% | 轮询间隔无法动态调整 |
| **is_completed** | `false` (布尔) | `0` (数字) | 状态判断失效 |
| **轮询次数** | < 50 次 | 221 次+ | 无效请求堆积 |

### 2.2 前端修复

#### ✅ 修复 3: 最大轮询次数限制（brandTestService.js）

**文件**: `services/brandTestService.js`

**修改位置**: `startLegacyPolling()` 函数

**修改内容**:
```javascript
// 【P0 修复 - 2026-03-02】最大轮询次数限制（最多 500 次，约 5 分钟）
const MAX_POLL_COUNT = 500;
let pollCount = 0;

// 在轮询函数中添加检查
if (pollCount >= MAX_POLL_COUNT) {
  controller.stop();
  console.error(`[轮询终止] 达到最大轮询次数 ${MAX_POLL_COUNT}，强制停止`);
  if (onError) {
    onError(new Error(`轮询超时，已尝试 ${MAX_POLL_COUNT} 次，请检查网络或重试`));
  }
  return;
}
```

**影响**: 防止无限轮询，最多 500 次（约 5 分钟）后强制停止

#### ✅ 修复 4: 状态解析容错增强（taskStatusService.js）

**文件**: `services/taskStatusService.js`

**修改位置**: `parseTaskStatus()` 函数

**修改内容**:
```javascript
// 【P0 修复 - 2026-03-02】增强数据类型容错（后端可能返回数字 0/1 而非布尔值）
if (typeof statusData.is_completed === 'boolean') {
  parsed.is_completed = statusData.is_completed;
} else if (typeof statusData.is_completed === 'number') {
  // 后端返回数字 0/1，转换为布尔值
  parsed.is_completed = statusData.is_completed !== 0;
}

// 【P0 修复 - 2026-03-02】should_stop_polling 容错（后端可能返回数字 0/1）
if (typeof statusData.should_stop_polling === 'boolean') {
  backendShouldStopPolling = statusData.should_stop_polling;
} else if (typeof statusData.should_stop_polling === 'number') {
  backendShouldStopPolling = statusData.should_stop_polling !== 0;
}
```

**影响**: 即使后端返回数字 0/1，前端也能正确解析为布尔值
