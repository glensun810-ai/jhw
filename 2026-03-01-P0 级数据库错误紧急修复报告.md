## 根因分析

### 问题 1：SQL 查询引用了错误的表字段

**错误位置**: `wechat_backend/cache/cache_warmup_tasks.py` 第 54-68 行

**问题代码**:
```python
cursor.execute('''
    SELECT
        brand_name,
        COUNT(*) as report_count,
        AVG(
            CAST(json_extract(geo_data, '$.quality_score') AS REAL)
        ) as avg_score
    FROM diagnosis_reports  # ❌ 错误：这个表没有 geo_data 字段
    WHERE brand_name IS NOT NULL
      AND brand_name != ''
      AND brand_name != '无'
    GROUP BY brand_name
    ORDER BY report_count DESC
    LIMIT ?
''', (cache_warmup_config.WARMUP_POPULAR_BRANDS_COUNT,))
```

**真相**:
- `diagnosis_reports` 表：**没有** `geo_data` 字段
- `diagnosis_results` 表：**有** `geo_data` 字段

**数据库表结构对比**:

| 表名 | geo_data 字段 | 用途 |
|------|--------------|------|
| diagnosis_reports | ❌ 不存在 | 存储诊断报告元数据（执行 ID、品牌名、状态等） |
| diagnosis_results | ✅ 存在 | 存储具体诊断结果（包含 GEO 分析数据） |

### 问题 2：索引初始化时机问题

**原因**: 索引优化器在数据库表完全创建前就开始执行

**影响**: 索引创建被跳过，可能导致查询性能下降

---

## 修复方案

### 修复 1：修正 SQL 查询

**文件**: `wechat_backend/cache/cache_warmup_tasks.py`

**修改内容**:

1. **移除错误的 geo_data 引用** (第 54-68 行)
```python
# 修改前
cursor.execute('''
    SELECT
        brand_name,
        COUNT(*) as report_count,
        AVG(
            CAST(json_extract(geo_data, '$.quality_score') AS REAL)
        ) as avg_score
    FROM diagnosis_reports
    ...
''')

# 修改后
cursor.execute('''
    SELECT
        brand_name,
        COUNT(*) as report_count
    FROM diagnosis_reports
    WHERE brand_name IS NOT NULL
      AND brand_name != ''
      AND brand_name != '无'
    GROUP BY brand_name
    ORDER BY report_count DESC
    LIMIT ?
''')
```

2. **调整 avg_score 计算逻辑** (第 73-80 行)
```python
# 修改前
avg_score = brand['avg_score'] or 0

# 修改后
# 从 stats 中获取平均分（在 calculate_brand_stats 中从 diagnosis_results 表查询）
avg_score = stats.get('avg_quality_score', 0)
```

**修复逻辑**:
- `diagnosis_reports` 表只查询品牌名称和报告数量
- 平均分等统计数据在 `calculate_brand_stats()` 函数中从 `diagnosis_results` 表查询
- 这样既避免了字段不存在的错误，又保证了数据的正确性

### 修复 2：索引初始化重试机制（已在之前修复）

**文件**: `wechat_backend/database/query_optimizer.py`

添加了重试机制，等待表完全创建后再创建索引：
```python
max_retries = 3
retry_delay = 0.5  # 秒

for attempt in range(max_retries):
    tables = index_manager.list_tables()
    required_tables = ['users', 'test_records', 'brand_test_results', ...]
    missing_tables = [t for t in required_tables if t not in tables]
    
    if not missing_tables:
        api_logger.info(f"所有必需表已存在，开始创建索引")
        break
    else:
        time.sleep(retry_delay)
```
