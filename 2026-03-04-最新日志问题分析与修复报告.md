### 🟡 问题 4: 数据库 UNIQUE constraint 错误（需优化）

**错误日志**:
```
2026-02-26 22:37:14,587 - wechat_backend.database - ERROR - 
数据库操作失败：UNIQUE constraint failed: diagnosis_reports.execution_id
```

**问题位置**: `backend_python/wechat_backend/diagnosis_report_repository.py:104`

**根本原因**:
1. 任务失败后重试
2. 使用相同 execution_id 创建报告
3. 数据库已存在该 ID 的记录

**影响**:
- ⚠️ 日志报错（6 次）
- ✅ 有降级处理（更新而非创建）
- ⚠️ 但日志不够清晰

**优化方案**（已在之前的修复报告中提出）:
```python
# diagnosis_report_repository.py

def create_report(self, execution_id: str, user_id: str, config: dict) -> int:
    # ✅ 先检查是否已存在
    existing = self.get_by_execution_id(execution_id)
    if existing:
        db_logger.info(f"⚠️ 诊断报告已存在，返回已有记录：{execution_id}, report_id: {existing['id']}")
        return existing['id']

    # 不存在则创建
    with self.get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute('INSERT INTO diagnosis_reports (...) VALUES (...)')
        return cursor.lastrowid
```

**优先级**: P2（有降级机制，不影响功能）
