### 问题 P1: 数据库连接泄漏 🟡

**日志证据**:
```
2026-03-12 00:08:06,042 - wechat_backend.database - WARNING - 
[连接泄漏] 连接超时未归还：id=6313272528, 年龄=36.8 秒，
thread_id=8387939328, 池中剩余=0, 使用中=1

2026-03-12 00:08:06,043 - wechat_backend.database - WARNING - 
[连接泄漏] 强制归还连接：id=6313272528

2026-03-12 00:08:06,043 - wechat_backend.database - WARNING - 
[连接泄漏] 检测并修复 1 个泄漏连接
```

**问题分析**:
- **泄漏连接 ID**: 6313272528
- **泄漏时间**: 36.8 秒
- **线程 ID**: 8387939328
- **连接池状态**: 池中剩余=0, 使用中=1

**根本原因**:
1. 某个数据库操作未在 `finally` 块中关闭连接
2. 或者使用了 `with` 语句但上下文管理器有 bug
3. 长时间运行的查询未及时释放连接

**影响评估**:
- 🔴 **短期影响**: 连接池可能耗尽，导致新请求无法获取连接
- 🟡 **中期影响**: 数据库性能下降，响应时间增加
- 🟢 **长期影响**: 可能触发连接泄漏监控告警

**修复建议**:
```python
# ❌ 错误示例
def query_data():
    conn = get_connection()
    cursor = conn.cursor()
    cursor.execute("SELECT ...")  # 如果这里异常，连接未关闭
    return cursor.fetchall()

# ✅ 正确示例
def query_data():
    with get_connection() as conn:
        with conn.cursor() as cursor:
            cursor.execute("SELECT ...")
            return cursor.fetchall()
```

**优先级**: 🟡 中等（当前有自动修复机制，但需彻底解决）
