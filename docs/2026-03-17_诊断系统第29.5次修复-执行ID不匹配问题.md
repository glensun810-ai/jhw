# 诊断系统第 29.5 次修复 - 执行 ID 不匹配问题

**问题**: 前端收到 `{"uniqueId":17737523875363016}`，数据为空

**时间**: 2026-03-17 21:45

**根因**: 前端使用了云开发数据库的 `_id` 而非后端的 `execution_id`

---

## 问题分析

### 错误现象

前端日志：
```javascript
[DiagnosisService] ❌ res.data 为空，完整 res: {"uniqueId":17737523875363016}
```

### 根本原因

1. **ID 来源混乱**
   - 后端数据库使用 UUID：`30a8af4b-943a-4c29-9886-728cc9d33e92`
   - 云开发数据库使用 `_id`：`17737523875363016`（数字类型）
   - 前端可能混淆了这两种 ID

2. **数据流问题**
   ```
   后端诊断完成 → 保存到 SQLite → execution_id = UUID
   ↓
   云函数同步 → 保存到云开发 → _id = 数字 ID
   ↓
   前端历史列表 → 使用了云开发的_id → 请求后端 API → 404 报告不存在
   ```

3. **API 返回错误**
   - 后端 API 收到错误的 `executionId`（数字 ID）
   - 在数据库中找不到对应记录
   - 返回"报告不存在"错误

### 步骤 2：临时解决方案

如果 `executionId` 是数字格式，尝试从 Storage 中查找对应的 UUID：

```javascript
// 在 report-v2.js 的 initPage 方法中
async initPage(options) {
  let executionId = options.executionId;
  
  // 【P29.5 临时修复】如果是数字 ID，尝试从 Storage 中查找 UUID
  if (/^\d+$/.test(executionId)) {
    console.warn('[ReportPageV2] 检测到数字 ID，尝试从 Storage 中查找 UUID');
    const history = wx.getStorageSync('diagnosis_history_list') || [];
    const record = history.find(item => item.uniqueId === executionId || item._id === executionId);
    
    if (record && record.executionId) {
      executionId = record.executionId;
      console.log('[ReportPageV2] ✅ 找到对应的 UUID:', executionId);
    } else {
      console.error('[ReportPageV2] ❌ 无法找到对应的 UUID');
    }
  }
  
  // 继续使用 executionId 加载
  await this._loadFromAPI(executionId);
}
```
