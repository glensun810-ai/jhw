## 一、问题回顾

### 1.1 症状

```
诊断完成后，显示详细诊断报告的过程中卡死
没有成功跳转到详情页
模拟器长时间无响应
```

### 1.2 根因

**核心问题**: 一次性加载和渲染大量数据

**具体表现**:
1. 单次 setData 数据量过大 (50-250KB)
2. 渲染节点过多 (1000+ 个)
3. 同步阻塞处理

---

## 二、修复方案

### 2.1 分层加载策略

**第一层**: 核心数据（立即显示）
```javascript
this.setData({
  brandName,
  overallScore,
  overallGrade,
  overallSummary,
  loading: false  // 关键：先解除加载状态
});
```

**第二层**: 评分维度（延迟 100ms）
```javascript
setTimeout(() => {
  this.setData({
    overallAuthority, overallVisibility,
    overallPurity, overallConsistency
  });
}, 100);
```

**第三层**: 核心指标（延迟 200ms）
```javascript
setTimeout(() => {
  this.setData({
    sovShare, sentimentScore,
    physicalRank, influenceScore
  });
}, 200);
```

**第四层**: 问题诊断（延迟 300ms）
```javascript
setTimeout(() => {
  this.setData({
    highRisks, mediumRisks, suggestions
  });
}, 300);
```

**第五层**: 信源列表（延迟 400ms）
```javascript
setTimeout(() => {
  this.setData({ sourceList });
}, 400);
```

**第六层**: 详细结果（延迟 500ms，只加载前 20 条）
```javascript
setTimeout(() => {
  const limitedResults = detailedResults.slice(0, 20);
  this.setData({
    detailedResults: limitedResults,
    hasMoreResults: detailedResults.length > 20,
    totalResults: detailedResults.length
  });
}, 500);
```

### 2.2 数据精简

**简化计算**:
```javascript
// 限制计算数据量，最多计算 50 条
const limitedResults = detailedResults.slice(0, 50);
```

**分页加载**:
```javascript
// 只加载前 20 条
detailedResults: detailedResults.slice(0, 20)
```

### 2.3 加载更多功能

**UI**:
```xml
<view wx:if="{{hasMoreResults}}" class="load-more" bindtap="loadMoreResults">
  <text>加载更多 ({{totalResults - detailedResults.length}} 条)</text>
</view>
```

**逻辑**:
```javascript
loadMoreResults: function() {
  const currentLength = this.data.detailedResults.length;
  const pageSize = 20;
  const nextResults = this._getAllDetailedResults().slice(currentLength, currentLength + pageSize);
  
  this.setData({
    detailedResults: [...this.data.detailedResults, ...nextResults],
    hasMoreResults: (currentLength + nextResults.length) < this.data.totalResults
  });
}
```
