## 一、问题回顾

### 1.1 用户反馈

> "为什么用户没有选择豆包 AI 平台，但诊断结果是因为豆包 API 配置问题导致的失败？"

### 1.2 根因分析

**原代码逻辑**:
```python
# backend_python/wechat_backend/ai_adapters/factory.py:375
if platform_type == AIPlatformType.DOUBAO and DoubaoPriorityAdapter:
    if priority_models and Config.is_doubao_auto_select():
        # ❌ 错误：无条件使用豆包优先级适配器
        return DoubaoPriorityAdapter(api_key, model_name, **kwargs)
```

**问题**:
- 只要 `DOUBAO_AUTO_SELECT_MODEL=true`（默认）
- 即使用户未选择豆包
- 也会调用 `DoubaoPriorityAdapter`
- 初始化时自动调用豆包 API
- 导致豆包 API 配置错误时诊断失败

---

## 二、修复方案

### 2.1 修复逻辑

**修复后逻辑**:
```python
# 【P0 修复 - 2026-03-11】尊重用户选择
# 只有用户选择了豆包，才使用豆包优先级适配器

if platform_type == AIPlatformType.DOUBAO and DoubaoPriorityAdapter:
    if priority_models and Config.is_doubao_auto_select():
        # ✅ 修复：检查 model_name 是否在优先级模型列表中
        is_priority_model = model_name in priority_models
        
        if is_priority_model:
            # 用户选择了豆包，使用豆包优先级适配器
            return DoubaoPriorityAdapter(api_key, model_name, **kwargs)
        else:
            # 用户未选择豆包，使用普通豆包适配器
            api_logger.info(f"用户未选择豆包优先级模型 (model={model_name})")

# 使用普通适配器
return adapter_class(api_key, model_name, **kwargs)
```

### 2.2 判断条件

**使用豆包优先级适配器的条件**（需同时满足）:
1. ✅ 平台类型是豆包
2. ✅ 配置了豆包优先级模型列表
3. ✅ 启用了自动选择功能
4. ✅ **model_name 在优先级模型列表中**（新增）

**新增条件 4**:
- 确保只有用户选择了豆包，才使用豆包优先级适配器
- 如果用户未选择豆包，使用普通豆包适配器
- 普通豆包适配器不会在初始化时调用 API
