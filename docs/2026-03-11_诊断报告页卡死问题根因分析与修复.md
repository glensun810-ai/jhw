# 诊断报告页卡死问题 - 根因分析与修复

**文档版本**: v9.0  
**创建日期**: 2026-03-11  
**问题**: 模拟器长时间无响应  
**根因**: WXML 三元表达式嵌套过深 + 响应式数据绑定导致渲染阻塞  

---

## 一、问题根因

### 1.1 WXML 中的三元表达式嵌套

**问题代码**:
```xml
<!-- ❌ 错误：三元表达式嵌套过深 -->
<text class="score-value-simple {{item.score >= 80 ? 'excellent' : (item.score >= 60 ? 'good' : 'poor')}}">
  {{item.score}}分
</text>

<!-- ❌ 错误：响应式数据绑定 -->
<text class="response-text">{{item.expanded ? item.fullResponse : item.response}}</text>
```

**问题**:
1. 三元表达式嵌套 3 层 → 渲染时多次计算
2. 响应式数据绑定 → 每次 setData 都重新计算
3. 5 条结果 × 复杂计算 = 渲染阻塞

### 1.2 数据简化不足

**问题代码**:
```javascript
// ❌ 错误：保存了完整数据到全局变量
app.globalData.currentReportData = {
  detailedResults: detailedResults,  // 可能 100+ 条，每条 1000+ 字
  executionId: ...
};
```

**问题**:
- 完整数据占用大量内存
- 全局变量更新触发重新渲染
- 导致卡死

---

## 二、修复方案

### 2.1 简化 WXML 表达式

**修复前**:
```xml
<text class="score-value-simple {{item.score >= 80 ? 'excellent' : (item.score >= 60 ? 'good' : 'poor')}}">
```

**修复后**:
```xml
<!-- ✅ 正确：预先计算 class，WXML 只绑定 -->
<text class="score-value-simple {{item.scoreClass}}">
```

**JS 处理**:
```javascript
const simplifiedResults = detailedResults.slice(0, 5).map((r, index) => {
  // 预先计算 scoreClass
  let scoreClass = 'poor';
  if (r.score >= 80) scoreClass = 'excellent';
  else if (r.score >= 60) scoreClass = 'good';
  
  return {
    id: r.id || `result_${index}`,
    brand: r.brand || r.brandName,
    model: r.aiModel || r.ai_model || '未知模型',
    score: r.score,
    scoreClass: scoreClass,  // 预先计算
    question: (r.question || '').substring(0, 50),
    response: (r.response || r.answer || '').substring(0, 100),
    truncated: (r.response?.length || r.answer?.length || 0) > 100,
    fullResponse: r.response || r.answer || '',
    expanded: false
  };
});
```

### 2.2 移除响应式绑定

**修复前**:
```xml
<text class="response-text">{{item.expanded ? item.fullResponse : item.response}}</text>
```

**修复后**:
```xml
<!-- ✅ 正确：使用 wx:if 切换 -->
<text class="response-text" wx:if="{{!item.expanded}}">{{item.response}}</text>
<text class="response-text" wx:else>{{item.fullResponse}}</text>
```

### 2.3 优化全局变量

**修复前**:
```javascript
// ❌ 错误：保存完整大对象
app.globalData.currentReportData = {
  detailedResults: detailedResults  // 100+ 条
};
```

**修复后**:
```javascript
// ✅ 正确：只保存必要信息
const app = getApp();
if (!app.globalData) app.globalData = {};
app.globalData.currentReportData = {
  executionId: record.executionId,
  totalResults: detailedResults.length
  // 不保存 detailedResults，需要时再从 storage 读取
};
```

## 三、完整修复代码

### 3.1 JS 修复

```javascript
// 第 3 层：详细结果（200ms，异步简化）
setTimeout(() => {
  try {
    const detailedResults = results.detailed_results || results.detailedResults || [];
    
    // 异步简化数据（只处理前 5 条）
    const simplifiedResults = detailedResults.slice(0, 5).map((r, index) => {
      // 【性能优化】预先计算 scoreClass，避免 WXML 中三元表达式
      let scoreClass = 'poor';
      if (r.score >= 80) scoreClass = 'excellent';
      else if (r.score >= 60) scoreClass = 'good';
      
      return {
        id: r.id || `result_${index}`,
        brand: r.brand || r.brandName,
        model: r.aiModel || r.ai_model || '未知模型',
        score: r.score,
        scoreClass: scoreClass,  // ✅ 预先计算
        question: (r.question || '').substring(0, 50),
        response: (r.response || r.answer || '').substring(0, 100),
        truncated: (r.response?.length || r.answer?.length || 0) > 100,
        fullResponse: r.response || r.answer || '',
        expanded: false
      };
    });
    
    // 【性能优化】只保存必要信息到全局变量
    const app = getApp();
    if (!app.globalData) app.globalData = {};
    app.globalData.currentReportData = {
      executionId: record.executionId || record.result_id || record.id,
      totalResults: detailedResults.length
      // 不保存 detailedResults 完整数据
    };
    
    this.setData({
      detailedResults: simplifiedResults,
      aiModels: results.ai_models || [],
      competitors: results.competitors || [],
      questions: results.questions || [],
      hasMoreResults: detailedResults.length > 5,
      totalResults: detailedResults.length
    });
  } catch (error) {
    console.error('[第 3 层] 加载详细结果失败:', error);
    this.setData({
      detailedResults: [],
      hasMoreResults: false,
      totalResults: 0
    });
  }
}, 200);
```

### 3.2 WXML 修复

```xml
        </view>

        <view class="result-response" bindtap="toggleExpand" data-id="{{item.id}}">
          <text class="response-label">答：</text>
          <text class="response-text" wx:if="{{!item.expanded}}">{{item.response}}</text>
          <text class="response-text" wx:else>{{item.fullResponse}}</text>
          
          <text wx:if="{{item.truncated && !item.expanded}}" class="expand-text">... 点击展开</text>
          <text wx:if="{{item.expanded}}" class="collapse-text"> 点击收起</text>
        </view>
        
        <!-- ✅ 修复：使用预先计算的 scoreClass -->
        <view class="result-score-simple">
          <text class="score-label-simple">综合评分：</text>
          <text class="score-value-simple {{item.scoreClass}}">{{item.score}}分</text>
        </view>
      </view>
    </view>

    <!-- 加载更多 -->
    <view wx:if="{{hasMoreResults}}" class="load-more" bindtap="loadMoreResults">
      <text class="load-more-text">加载更多 ({{totalResults - detailedResults.length}} 条)</text>
    </view>

    <!-- 已全部加载 -->
    <view wx:if="{{!hasMoreResults && totalResults > 5}}" class="load-complete">
      <text class="load-complete-text">已全部加载</text>
    </view>
  </scroll-view>
</view>
```
