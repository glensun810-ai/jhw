一、完整流程梳理
1.1 用户操作流程
┌─────────────────────────────────────────────────────────────────────┐
│ 用户操作流程（前端：微信小程序）                                      │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 首页输入 (pages/index/index.js)                                   │
│    - 品牌名称：用户输入                                             │
│    - 竞争品牌：可选添加                                             │
│    - 诊断问题：默认 3 题，可编辑                                       │
│    - AI 模型选择：豆包/通义千问/DeepSeek 等（国内/海外 Tab 切换）            │
│                                                                      │
│ 2. 点击"开始诊断"按钮                                                │
│    - validateInput() 验证输入                                        │
│    - buildPayload() 构建请求体                                       │
│    - startDiagnosis() 调用云函数 startDiagnosis                      │
│    - 跳转到报告页 (pages/report-v2/report-v2.js)                     │
│                                                                      │
│ 3. 报告页等待诊断完成                                                │
│    - onLoad({executionId}) 接收执行 ID                                │
│    - startListening() 启动监听                                       │
│    - 优先 WebSocket，降级到轮询                                      │
│    - 实时显示进度（进度条 + 阶段提示）                                │
│                                                                      │
│ 4. 诊断完成跳转                                                      │
│    - handleComplete() 接收完成回调                                   │
│    - 跳转到报告详情页 (pages/report-detail/report-detail.js)         │
└─────────────────────────────────────────────────────────────────────┘
1.2 后端处理流程
┌─────────────────────────────────────────────────────────────────────┐
│ 后端处理流程（Flask + Python）                                        │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 请求接收 (views.py / diagnosis_views.py)                          │
│    POST /api/perform-brand-test                                     │
│    - @require_auth_optional 认证                                    │
│    - @rate_limit 限流                                                │
│    - validate_and_sanitize_request 数据验证                         │
│                                                                      │
│ 2. 诊断任务创建 (diagnosis_report_service.py)                        │
│    - 创建 diagnosis_reports 记录（status='initializing'）            │
│    - 创建 task_statuses 记录（progress=0, stage='init'）            │
│    - 返回 execution_id                                              │
│                                                                      │
│ 3. 异步执行诊断 (nxm_execution_engine.py)                            │
│    - execute_nxm_test() NxM 执行引擎                                  │
│    - multi_model_executor.py 多模型冗余调用                          │
│      - 主模型：用户选择的模型（如 qwen）                              │
│      - 备用模型：['doubao', 'deepseek']                              │
│    - AI 适配器调用 (ai_adapters/*.py)                                │
│      - send_prompt() 发送提示词                                      │
│      - 返回 AIResponse(success=True/False)                          │
│                                                                      │
│ 4. 结果持久化 (state_manager.py)                                     │
│    - 更新 execution_store（内存）                                    │
│    - 更新 task_statuses（数据库）                                    │
│    - 更新 diagnosis_reports（数据库）                                │
│                                                                      │
│ 5. 状态查询接口 (diagnosis_views.py)                                 │
│    GET /test/status/{execution_id}                                  │
│    - 返回当前进度、阶段、状态                                        │
│                                                                      │
│ 6. 报告获取接口 (diagnosis_report_service.py)                        │
│    GET /api/report/{execution_id}                                   │
│    - 获取完整报告（report + results + analysis）                     │
│    - 验证报告结构                                                    │
└─────────────────────────────────────────────────────────────────────┘

3.1 P0 问题：AI 调用完全失败
问题链路：
用户选择 qwen → 主模型调用失败（AIResponse.status Bug）
              → 尝试备用模型 doubao（429 配额用尽）
              → 尝试备用模型 deepseek（可能未执行）
              → 尝试备用模型 wenxin（API Key 未配置）
              → 所有模型失败 → 诊断执行失败
已修复代码（需重启生效）：
1.
AIResponse.status Bug → 改为 ai_result.success
2.
顺序尝试模型 → 逐个尝试，记录详细错误
3.
API Key 检测 → 调用前检查是否配置
4.
优化备用列表 → 移除未配置的 wenxin
3.2 P0 问题：前端轮询未启动
问题链路：
report-v2.js: setData({executionId})
            → startPolling() 未传递 executionId
            → diagnosisService.startPolling() 检查 this.currentTask 为 null
            → 抛出 "No active diagnosis task" 错误
            → 错误被 catch 静默处理
            → 轮询从未启动 → 前端卡住
已修复代码：
1.
diagnosisService.js → 支持直接传入 executionId 参数
2.
report-v2.js → 传递 this.data.executionId 给 diagnosisService
