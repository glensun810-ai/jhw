### ISSUE-001: 报告聚合层缺失 Python 实现

**问题描述**:

当前系统中，报告聚合逻辑仅存在于前端 JavaScript 文件 `services/reportAggregator.js` 中，但 `DiagnosisOrchestrator` 的阶段 6（报告聚合阶段）尝试调用 Python 版本的聚合器，导致：

1. **模块导入失败**: 编排器尝试导入 `services.reportAggregator.aggregateReport`，但该 Python 模块不存在
2. **报告生成失败**: 阶段 6 执行失败，无法生成最终报告
3. **前端数据缺失**: 前端轮询获取的报告数据不完整

**位置**: 
- `backend_python/wechat_backend/services/diagnosis_orchestrator.py` (第 6 阶段)
- `services/reportAggregator.js` (仅前端实现)

**现状代码**:
```python

final_report = aggregateReport(
    rawResults=db_results,
    brandName=brand_name,
    competitors=competitors,
    additionalData=analysis_results

```

**理想架构要求**:

根据 `2026-03-02-系统逻辑漏洞分析与架构优化计划.md` 第 2.1 节：

```

│   报告聚合层       │
```

**根本原因**:

1. 重构过程中只创建了前端聚合器，未创建对应的 Python 后端服务
2. 编排器设计时假设该服务存在，但未实际实现
3. 缺少跨语言模块的集成设计

### ISSUE-002: 后台分析任务未实际执行

**问题描述**:

`DiagnosisOrchestrator` 的阶段 5（后台分析阶段）虽然提交了任务到 `BackgroundServiceManager`，但实际分析任务并未执行，导致：

1. **品牌分析缺失**: `brand_analysis` 数据为空
2. **竞争分析缺失**: `competitive_analysis` 数据为空
3. **统计数据缺失**: 统计计算未执行

**位置**: 
- `backend_python/wechat_backend/services/diagnosis_orchestrator.py` (阶段 5)
- `backend_python/wechat_backend/services/background_service_manager.py`

```python
# background_service_manager.py

def submit_analysis_task(
    self,
    execution_id: str,
    task_type: str,
    payload: Dict[str, Any],
    timeout_seconds: int = 300
) -> str:
    """
    提交分析任务到后台队列并立即执行
    
    Args:
        execution_id: 执行 ID
        task_type: 任务类型 (brand_analysis/competitive_analysis/statistics)
        payload: 任务参数
        timeout_seconds: 超时时间（秒）
    
    Returns:
        task_id: 任务 ID
    """
    task_id = f"{execution_id}_{task_type}_{uuid.uuid4().hex[:8]}"
    
    # 创建任务记录
    task_record = AnalysisTask(
        task_id=task_id,
        execution_id=execution_id,
        task_type=task_type,
        payload=payload,
        status=TaskStatus.PENDING,
        timeout_seconds=timeout_seconds
    )
    
    with self._lock:
        self._analysis_tasks[task_id] = task_record
    
    api_logger.info(
        f"[BackgroundService] 提交分析任务：{task_id}, "
        f"type={task_type}, execution_id={execution_id}"
    )
    
    # 【关键修复】提交到线程池执行
    self._executor.submit(
        self._execute_analysis_task,
        task_id,
        task_type,
        payload,
        timeout_seconds
    )
    
    return task_id
```

**步骤 2**: 实现 `_execute_analysis_task` 方法

```python
# background_service_manager.py

def _execute_analysis_task(
    self,
    task_id: str,
    task_type: str,
    payload: Dict[str, Any],
    timeout_seconds: int
):
    """
    执行分析任务
    
    Args:
        task_id: 任务 ID
        task_type: 任务类型
        payload: 任务参数
        timeout_seconds: 超时时间
    """
    start_time = datetime.now()
    
    try:
        # 更新状态为执行中
        with self._lock:
            self._analysis_tasks[task_id].status = TaskStatus.RUNNING
            self._analysis_tasks[task_id].started_at = datetime.now()
        
        api_logger.debug(f"[BackgroundService] 开始执行任务：{task_id}")
        
        # 根据任务类型执行不同的分析
        if task_type == 'brand_analysis':
            result = self._execute_brand_analysis(payload)
        elif task_type == 'competitive_analysis':
            result = self._execute_competitive_analysis(payload)
        elif task_type == 'statistics':
            result = self._execute_statistics(payload)
        else:
            raise ValueError(f"未知任务类型：{task_type}")
        
        # 计算执行时间
        execution_time = (datetime.now() - start_time).total_seconds()
        
        # 更新状态为完成
        with self._lock:
            self._analysis_tasks[task_id].status = TaskStatus.COMPLETED
            self._analysis_tasks[task_id].completed_at = datetime.now()
            self._analysis_tasks[task_id].result = result
        
        api_logger.info(
            f"[BackgroundService] 分析任务完成：{task_id}, "
            f"type={task_type}, 耗时={execution_time:.2f}秒"
        )
        
    except Exception as e:
        api_logger.error(
            f"[BackgroundService] 分析任务失败：{task_id}, "
            f"type={task_type}, error={e}",
            exc_info=True
        )
        
        with self._lock:
            self._analysis_tasks[task_id].status = TaskStatus.FAILED
            self._analysis_tasks[task_id].error = str(e)
            self._analysis_tasks[task_id].completed_at = datetime.now()
    
    finally:
        # 清理超时任务记录（可选）
        self._cleanup_old_tasks(max_age_hours=1)
```

```python
def _execute_brand_analysis(self, payload: Dict[str, Any]) -> Dict[str, Any]:

def _execute_competitive_analysis(self, payload: Dict[str, Any]) -> Dict[str, Any]:
    """执行竞争分析"""
    from wechat_backend.services.competitive_analysis_service import get_competitive_analysis_service

    service = get_competitive_analysis_service()

    return {
        'type': 'competitive_analysis',
        'data': analysis,

def _execute_statistics(self, payload: Dict[str, Any]) -> Dict[str, Any]:
    """执行统计计算"""
    results = payload.get('results', [])
    
    # 计算基础统计

        'total_results': len(results),
        'platforms': list(set(r.get('model', 'unknown') for r in results)),

    }
    
    return {
        'type': 'statistics',
        'data': stats,
        'timestamp': datetime.now().isoformat()
    }
```

```python
# background_service_manager.py

def get_analysis_status(self, execution_id: str) -> Dict[str, Any]:
    """
    获取分析任务状态
    
    Args:
        execution_id: 执行 ID
    
    Returns:
        分析状态字典
    """
    with self._lock:
        # 查找所有属于该 execution_id 的任务
        tasks = [
            task for task in self._analysis_tasks.values()
            if task.execution_id == execution_id
        ]
    
    if not tasks:
        return {
            'execution_id': execution_id,
            'analysis_complete': True,  # 无任务视为完成
            'tasks': []
        }
    
    # 检查是否所有任务都完成
    all_completed = all(task.status == TaskStatus.COMPLETED for task in tasks)
    any_failed = any(task.status == TaskStatus.FAILED for task in tasks)
    
    # 聚合结果
    analysis_results = {}
    for task in tasks:
        if task.result:
            analysis_results[task.task_type] = task.result.data
    
    return {
        'execution_id': execution_id,
        'total_tasks': len(tasks),
        'completed_tasks': sum(1 for t in tasks if t.status == TaskStatus.COMPLETED),
        'failed_tasks': sum(1 for t in tasks if t.status == TaskStatus.FAILED),
        'analysis_complete': all_completed,
        'any_failed': any_failed,
        'analysis_results': analysis_results,
        'tasks': [
            {
                'task_id': t.task_id,
                'task_type': t.task_type,
                'status': t.status.value,
                'error': t.error
            }
            for t in tasks
        ]
    }
```

### ISSUE-003: 数据流双轨制导致状态不一致

**问题描述**:

当前系统存在**两条独立的数据流**，导致前端获取的状态和结果不一致：

1. **轨道 A**: `execution_store` (内存) → WebSocket/SSE → 前端
2. **轨道 B**: 数据库 → API 轮询 → 前端

两条轨道之间缺乏同步机制，导致：
- 内存状态显示 completed，但数据库记录仍为 running
- 前端轮询获取的数据与 WebSocket 推送不一致
- 刷新页面后数据丢失

**位置**: 
- `backend_python/wechat_backend/state_manager.py`
- `backend_python/wechat_backend/views/diagnosis_views.py`
- `backend_python/wechat_backend/services/realtime_push_service.py`

**现状分析**:

```python

    'progress': 100,
    'results': [...]  # 内存中的结果
```

**理想架构要求**:

> **状态一致**: 内存状态与数据库状态原子性更新

**修复方案**:

**步骤 1**: 增强 `DiagnosisStateManager.update_state` 方法，确保强制同步

```python
# state_manager.py

def update_state(self, execution_id, status=None, stage=None, progress=None,
                 is_completed=None, results=None, error_message=None,
                 should_stop_polling=None, write_to_db=True, **kwargs):
    """
    统一状态更新接口（增强版）
    
    【P0 关键修复 - 2026-03-04】添加强制同步标志
    """
    if execution_id not in self.state_lock:
        self.state_lock[execution_id] = threading.Lock()
    
    lock = self.state_lock[execution_id]
    
    with lock:
        try:
            # 初始化状态
            self._init_execution_state(execution_id)
            
            # 保存旧状态
            old_state = copy.deepcopy(self.execution_store[execution_id])
            self._save_snapshot(execution_id)
            
            # 更新内存状态
            if status is not None:
                self.execution_store[execution_id]['status'] = status
            if stage is not None:
                self.execution_store[execution_id]['stage'] = stage
            if progress is not None:
                self.execution_store[execution_id]['progress'] = progress
            # ... 其他字段更新 ...
            
            # 【P0 关键修复】如果 write_to_db=True，强制同步到数据库
            if write_to_db:
                self._force_sync_to_database(execution_id, **kwargs)
            
            # 记录状态变更

            return True

def _force_sync_to_database(self, execution_id, **kwargs):
    """
    强制同步内存状态到数据库
    
    【P0 关键修复 - 2026-03-04】确保内存和数据库状态一致
    """
    try:
        state = self.execution_store[execution_id]
        
        # 准备数据库更新数据
        update_data = {
            'status': state.get('status'),
            'stage': state.get('stage'),
            'progress': state.get('progress'),
            'is_completed': state.get('is_completed', False),
            'should_stop_polling': state.get('should_stop_polling', False),
            'error_message': state.get('error'),
            'updated_at': datetime.now().isoformat()
        }
        
        # 添加额外字段
        if 'brand_name' in kwargs:
            update_data['brand_name'] = kwargs['brand_name']
        if 'competitor_brands' in kwargs:
            update_data['competitor_brands'] = json.dumps(kwargs['competitor_brands'])
        if 'selected_models' in kwargs:
            update_data['selected_models'] = json.dumps(kwargs['selected_models'])
        if 'custom_questions' in kwargs:
            update_data['custom_questions'] = json.dumps(kwargs['custom_questions'])
        
        # 执行数据库更新
        from wechat_backend.diagnosis_report_repository import DiagnosisReportRepository
        
        report_repo = DiagnosisReportRepository()
        report_repo.update_by_execution_id(execution_id, update_data)
```

```python
# diagnosis_orchestrator.py - _phase_results_validating 方法增强版

async def _phase_results_validating(self, results, brand_list):
    """
    阶段 4: 结果验证（增强版）
    
    增强内容：
    1. 根据用户等级调整验证级别
    2. 验证失败时记录详细错误
    3. 处理验证警告
    """
    api_logger.info(f"[Orchestrator] 阶段 4: 结果验证 - {self.execution_id}")
    self.current_phase = DiagnosisPhase.RESULTS_VALIDATING
    
    try:
        # 更新状态
        self._update_phase_status(
            status='results_validating',
            stage='results_validating',
            progress=70,
            write_to_db=True
        )

            get_result_validator,
            ValidationLevel,
            ValidationStatus
        )

        user_level = self._initial_params.get('user_level', 'Free')

            # 付费用户使用严格模式
            validation_level = ValidationLevel.STRICT

            # 免费用户使用普通模式
            validation_level = ValidationLevel.NORMAL

        validation_result, saved_results = validator.validate(
            execution_id=self.execution_id,
            expected_results=results,
            validate_quality=True,
            validate_completeness=True
        )
        
        # 【增强 2】处理验证失败
        if validation_result.status == ValidationStatus.CRITICAL:

                error=Exception(validation_result.message),
                error_code=DiagnosisErrorCode.RESULT_VALIDATION_FAILED.value,

                additional_info={
                    'validation_details': validation_result.to_dict(),
                    'expected_count': validation_result.expected_count,
                    'actual_count': validation_result.actual_count,
                    'invalid_count': len(validation_result.invalid_results)
                }
            )
            
            return PhaseResult(
                success=False,
                error=f"结果验证严重失败：{validation_result.message}"
            )
        
        if validation_result.status == ValidationStatus.FAILED:
            # 普通失败，记录警告
            api_logger.warning(
                f"[Orchestrator] 结果验证失败：{self.execution_id}, "

                success=False,
                error=f"结果验证失败：{validation_result.message}"

        if validation_result.status == ValidationStatus.WARNING:

                f"[Orchestrator] 结果验证警告：{self.execution_id}, "
                f"警告={validation_result.warnings}"
            )
            # 警告不影响流程继续，但记录到日志
        
        # 验证通过
        api_logger.info(
            f"[Orchestrator] ✅ 阶段 4 完成：结果验证 - {self.execution_id}, "
            f"有效结果数={len(saved_results)}/{validation_result.expected_count}"
        )
        
        return PhaseResult(
            success=True,
            data={
                'saved_results': saved_results,
                'validation_result': validation_result.to_dict()
            }
        )
        
    except Exception as e:
        api_logger.error(
            f"[Orchestrator] ❌ 阶段 4 失败：结果验证 - {self.execution_id}, "
            f"错误={e}"
        )
```

### ISSUE-005: 事务管理与状态更新冲突

**问题描述**:

`DiagnosisOrchestrator` 使用事务管理器（`DiagnosisTransaction`）包裹整个诊断流程，但阶段 1（初始化）和阶段 2（AI 调用）中的状态更新会写入数据库，导致：

1. **事务冲突**: 阶段 1/2 的数据库写入与阶段 3 的事务管理冲突
2. **回滚不一致**: 阶段 3 之后的失败会回滚，但阶段 1/2 的写入不会回滚
3. **数据库记录残留**: 失败诊断的数据库记录未清理

**位置**: 
- `backend_python/wechat_backend/services/diagnosis_orchestrator.py`
- `backend_python/wechat_backend/services/diagnosis_transaction.py`

**现状代码**:

**理想架构要求**:

根据 `2026-03-02-系统逻辑漏洞分析与架构优化计划.md` 第 1.2 节：

> **错误隔离**: 单个阶段失败不影响已完成的阶段

```python
    """

    数据库记录由 Phase 3 使用事务管理器统一创建
    """
    api_logger.info(f"[Orchestrator] 阶段 1: 初始化 - {self.execution_id}")

    try:
        params = self._initial_params

                'competitor_brands': params['brand_list'][1:],
                'selected_models': params['selected_models'],
                'custom_questions': params['custom_questions']
            }

        self._update_phase_status(
            status='initializing',

            progress=0,
            write_to_db=False,  # ✅ 不写数据库
```

```python
# diagnosis_orchestrator.py - _phase_failed 方法增强

async def _phase_failed(self, error_message, error_code):
    """阶段 8: 失败处理（增强版）"""
    
    api_logger.error(f"[Orchestrator] 诊断失败：{self.execution_id}, 错误={error_message}")
    self.current_phase = DiagnosisPhase.FAILED
    
    try:
        # 1. 更新内存状态
        if self.execution_id in self.execution_store:

                'status': 'failed',
                'stage': 'failed',
                'progress': 100,
                'is_completed': True,
                'should_stop_polling': True,
                'error': error_message
            })
        
        # 2. 更新数据库状态（如果报告已创建）
        if hasattr(self, '_current_transaction') and self._current_transaction:
            # 事务已启动，使用事务更新
            self._current_transaction.update_report_status(
                execution_id=self.execution_id,
                status='failed',
                error_message=error_message
            )
        else:
            # 事务未启动，直接更新
            from wechat_backend.diagnosis_report_repository import DiagnosisReportRepository
            report_repo = DiagnosisReportRepository()
            report_repo.update_by_execution_id(self.execution_id, {
                'status': 'failed',
                'error_message': error_message,
                'is_completed': True,
                'should_stop_polling': True
            })
        
        # 3. 发送错误通知
        from wechat_backend.services.realtime_push_service import get_realtime_push_service
        push_service = get_realtime_push_service()
        await push_service.send_error(
            execution_id=self.execution_id,
            error=error_message,
            error_type='diagnosis_failed',
            user_openid=self._initial_params.get('user_openid') if hasattr(self, '_initial_params') else None
        )
        
        api_logger.info(f"[Orchestrator] ✅ 失败处理完成：{self.execution_id}")
        
    except Exception as e:
        api_logger.error(f"[Orchestrator] 失败处理异常：{self.execution_id}, 错误={e}")
```
