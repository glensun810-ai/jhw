#### 修复 2: `diagnosis_api.py`

**修改内容**:

1. **增强错误处理**
   ```python
   try:
       report = service.get_full_report(execution_id)
       
       if not report:
           return jsonify({
               'error': '报告不存在',
               'execution_id': execution_id,
               'suggestion': '请检查执行 ID 是否正确，或重新进行诊断'
           }), 404
       
       if report.get('error'):
           # 返回失败报告，但状态码为 200
           return jsonify(report), 200
       
       # 验证并返回验证信息
       validation = get_validation_service().validate_report(report)
       report['validation'] = validation
       
       return jsonify(report), 200
   except Exception as e:
       api_logger.error(f"获取完整报告失败：{execution_id}, 错误：{e}", exc_info=True)
       return jsonify({
           'error': '获取报告失败',
           'message': str(e),
           'execution_id': execution_id,
           'suggestion': '请稍后重试或联系技术支持'
       }), 500
   ```

#### 修复 5: `results.js`

**修改内容**:

1. **增强 `onLoad()` 函数**
   ```javascript
   onLoad: async function(options) {
       const executionId = options.executionId;
       
       // 增强错误处理
       if (!executionId) {
           this.showNoDataModal('缺少执行 ID，请从诊断页面重新进入');
           return;
       }

       try {
           // 带重试的加载逻辑
           const bestResult = await this.loadWithRetry(executionId, brandName, 3);
           
           if (bestResult) {
               this.processAndDisplayResults(bestResult, brandName);
           } else {
               this.showLoadErrorModal(executionId);
           }
       } catch (error) {
           this.showLoadErrorModal(executionId, error.message);
       }
   }
   ```

2. **新增 `loadWithRetry()` 方法**
   - 并行加载 3 个数据源（API、Storage、Cache）
   - 选择最优结果（优先级：API > Storage > Cache）
   - 指数退避重试（1s, 2s, 3s）
   - 最多重试 3 次

3. **新增 `showLoadErrorModal()` 方法**
   - 友好的错误提示
   - 提供重试和返回首页选项

**代码行数**: +120 行
