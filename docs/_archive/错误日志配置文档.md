# 错误日志配置文档

## 概述

本项目实现了增强的错误日志系统，提供以下功能：

- **错误分类记录**：按 API、数据库、AI、系统等分类记录错误
- **专用日志文件**：每个错误类型有独立的日志文件
- **错误上下文增强**：包含堆栈、环境变量、请求信息等详细上下文
- **错误频率限制**：避免相同错误大量记录导致日志风暴
- **日志轮转和保留**：自动轮转、压缩和清理过期日志
- **错误分析工具**：命令行工具分析错误日志并生成报告

## 快速开始

### 1. 基础使用

```python
from wechat_backend.error_logging import (
    log_api_error,
    log_db_error,
    log_ai_error,
    log_system_error,
)

# 记录 API 错误
try:
    result = api_call()
except Exception as e:
    log_api_error("API 请求失败", request_data=data, exception=e)

# 记录数据库错误
try:
    db.execute(query)
except Exception as e:
    log_db_error("数据库操作失败", query=query, params=params, exception=e)

# 记录 AI 调用错误
try:
    response = call_ai_model(prompt)
except Exception as e:
    log_ai_error("AI 调用失败", model_name="deepseek", prompt=prompt, exception=e)

# 记录系统错误
try:
    process_data()
except Exception as e:
    log_system_error("系统处理失败", module_name="data_processor", exception=e)
```

### 2. 使用 ErrorLogger 实例

```python
from wechat_backend.error_logging import (
    get_error_logger,
    ErrorCategory,
    ErrorSeverity,
)

error_logger = get_error_logger()

# 记录错误
error_logger.log_error(
    category=ErrorCategory.API,
    message="处理请求失败",
    exception=e,
    extra_context={
        'user_id': 'user_123',
        'endpoint': '/api/chat',
        'method': 'POST',
    },
    severity=ErrorSeverity.HIGH,
)
```

### 3. 初始化配置

```python
from wechat_backend.error_logging import init_error_logging

# 应用启动时初始化

```

## 配置选项

### 环境变量配置

在 `.env` 文件中配置：

```bash
# ==================== 增强错误日志配置 ====================
# 是否启用错误日志专用处理器（true/false）
LOG_ENABLE_ERROR_HANDLER=true

# 错误日志文件最大大小（字节，默认 50MB）
LOG_ERROR_MAX_BYTES=52428800

# 错误日志备份数量（默认 10）
LOG_ERROR_BACKUP_COUNT=10

# 日志保留天数（默认 30 天）
LOG_RETENTION_DAYS=30

# 日志总大小上限（MB，默认 1000MB）
LOG_MAX_TOTAL_SIZE_MB=1000

# 多少天后压缩日志（默认 7 天）
LOG_COMPRESS_AFTER_DAYS=7

# 是否启用后台清理任务（true/false）
LOG_ENABLE_RETENTION_CLEANUP=true

# 控制台日志级别
LOG_CONSOLE_LEVEL=INFO

# 文件日志级别
LOG_FILE_LEVEL=DEBUG

# 日志队列大小
LOG_QUEUE_SIZE=10000
```

## 错误分类

系统支持以下错误分类：

| 分类 | 说明 | 日志文件 |
|------|------|----------|
| `api` | API 请求和处理错误 | `errors_api.log` |
| `database` | 数据库操作错误 | `errors_database.log` |
| `ai` | AI 模型调用错误 | `errors_ai.log` |
| `system` | 系统级错误 | `errors_system.log` |
| `security` | 安全相关错误 | `errors_security.log` |
| `network` | 网络连接错误 | `errors_network.log` |
| `validation` | 数据验证错误 | `errors_validation.log` |
| `configuration` | 配置错误 | `errors_configuration.log` |

## 错误严重程度

| 级别 | 说明 | 触发告警 |
|------|------|----------|
| `LOW` | 低 - 可恢复的错误 | 否 |
| `MEDIUM` | 中 - 影响部分功能 | 否 |
| `HIGH` | 高 - 影响核心功能 | 是 |
| `CRITICAL` | 严重 - 系统不可用 | 是 |

## 错误分析工具

### 命令行使用

```bash
# 分析最近 7 天的日志
python -m wechat_backend.error_log_analyzer --log-dir logs

# 分析最近 30 天的日志
python -m wechat_backend.error_log_analyzer --log-dir logs --days 30

# 分析指定日期范围
python -m wechat_backend.error_log_analyzer --log-dir logs --start 2026-03-01 --end 2026-03-05

# 只分析特定分类
python -m wechat_backend.error_log_analyzer --log-dir logs --categories api,database

# 导出 JSON 报告
python -m wechat_backend.error_log_analyzer --log-dir logs --output report.json --format json

# 导出文本报告
python -m wechat_backend.error_log_analyzer --log-dir logs --output report.txt --format text
```

### Python 代码使用

```python
from wechat_backend.error_log_analyzer import ErrorLogAnalyzer

# 创建分析器
analyzer = ErrorLogAnalyzer('logs')

# 加载日志
count = analyzer.load_logs(days=7)
print(f"Loaded {count} errors")

# 生成报告
report = analyzer.analyze()
print(report.summary())

# 导出报告
analyzer.export_report('error_report.json', format='json')
```

### 报告内容

分析报告包含：

- **基本统计**：总错误数、唯一错误数、时间范围
- **错误分布**：按级别、分类、异常类型分布
- **Top 错误**：最常见的错误消息和异常
- **时间分析**：按小时、按天的错误分布
- **趋势分析**：错误趋势（上升/下降/稳定）
- **建议**：基于分析结果的优化建议

## 告警集成

### 注册告警回调

```python
from wechat_backend.error_logging import (
    register_error_alert,
    ErrorSeverity,
)

def send_alert(alert_data):
    """发送告警"""
    # 实现告警逻辑（如发送邮件、钉钉消息等）
    print(f"ALERT: {alert_data['category']} - {alert_data['message']}")

# 注册告警回调
register_error_alert(send_alert)
```

- `CRITICAL` 严重程度的错误

## 最佳实践

### 1. 记录错误的正确方式

```python
# ✅ 推荐：记录完整的上下文
try:
    result = process_user_request(user_id, data)
except Exception as e:
    log_api_error(
        "处理用户请求失败",
        request_data={'user_id': user_id, 'action': 'create'},
        exception=e,
        extra={'ip_address': request.remote_addr}
    )

# ❌ 不推荐：只记录简单消息
try:
    process_user_request(user_id, data)
except Exception as e:
    logger.error("Failed")
```

### 2. 使用合适的严重程度

```python
# 低严重程度：可自动恢复的错误
error_logger.log_error(
    category=ErrorCategory.NETWORK,
    message="网络超时，自动重试",
    severity=ErrorSeverity.LOW,
)

# 高严重程度：需要人工介入
error_logger.log_error(
    category=ErrorCategory.DATABASE,
    message="数据库连接失败",
    severity=ErrorSeverity.HIGH,
)
```

### 3. 避免日志风暴

```python
# 系统自动频率限制
# 相同错误在 60 秒内最多记录 10 次
# 超过后会记录警告并跳过重复错误

# 手动控制
if should_log_debug('my_module'):
    logger.debug("详细调试信息")
```

### 4. 性能优化

```python
# ✅ 推荐：使用条件判断
if logger.isEnabledFor(logging.DEBUG):
    logger.debug(f"Expensive operation: {expensive_calculation()}")

# ❌ 不推荐：无条件执行
logger.debug(f"Expensive operation: {expensive_calculation()}")
```

## 故障排查

### 问题：错误日志没有生成

1. 检查 `LOG_ENABLE_ERROR_HANDLER` 是否为 `true`
2. 检查日志目录是否有写权限
3. 检查日志级别配置是否正确

### 问题：日志文件过大

1. 调整 `LOG_ERROR_MAX_BYTES` 减小单文件大小
2. 调整 `LOG_RETENTION_DAYS` 缩短保留时间
3. 启用 `LOG_ENABLE_RETENTION_CLEANUP` 自动清理

### 问题：错误频率过高

1. 查看错误分析报告识别主要错误源
2. 修复高频错误
3. 调整 `LOG_ENABLE_ERROR_HANDLER` 暂时关闭
