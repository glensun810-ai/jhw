二、数据一致性问题
2.1 内存 vs 数据库状态同步
问题: 状态管理器同时维护内存 (execution_store) 和数据库状态
当前机制 (state_manager.py 第 253-297 行):
def update_state(self, execution_id, ..., write_to_db=True):
    # 步骤 1: 更新 execution_store
    store = self.execution_store[execution_id]
    if status is not None: store['status'] = status
    ...

    # 步骤 2: 更新数据库 (可选)
    if write_to_db:
        self._force_sync_to_database(...)
潜在风险:
1.
原子性风险: 内存更新成功后，数据库更新可能失败
2.
可见性延迟: SQLite WAL 模式下，COMMIT 后数据可能短暂不可见
3.
状态推导问题: 当 stage='init' 但 progress>0 时，状态不一致
已实施的缓解措施:
•
阶段 1 不写数据库，避免重复创建记录
•
验证器添加重试机制处理可见性延迟
•
状态 API 添加推导逻辑 (if stage == 'init' and progress > 0: stage = 'ai_fetching')

    if exc_type is not None:
        self.error = str(exc_val)
        if self.auto_rollback:
            self.rollback()  # 自动回滚
        self.status = "failed"
        return False
    else:
        self.status = "committed"
        return False
