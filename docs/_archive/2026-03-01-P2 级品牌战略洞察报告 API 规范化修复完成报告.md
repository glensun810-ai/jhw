### 2.1 OpenAPI 规范文档 (`api-spec.yaml`)

**核心内容**:

1. **API 端点定义** (4 个)
   - `GET /api/diagnosis/report/{execution_id}` - 获取完整报告
   - `GET /api/diagnosis/report/{execution_id}/validate` - 验证报告
   - `GET /api/diagnosis/history` - 获取历史记录
   - `GET /test/status/{execution_id}` - 获取任务状态

2. **Schema 定义** (20+)
   - FullReportResponse - 完整报告响应
   - Report - 报告主数据
   - Result - 诊断结果
   - Analysis - 分析数据
   - Validation - 验证信息
   - Error 响应系列

3. **认证和错误处理**
   - Bearer Token 认证
   - 标准错误响应格式
   - 频率限制响应

**示例 Schema**:

```yaml
FullReportResponse:
  type: object
  required:
    - report
    - results
    - analysis
    - meta
  properties:
    report:
      $ref: '#/components/schemas/Report'
    results:
      type: array
      items:
        $ref: '#/components/schemas/Result'
    validation:
      $ref: '#/components/schemas/Validation'

      $ref: '#/components/schemas/QualityHints'
```

## 六、最佳实践

### 6.1 API 设计规范

1. **命名规范**
   - 使用小写字母和连字符：`/api/diagnosis/report`
   - 资源名用复数：`/reports`
   - 操作名用驼峰：`getFullReport`

2. **版本控制**
   - URL 中包含版本号：`/api/v1/...`
   - 或使用 Header：`Accept: application/vnd.api.v1+json`

3. **错误处理**
   - 统一错误格式
   - 提供错误建议
   - 记录详细日志

### 6.2 类型定义规范

1. **必填 vs 可选**
   - 必填字段不使用 `?`
   - 可选字段使用 `?` 或 `| null`

2. **枚举类型**
   - 使用字面量联合类型
   - 提供完整的枚举值

3. **文档注释**
   - 每个接口添加 JSDoc
   - 说明字段含义和用法

### 6.3 契约测试规范

1. **测试覆盖**
   - 必填字段测试
   - 字段类型测试
   - 枚举值测试
   - 边界值测试

2. **测试数据**
   - 使用 fixtures 提供测试数据
   - 覆盖正常和异常场景

3. **测试命名**
   - 清晰表达测试目的
   - 使用 `should` 或 `test_` 前缀
