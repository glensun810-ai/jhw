## 执行摘要

### 问题描述

系统在品牌诊断流程中，前端反复无法输出品牌详细诊断报告。经过数十次至数百次诊断尝试均失败。

### 核心症状

1. **前端轮询持续失败**: 日志显示前端在 0.5 秒内发起 6 次轮询请求，均返回 `status=failed`
2. **结果无法展示**: 诊断报告无法在前端页面正常展示
3. **状态不一致**: 内存状态与数据库状态可能不同步
4. **流程中断**: 诊断流程在某个环节中断，无法完成端到端执行

### 文档目标

本文档旨在通过**分步骤、模块化、可验证**的方式，对系统进行全面摸底，识别并定位导致前端无法输出诊断报告的根本原因。

```
│  diagnosis_reports / diagnosis_results / diagnosis_analysis     │
```

### 1.2 诊断流程（7 阶段）

| 阶段 | 名称 | 职责 | 事务 | 预期耗时 |
|------|------|------|------|----------|
| 1 | 初始化 (init) | 设置初始状态，创建数据库记录 | 无 | <1s |
| 2 | AI 调用 (ai_fetching) | 并行调用多个 AI 平台 | 无 | 20-40s |
| 3 | 结果保存 (results_saving) | 将 AI 结果保存到数据库 | 事务 A | 2-5s |
| 4 | 结果验证 (results_validating) | 验证结果数量和质量 | 无 | <1s |
| 5 | 后台分析 (background_analysis) | 异步执行品牌/竞争分析 | 异步 | 5-15s |
| 6 | 报告聚合 (report_aggregating) | 聚合所有结果为最终报告 | 事务 B | 2-5s |
| 7 | 完成 (complete) | 更新状态并发送通知 | 事务 C | <1s |

## 四、常见失败模式与根因分析

### 模式 1: 前端持续轮询失败

**症状**:
```
前端日志：
[PollingManager] 轮询响应：{status: 'failed', error_message: 'xxx'}
[PollingManager] 轮询响应：{status: 'failed', error_message: 'xxx'}
...（6 次/0.5 秒）
```

**可能原因**:
1. 后端诊断流程失败，但 should_stop_polling 未设置
2. 前端未检测到失败状态，继续轮询
3. 状态同步失败（内存 vs 数据库）

**排查步骤**:
1. 检查后端日志，找到失败根因
2. 验证数据库中 should_stop_polling 是否为 1
3. 检查 state_manager 同步逻辑

## 五、快速诊断脚本

### 5.1 后端健康检查

```python
#!/usr/bin/env python3
"""
后端健康检查脚本
检查所有必需组件是否正常
"""

import sys
import sqlite3
from pathlib import Path

# 添加项目路径
sys.path.insert(0, str(Path(__file__).parent / 'backend_python'))

def check_database():
    """检查数据库"""
    from wechat_backend.database import get_db_connection
    try:
        conn = get_db_connection()
        cursor = conn.cursor()
        cursor.execute("SELECT name FROM sqlite_master WHERE type='table'")
        tables = [row[0] for row in cursor.fetchall()]
        print(f"✅ 数据库连接正常，表数量：{len(tables)}")
        
        # 检查必需表
        required_tables = ['diagnosis_reports', 'diagnosis_results', 'diagnosis_analysis']
        missing = [t for t in required_tables if t not in tables]
        if missing:
            print(f"❌ 缺少必需表：{missing}")
            return False
        
        # 检查 schema
        cursor.execute("PRAGMA table_info(diagnosis_results)")
        columns = [row[1] for row in cursor.fetchall()]
        if 'sentiment' not in columns:
            print("❌ diagnosis_results 缺少 sentiment 列")
            return False
        
        print("✅ 数据库 schema 验证通过")
        return True
    except Exception as e:
        print(f"❌ 数据库检查失败：{e}")
        return False
    finally:
        conn.close()
```
