## 一、问题根因

### 1.1 泄漏源头定位

**问题文件**: `wechat_backend/database/transaction.py`

**问题代码** (第 35-68 行):
```python
@contextmanager
def database_transaction(description: str = "数据库操作"):
    conn = None
    try:
        # ❌ 问题：直接使用 sqlite3.connect() 而不是从连接池获取
        conn = sqlite3.connect(DB_PATH)
        conn.execute('PRAGMA journal_mode=WAL')
        
        yield conn
        
        conn.commit()
    except Exception as e:
        if conn:
            conn.rollback()
        raise
    finally:
        # ⚠️ 虽然 finally 中关闭了连接，但绕过了连接池管理
        if conn:
            conn.close()
```

**问题代码** (第 119-132 行):
```python
def get_connection(readonly: bool = False):
    """❌ 问题：直接创建连接，不使用连接池"""
    if readonly:
        return sqlite3.connect(f'file:{DB_PATH}?mode=ro', uri=True)
    else:
        conn = sqlite3.connect(DB_PATH)
        conn.execute('PRAGMA journal_mode=WAL')
        return conn
```

### 1.2 为什么会导致泄漏

**连接池监控机制失效**:
```
连接池泄漏检测 ──→ 监控 _connection_use_time 和 _in_use
                      ↓
直接 sqlite3.connect() ──→ 绕过连接池 ──→ 无法被监控
                              ↓
                        连接使用完后关闭 ──→ 但高并发下可能未及时关闭
                              ↓
                        连接池检测到"泄漏"（实际是未使用连接池的连接）
```

**日志中的泄漏连接**:
```
[连接泄漏] 连接超时未归还：id=6313272528, 年龄=36.8 秒，
thread_id=8387939328, 池中剩余=0, 使用中=1
```

**分析**:
- `id=6313272528` - 这是连接对象的内存地址
- `thread_id=8387939328` - 占用该连接的线程 ID
- `池中剩余=0, 使用中=1` - 连接池中实际没有这个连接

**结论**: 泄漏的连接**不是来自连接池**，而是直接使用 `sqlite3.connect()` 创建的！

## 三、修复方案

### 3.1 方案 A: 统一使用连接池（推荐）✅

**修改 `database/transaction.py`**:

```python
#!/usr/bin/env python3
"""
DS-P1-2 修复：数据库事务处理上下文管理器

【修复 - 2026-03-12】
- 使用连接池获取连接，而不是直接 sqlite3.connect()
- 确保连接被连接池监控和管理
"""

import sqlite3
from contextlib import contextmanager
from pathlib import Path
from typing import Optional
from datetime import datetime

from wechat_backend.logging_config import db_logger
# 【修复】导入连接池
from wechat_backend.database_connection_pool import get_db_pool

# 数据库路径
DB_PATH = Path(__file__).parent.parent / 'database.db'


@contextmanager
def database_transaction(description: str = "数据库操作"):
    """
    数据库事务上下文管理器

    【修复 - 2026-03-12】
    - 从连接池获取连接，而不是直接创建
    - 确保连接被连接池监控和管理
    """
    conn = None
    try:
        # 【修复】从连接池获取连接
        pool = get_db_pool()
        conn = pool.get_connection(timeout=15.0)
        
        db_logger.info(f"[Transaction] 开始事务：{description}")

        # 产出连接供外部使用
        yield conn

        # 提交事务
        conn.commit()
        db_logger.info(f"[Transaction] 事务成功：{description}")

    except Exception as e:
        # 发生异常时回滚
        if conn:
            conn.rollback()
            db_logger.error(f"[Transaction] 事务回滚：{description}, 错误：{e}")
        raise

    finally:
        # 【修复】归还连接到连接池，而不是关闭
        if conn:
            pool = get_db_pool()
            pool.return_connection(conn)
            db_logger.debug(f"[Transaction] 连接归还：{description}")


@contextmanager
def database_readonly_transaction(description: str = "数据库查询"):
    """
    数据库只读事务上下文管理器

    【修复 - 2026-03-12】
    - 从连接池获取连接
    """
    conn = None
    try:
        # 【修复】从连接池获取只读连接
        pool = get_db_pool()
        conn = pool.get_connection(timeout=15.0)
        
        # 设置为只读模式
        conn.execute('PRAGMA query_only=ON')
        
        db_logger.info(f"[Transaction] 开始只读事务：{description}")

        yield conn

        db_logger.debug(f"[Transaction] 只读事务完成：{description}")

    except Exception as e:
        db_logger.error(f"[Transaction] 只读事务失败：{description}, 错误：{e}")
        raise

    finally:
        # 【修复】归还连接到连接池
        if conn:
            pool = get_db_pool()
            pool.return_connection(conn)


def get_connection(readonly: bool = False):
    """
    获取数据库连接（从连接池）

    【修复 - 2026-03-12】
    - 从连接池获取连接，而不是直接创建
    """
    pool = get_db_pool()
    return pool.get_connection(timeout=15.0)
```

## 六、总结

### 6.1 根因

**连接泄漏的根本原因**:
- `database_transaction()` 直接使用 `sqlite3.connect()` 创建连接
- 绕过了连接池的监控和管理
- 高并发下连接未及时关闭导致泄漏

### 6.2 修复方案

**统一使用连接池**:
```python
# ❌ 修复前
conn = sqlite3.connect(DB_PATH)
# ... 使用连接
conn.close()

# ✅ 修复后
pool = get_db_pool()
conn = pool.get_connection()
# ... 使用连接
pool.return_connection(conn)
```

- ✅ 连接池监控生效
- ✅ 连接泄漏检测准确
- ✅ 高并发下连接复用
