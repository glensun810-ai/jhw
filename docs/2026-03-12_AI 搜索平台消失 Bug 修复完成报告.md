## 二、根因分析

### 2.1 问题根因

**核心问题**: 诊断过程中 AI 平台数据 (`domesticAiModels`, `overseasAiModels`) 被意外清空或覆盖

**触发场景**:
1. 用户点击"诊断中"按钮
2. 页面进入 `appState: 'testing'` 状态
3. 某个地方修改了 AI 平台数据
4. 平台卡片消失

### 2.2 代码分析

**问题代码流程**:
```javascript
startBrandTest()
  ↓
setData({ isTesting: true, appState: 'testing', ... })
  ↓
callBackendBrandTest()
  ↓
诊断过程中...
  ↓
AI 平台数据被修改/清空 ❌
  ↓
页面渲染时 domesticAiModels 为空
  ↓
平台卡片消失
```

## 三、修复方案

### 3.1 修复策略

**数据快照保护机制**:
1. 诊断开始时保存 AI 平台数据快照
2. 诊断完成/错误时恢复数据
3. 确保数据不丢失

### 3.2 修复内容

**修复文件**: `pages/index/index.js`

**修复位置 1**: `startBrandTest()` 函数

**修复代码**:
```javascript
startBrandTest: function() {
```

## 五、技术亮点

### 5.1 数据快照机制

**优点**:
- ✅ 简单高效
- ✅ 不影响现有逻辑
- ✅ 自动恢复
- ✅ 无性能开销

**实现**:
```javascript
// 保存快照（深拷贝）
this._aiModelsSnapshot = {
  domesticAiModels: JSON.parse(JSON.stringify(data)),
  overseasAiModels: JSON.parse(JSON.stringify(data)),
  selectedMarketTab: this.data.selectedMarketTab
};

// 恢复快照
this.setData(this._aiModelsSnapshot);
this._aiModelsSnapshot = null;
```

### 5.2 全场景覆盖

2. ✅ 诊断错误 → 恢复数据
