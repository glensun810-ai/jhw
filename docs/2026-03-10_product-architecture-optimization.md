**文档版本**: v1.0  
**创建日期**: 2026-03-10  
**文档类型**: 产品架构设计  
**优先级**: P0 - 战略级优化  

---

## 一、执行摘要

### 1.1 核心结论

经过对当前系统架构、用户行为路径和功能模块的全面分析，**推荐采用方案 B+**（增强版）作为最优产品架构方案：

| 页签序号 | 页签名称 | 核心功能 | 用户价值 |
|---------|---------|---------|---------|
| **Tab 1** | AI 搜索 | 品牌诊断入口、AI 模型选择、实时诊断 | 核心功能，最高频使用 |
| **Tab 2** | 诊断记录 | 历史记录列表、记录详情（含报告）、快速操作 | 任务管理中枢 |
| **Tab 3** | 统计分析 | 多维度数据分析、品牌对比、趋势洞察 | 决策支持 |
| **Tab 4** | 我的 | 个人中心、设置管理、收藏报告 | 用户管理 |

### 1.2 关键设计原则

1. **用户旅程优先**: 按"诊断→查看→分析→管理"的自然流程组织
2. **功能聚合**: 将诊断报告和诊断记录合并，减少认知负担
3. **价值递进**: 从工具使用→结果查看→深度分析→个人管理
4. **可扩展性**: 为未来功能预留扩展空间

### 2.4 现有架构问题诊断

| 问题 | 描述 | 影响 | 严重度 |
|-----|------|------|--------|
| **功能分散** | 诊断记录和诊断报告分离，用户需多次跳转 | 增加认知负担，降低效率 | 🔴 高 |

## 三、方案对比

### 3.1 方案 A：现状优化版

```
┌─────────────────────────────────────────────────────────┐
│  [AI 搜索]  │  [诊断记录]  │  [诊断报告]  │  [我的]    │
│  index      │  history     │  report      │  profile   │
└─────────────────────────────────────────────────────────┘
```

**设计逻辑**: 将"历史记录"改为"诊断记录"，"收藏报告"改为"诊断报告"

**优点**:
- ✅ 保持 4 个 Tab 的稳定结构
- ✅ 诊断报告独立成页，便于深度查看

**缺点**:
- ❌ 诊断记录和诊断报告高度关联，分离增加跳转
- ❌ 诊断报告本质是诊断记录的详情，独立成 Tab 逻辑不通
- ❌ 缺少统计分析能力

**评分**: ⭐⭐⭐ (3/5)

---

### 3.2 方案 B：推荐方案

```
┌─────────────────────────────────────────────────────────┐
│  [AI 搜索]  │  [诊断记录]  │  [统计分析]  │  [我的]    │
│  index      │  history     │  analytics   │  profile   │
└─────────────────────────────────────────────────────────┘
```

**设计逻辑**: 按"工具→记录→分析→管理"的用户心智模型组织

**优点**:
- ✅ 符合用户自然使用流程
- ✅ 诊断记录包含详情（报告），减少跳转
- ✅ 统计分析提供决策价值，提升产品深度
- ✅ 收藏功能整合到"我的"，逻辑更清晰

**缺点**:
- ⚠️ 需要新增统计分析页面（开发成本）
- ⚠️ 收藏报告入口变深（但使用频率低，影响小）

**评分**: ⭐⭐⭐⭐⭐ (5/5)

---

### 3.3 方案 C：五 Tab 扩展版

```
┌─────────────────────────────────────────────────────────────┐
│ [AI 搜索] │ [诊断记录] │ [统计分析] │ [收藏] │ [我的] │
│  index    │  history   │  analytics │ saved  │ profile │
└─────────────────────────────────────────────────────────────┘
```

**设计逻辑**: 增加 Tab 数量，每个核心功能独立入口

**优点**:
- ✅ 功能入口最浅，操作最便捷
- ✅ 收藏功能独立，便于管理

**缺点**:
- ❌ 微信小程序 TabBar 最多支持 5 个，已接近上限
- ❌ 功能过于分散，增加选择负担
- ❌ 诊断记录和收藏报告仍有功能重叠

**评分**: ⭐⭐⭐⭐ (4/5)

### 4.2 Tab 1: AI 搜索（核心功能）

**定位**: 品牌诊断的核心入口

**核心功能**:
- 品牌名称输入
- 竞品品牌添加
- AI 模型选择（国内/海外）
- 自定义问题设置
- 诊断任务启动
- 实时进度展示
- 诊断结果预览

**页面流程**:
```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  输入配置    │ →  │  诊断中      │ →  │  结果预览    │
│  - 品牌      │    │  - 进度条    │    │  - 核心评分  │
│  - 竞品      │    │  - 阶段提示  │    │  - 关键发现  │
│  - AI 模型   │    │  - 预计时间  │    │  - 查看报告  │
│  - 问题      │    │              │    │    按钮      │
└──────────────┘    └──────────────┘    └──────────────┘
```

**用户体验优化点**:
1. **草稿自动保存**: 用户输入实时保存，避免意外丢失
2. **配置快捷加载**: 支持历史配置一键复用
3. **智能推荐**: 根据品牌自动推荐竞品和 AI 模型
4. **进度可视化**: 分阶段展示诊断进度，降低等待焦虑

### 7.1 Tab 切换体验优化

**问题**: 当前 Tab 切换时，页面状态可能丢失

**优化方案**:

```javascript
// app.js
App({
  globalData: {
    // 缓存各 Tab 页面状态

  // 页面切换时保存状态
  savePageState: function(pageKey, state) {

  },

  // 页面切换时恢复状态
  getPageState: function(pageKey) {
    return this.globalData.pageState[pageKey];
  }
});
```

### 7.3 统计分析 数据加载策略

```javascript
// pages/analytics/analytics.js
Page({
  data: {
    loading: false,
    dataLoaded: false,
    analysisData: null
  },

  onLoad: function(options) {
    // 首次加载
    this.loadAnalysisData();
  },

  onShow: function() {
    // 每次显示时检查是否需要刷新
    if (!this.data.dataLoaded || this.needRefresh()) {
      this.loadAnalysisData();
    }
  },

  // 智能刷新策略
  needRefresh: function() {
    const lastLoadTime = this.data.lastLoadTime;
    const now = Date.now();
    // 超过 5 分钟则刷新
    return now - lastLoadTime > 5 * 60 * 1000;
  },

  loadAnalysisData: function() {
    this.setData({ loading: true });
    
    // 并行加载多个分析维度
    Promise.all([
      this.loadBrandCompare(),
      this.loadTrendAnalysis(),
      this.loadPlatformCompare(),
      this.loadQuestionAnalysis()
    ]).then(([brandData, trendData, platformData, questionData]) => {
      this.setData({
        analysisData: {
          brandCompare: brandData,
          trend: trendData,
          platformCompare: platformData,
          questionAnalysis: questionData
        },
        dataLoaded: true,
        loading: false,
        lastLoadTime: Date.now()
      });
    });
  }
});
```

## 八、数据埋点设计

### 8.1 核心埋点事件

| 事件名称 | 触发时机 | 关键参数 |
|---------|---------|---------|
| `tab_click` | 点击 Tab | tab_name, from_page |
| `diagnosis_start` | 启动诊断 | brand_name, model_count, question_count |
| `diagnosis_complete` | 诊断完成 | execution_id, duration, score |
| `report_view` | 查看报告 | execution_id, view_duration |
| `report_favorite` | 收藏报告 | execution_id, is_favorite |
| `report_export` | 导出报告 | execution_id, export_type |
| `analysis_view` | 查看统计分析 | analysis_type, view_duration |

### 8.2 埋点实现示例

```javascript
// utils/analytics.js
const analytics = {
  // Tab 点击
  trackTabClick: function(tabName, fromPage) {
    wx.reportAnalytics('tab_click', {
      tab_name: tabName,
      from_page: fromPage
    });
  },

  // 诊断启动
  trackDiagnosisStart: function(brandName, modelCount, questionCount) {
    wx.reportAnalytics('diagnosis_start', {
      brand_name: brandName,
      model_count: modelCount,
      question_count: questionCount
    });
  },

  // 诊断完成
  trackDiagnosisComplete: function(executionId, duration, score) {
    wx.reportAnalytics('diagnosis_complete', {
      execution_id: executionId,
      duration: duration,
      score: score
    });
  },

  // 报告查看
  trackReportView: function(executionId, viewDuration) {
    wx.reportAnalytics('report_view', {
      execution_id: executionId,
      view_duration: viewDuration
    });
  }
};

module.exports = analytics;
```
