## 一、问题背景

### 1.1 当前问题

**核心问题**: 分层加载时，层级之间存在数据依赖，导致：
- 第 1 层加载等待第 2 层数据
- 第 2 层加载等待第 3 层数据
- 最终导致整体卡死

**错误示例**:
```javascript
// ❌ 错误：层级依赖导致阻塞
processHistoryData: function(record) {
  // 第 1 层：需要第 2 层的数据
  const dimensionScores = this.calculateDimensionScores(detailedResults);
  this.setData({ dimensionScores }); // 等待计算完成
  
  // 第 2 层：需要第 3 层的数据
  const simplifiedResults = this.simplifyResults(detailedResults);
  this.setData({ simplifiedResults }); // 等待处理完成
  
  // 结果：所有数据都处理完才显示，失去分层意义
}
```

### 1.2 设计目标

**目标**: 实现真正的分层加载，各层独立，互不依赖

**原则**:
1. **数据独立** - 每层数据独立获取，不依赖其他层
2. **异步处理** - 使用异步处理，不阻塞主线程
3. **容错设计** - 某层失败不影响其他层
4. **渐进增强** - 先显示核心，再逐步增强

## 三、分层加载设计

### 3.1 分层原则

**核心原则**: **每层数据独立，不依赖其他层**

```javascript
// ✅ 正确：每层独立
Layer 1: 直接从 record 获取基础字段
Layer 2: 从 record 获取分析数据
Layer 3: 从 record 获取详细结果（异步处理）

// ❌ 错误：层间依赖
Layer 1: 等待 Layer 2 计算完成
Layer 2: 等待 Layer 3 处理完成
```

### 3.2 数据分层

#### 第 1 层：核心信息（0ms，立即显示）

**数据来源**: 直接从 record 对象获取，无需计算

**字段**:
```javascript
{
  brandName: record.brandName,           // 直接获取
  overallScore: record.overallScore,     // 直接获取
  overallGrade: this.calculateGrade(record.overallScore), // 简单计算
  overallSummary: this.getGradeSummary(grade), // 简单映射
  gradeClass: this.getGradeClass(grade)  // 简单映射
}
```

**特点**:
- ✅ 数据量小（<1KB）
- ✅ 计算简单（<10ms）
- ✅ 不依赖其他层
- ✅ 立即显示

**代码**:
```javascript
// 第 1 层：核心信息（同步，立即执行）
this.setData({
  brandName: record.brandName || '未知品牌',
  overallScore: record.overallScore || 0,
  overallGrade: this.calculateGrade(record.overallScore),
  overallSummary: this.getGradeSummary(grade),
  gradeClass: this.getGradeClass(grade),
  loading: false  // 关键：先解除加载状态
});
```

#### 第 2 层：分析数据（100ms，独立计算）

**数据来源**: 从 record 对象获取，独立计算

**字段**:
```javascript
{
  // 评分维度
  overallAuthority: record.dimensionScores?.authority || 0,
  overallVisibility: record.dimensionScores?.visibility || 0,
  overallPurity: record.dimensionScores?.purity || 0,
  overallConsistency: record.dimensionScores?.consistency || 0,
  
  // 核心指标
  sovShare: record.coreMetrics?.sovShare || 0,
  sentimentScore: record.coreMetrics?.sentimentScore || 0,
  physicalRank: record.coreMetrics?.physicalRank || 0,
  influenceScore: record.coreMetrics?.influenceScore || 0,
  hasMetrics: true,
  
  // 问题诊断
  highRisks: record.problemDiagnosis?.highRisks || [],
  mediumRisks: record.problemDiagnosis?.mediumRisks || [],
  suggestions: record.problemDiagnosis?.suggestions || [],
  
  // 信源分析
  sourceList: record.sourceAnalysis?.sourceList?.slice(0, 10) || []
}
```

**特点**:
- ✅ 数据量中等（~5KB）
- ✅ 独立计算，不依赖第 3 层
- ✅ 后端已预计算，直接获取

**代码**:
```javascript
// 第 2 层：分析数据（延迟 100ms，独立获取）
setTimeout(() => {
  this.setData({
    // 评分维度
    overallAuthority: record.dimensionScores?.authority || 0,
    overallVisibility: record.dimensionScores?.visibility || 0,
    overallPurity: record.dimensionScores?.purity || 0,
    overallConsistency: record.dimensionScores?.consistency || 0,
    
    // 核心指标
    sovShare: record.coreMetrics?.sovShare || 0,
    sentimentScore: record.coreMetrics?.sentimentScore || 0,
    physicalRank: record.coreMetrics?.physicalRank || 0,
    influenceScore: record.coreMetrics?.influenceScore || 0,
    hasMetrics: true,
    
    // 问题诊断
    highRisks: record.problemDiagnosis?.highRisks || [],
    mediumRisks: record.problemDiagnosis?.mediumRisks || [],
    suggestions: record.problemDiagnosis?.suggestions || [],
    
    // 信源分析
    sourceList: record.sourceAnalysis?.sourceList?.slice(0, 10) || []
  });
}, 100);
```

**关键点**:
```javascript
// ✅ 正确：直接使用后端返回的预计算数据
overallAuthority: record.dimensionScores?.authority || 0

// ❌ 错误：不要从 detailedResults 重新计算
overallAuthority: this.calculateFromDetailedResults(detailedResults)
```

#### 第 3 层：详细结果（200ms，异步处理）

**数据来源**: detailedResults 数组，需要简化处理

**字段**:
```javascript
{
  detailedResults: [...],  // 简化后的前 5 条
  aiModels: [...],
  competitors: [...],
  questions: [...],
  hasMoreResults: true,
  totalResults: 100
}
```

**特点**:
- ⚠️ 数据量大（~50KB+）
- ⚠️ 需要简化处理
- ⚠️ 使用虚拟列表

**代码**:
```javascript
// 第 3 层：详细结果（延迟 200ms，异步简化）
setTimeout(() => {
  // 异步处理，不阻塞主线程
  const simplifiedResults = detailedResults.slice(0, 5).map((r, index) => ({
    id: r.id || `result_${index}`,
    brand: r.brand || r.brandName,
    model: r.aiModel || r.ai_model || '未知模型',
    score: r.score,
    question: r.question?.substring(0, 50) || '',
    response: r.response?.substring(0, 100) || '',
    truncated: (r.response?.length || 0) > 100,
    fullResponse: r.response || '',
    expanded: false
  }));
  
  // 保存完整数据到全局变量（用于加载更多）
  const app = getApp();
  app.globalData.currentReportData = {
    detailedResults: detailedResults,
    executionId: record.executionId
  };
  
  this.setData({
    detailedResults: simplifiedResults,
    aiModels: record.ai_models || [],
    competitors: record.competitors || [],
    questions: record.questions || [],
    hasMoreResults: detailedResults.length > 5,
    totalResults: detailedResults.length
  });
}, 200);
```

## 六、容错设计

### 6.1 层级容错

**原则**: 某层失败不影响其他层

```javascript
// 第 1 层：即使失败也要显示基本框架
try {
  this.setData({
    brandName: record.brandName || '未知品牌',
    overallScore: record.overallScore || 0,
    loading: false
  });
} catch (error) {
  console.error('第 1 层加载失败:', error);
  // 显示默认值
  this.setData({
    brandName: '未知品牌',
    overallScore: 0,
    loading: false
  });
}

// 第 2 层：失败不影响第 1 层和第 3 层
setTimeout(() => {
  try {
    this.setData({
      overallAuthority: record.dimensionScores?.authority || 0,
      // ...
    });
  } catch (error) {
    console.error('第 2 层加载失败:', error);
    // 不显示分析数据，但不影响其他层
  }
}, 100);

// 第 3 层：失败不影响第 1 层和第 2 层
setTimeout(() => {
  try {
    // ...
  } catch (error) {
    console.error('第 3 层加载失败:', error);
    // 不显示详细结果，但不影响其他层
  }
}, 200);
```

### 6.2 数据降级

**降级策略**:
```javascript
// 优先使用后端预计算数据
dimensionScores: record.dimensionScores || 
                 // 降级：从详细结果计算
                 this.calculateFromDetailedResults() ||
                 // 降级：默认值
                 { authority: 0, visibility: 0, purity: 0, consistency: 0 }
```

## 八、总结

### 8.1 核心原则

**数据独立**: 每层数据独立获取，不依赖其他层

**异步处理**: 使用 setTimeout 异步处理，不阻塞主线程

**容错设计**: 某层失败不影响其他层

**渐进增强**: 先显示核心，再逐步增强

### 8.2 关键要点

**✅ 正确做法**:
```javascript
// 直接使用后端预计算数据
overallAuthority: record.dimensionScores?.authority || 0

// 异步处理，不阻塞
setTimeout(() => {
  this.setData({ ... });
}, 100);

// 容错处理
data: record.field || defaultValue
```

**❌ 错误做法**:
```javascript
// 从详细结果重新计算
overallAuthority: this.calculateFromDetailedResults(results)

// 同步阻塞
const scores = this.calculateScores(results);
this.setData({ scores });

// 无容错
data: record.field  // 可能为 undefined
```
