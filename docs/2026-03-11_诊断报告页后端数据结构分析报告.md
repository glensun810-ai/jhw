## 二、问题分析

### 2.1 数据缺失

**缺失字段**:
- ❌ `dimension_scores` - 评分维度（权威度、可见度、纯净度、一致性）
- ❌ `recommendation_data` - 推荐数据（风险等级、优化建议）
- ❌ `quality_score` - 质量评分

**影响**:
- ⚠️ 第 2 层无法使用后端预计算数据
- ⚠️ 需要从 detailed_results 重新计算
- ⚠️ 导致层级依赖，回到原问题

### 2.2 当前代码处理

**当前实现**:
```javascript
// pages/history-detail/history-detail.js:213
const dimensionScores = results.dimension_scores || results.dimensionScores || {};

// 如果后端未返回，值为 {}
// 导致显示全 0
```

**实际情况**:
```javascript
// 后端未返回 dimension_scores
results.dimension_scores === undefined

// 前端容错处理
overallAuthority: dimensionScores.authority || 0  // 显示 0
overallVisibility: dimensionScores.visibility || 0  // 显示 0
overallPurity: dimensionScores.purity || 0  // 显示 0
overallConsistency: dimensionScores.consistency || 0  // 显示 0
```

**结果**: 用户看到全 0 的评分维度！

### 方案 A: 后端添加预计算数据（推荐⭐⭐⭐⭐⭐）

**实施方**: 后端团队

**新增字段**:

```javascript
      ],

    },

  },
  
  // === 新增：质量评分 ===

    overall_score: 85,

  }

```

**优势**:
- ✅ 前端直接使用，无需计算
- ✅ 消除层级依赖
- ✅ 性能最优

**劣势**:
- ⚠️ 需要后端开发
- ⚠️ 增加后端计算负担

**实施时间**: 1-2 天

### 方案 B: 前端计算（临时方案⭐⭐⭐）

**实施方**: 前端团队

**实施代码**:
```javascript
// 从 detailed_results 计算评分维度
calculateDimensionScores: function(detailedResults) {
  if (!detailedResults || detailedResults.length === 0) {
    return { authority: 0, visibility: 0, purity: 0, consistency: 0 };
  }
  
  // 限制计算数据量，最多 20 条
  const limited = detailedResults.slice(0, 20);
  const total = limited.length;
  
  let authSum = 0, visSum = 0, purSum = 0, conSum = 0;
  
  for (let i = 0; i < total; i++) {
    const r = limited[i];
    authSum += (r.authority_score || r.authority || 0);
    visSum += (r.visibility_score || r.visibility || 0);
    purSum += (r.purity_score || r.purity || 0);
    conSum += (r.consistency_score || r.consistency || 0);
  }
  
  return {
    authority: Math.round(authSum / total),
    visibility: Math.round(visSum / total),
    purity: Math.round(purSum / total),
    consistency: Math.round(conSum / total)
  };
}
```

**优势**:
- ✅ 无需后端支持
- ✅ 可立即实施

**劣势**:
- ❌ 需要计算 detailed_results
- ❌ 导致层级依赖
- ❌ 性能较差

**实施时间**: 30 分钟

### 方案 C: 混合方案（折中⭐⭐⭐⭐）

**核心思路**: 
- 第 2 层部分字段使用后端数据
- 部分字段前端计算
- 第 3 层不受影响

**实施代码**:
```javascript
// 第 2 层：部分后端数据，部分前端计算
setTimeout(() => {
  // 后端返回的数据（直接使用）
  const exposureAnalysis = results.exposure_analysis || {};
  const sourceIntelligence = results.source_intelligence || {};
  
  // 前端计算的数据（从 detailed_results）
  const dimensionScores = this.calculateDimensionScores(detailedResults);
  
  this.setData({
    // 后端数据
    sovShare: exposureAnalysis.brand_details?.[mainBrand]?.sov_share || 0,
    sentimentScore: exposureAnalysis.brand_details?.[mainBrand]?.sentiment_score || 0,
    sourceList: sourceIntelligence.source_pool?.slice(0, 10) || [],
    
    // 前端计算
    overallAuthority: dimensionScores.authority,
    overallVisibility: dimensionScores.visibility,
    overallPurity: dimensionScores.purity,
    overallConsistency: dimensionScores.consistency
  });
}, 100);
```

**优势**:
- ✅ 部分使用后端数据
- ✅ 可立即实施
- ✅ 性能可接受

**劣势**:
- ⚠️ 部分层级依赖
- ⚠️ 性能不如方案 A

**实施时间**: 1 小时
