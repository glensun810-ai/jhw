## 二、根因分析

### 2.1 核心问题 1: AI 调用失败 🔴

**错误信息**: `AI 调用失败：AI 调用返回空结果`

**可能原因**:
1. **豆包 API Key 配置问题** - API Key 格式不正确或已过期
2. **豆包 API 服务不可用** - 网络连接问题或服务宕机
3. **请求参数错误** - prompt 格式不符合 API 要求
4. **响应解析失败** - API 返回格式异常

**需要检查**:
```bash
# 1. 检查.env 文件中的 API Key
grep "^ARK_API_KEY=" .env

# 2. 测试豆包 API
python3 scripts/test_doubao.py

# 3. 查看完整的 AI 调用日志
grep "Doubao API" logs/app.log | tail -20
```

### 2.2 核心问题 2: 错误码处理 Bug 🟡

**错误信息**: `'tuple' object has no attribute 'code'`

**问题代码**:
```python
# diagnosis_orchestrator.py:474
'error_code': error_code.value.code if hasattr(error_code, 'value') else error_code.code,
```

**问题**:
- `_determine_error_code()` 方法返回的是 tuple
- 代码期望的是 ErrorCode 对象
- 导致 AttributeError

**修复方案**:
```python
# 修复错误码处理
error_code_or_tuple = self._determine_error_code(str(e))
if isinstance(error_code_or_tuple, tuple):
    # 从元组中提取错误码
    error_code = error_code_or_tuple[0] if error_code_or_tuple else None
    error_code_str = error_code.code if hasattr(error_code, 'code') else str(error_code)
else:
    error_code_str = error_code_or_tuple.value.code if hasattr(error_code_or_tuple, 'value') else error_code_or_tuple.code

return {
    'error_code': error_code_str,
    ...
}
```
