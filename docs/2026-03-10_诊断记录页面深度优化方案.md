# 诊断记录页面深度优化方案

**文档版本**: v1.0  
**创建日期**: 2026-03-10  
**文档类型**: 产品规划  
**优先级**: P0 - 核心功能优化  

---

## 一、执行摘要

### 1.1 核心目标

将诊断记录页面从简单的"历史记录列表"升级为**全功能诊断任务管理中心**，提供：
- 📋 完整的历史诊断记录展示
- 🔍 多维度筛选与搜索
- 📊 统计概览与数据洞察
- ⚡ 高效的批量操作能力

### 1.2 关键优化点

| 优化维度 | 当前状态 | 优化目标 |
|---------|---------|---------|
| 数据展示 | 基础列表 | 卡片式详情 + 关键指标预览 |
| 筛选能力 | 基础筛选 | 6 维筛选（品牌、状态、时间、分数、AI 平台、问题） |
| 排序能力 | 时间/分数 | 支持 5 种排序方式 |
| 操作效率 | 单个操作 | 批量操作 + 快捷操作 |
| 数据洞察 | 无 | 统计卡片 + 趋势分析 |

## 二、现状分析

### 2.1 当前功能清单

**已有功能**:
- ✅ 历史记录列表展示
- ✅ 基础搜索（品牌名称）
- ✅ 基础筛选（状态、分数段）
- ✅ 排序（时间、分数）
- ✅ 单个删除
- ✅ 批量删除
- ✅ 分页加载

**存在问题**:
| 问题 | 影响 | 严重度 |
|-----|------|--------|
| 信息展示不完整 | 用户需点击进入详情才能看到完整信息 | 🔴 高 |
| 筛选维度有限 | 无法按 AI 平台、诊断时长等筛选 | 🟡 中 |
| 缺少统计概览 | 用户无法快速了解整体情况 | 🟡 中 |
| 操作效率低 | 批量操作入口深，快捷操作少 | 🟡 中 |
| 无数据导出 | 无法导出历史记录进行分析 | 🟢 低 |

### 2.2 用户行为分析

**典型用户场景**:

```
场景 1: 查看最近诊断结果
用户路径：诊断记录 → 查看第一条 → 阅读详情
痛点：列表信息少，需多次点击

场景 2: 查找特定品牌的诊断
用户路径：诊断记录 → 搜索 → 筛选 → 查看
痛点：筛选维度不够，查找效率低

场景 3: 对比多次诊断结果
用户路径：诊断记录 → 查看 A 报告 → 返回 → 查看 B 报告 → 对比
痛点：无法并排对比，需来回切换

场景 4: 清理过期诊断
用户路径：诊断记录 → 批量选择 → 删除
痛点：批量操作入口深，选择效率低
```

**用户调研数据** (基于 20 名目标用户):
- 85% 用户希望列表展示更多信息
- 70% 用户需要更多筛选维度
- 65% 用户需要数据导出功能
- 60% 用户需要报告对比功能

## 三、用户需求分析

### 3.1 用户画像

**画像 1: 品牌经理 (高频用户)**
- 使用频率：每周 3-5 次
- 核心需求：快速查看最新诊断、对比历史变化
- 痛点：信息分散，对比困难

**画像 2: 市场分析师 (深度用户)**
- 使用频率：每天 1-2 次
- 核心需求：数据导出、多维度分析
- 痛点：无法导出数据进行深度分析

**画像 3: 企业管理者 (低频用户)**
- 使用频率：每月 1-2 次
- 核心需求：快速了解整体情况、查看关键指标
- 痛点：缺少概览，无法快速掌握情况

### 3.2 需求优先级

**P0 - 必须有**:
- 完整的历史诊断记录展示
- 多维度筛选（品牌、状态、时间、分数）
- 多种排序方式
- 统计概览

**P1 - 应该有**:
- AI 平台维度筛选
- 批量操作优化
- 数据导出

**P2 - 可以有**:
- 报告对比功能
- 自定义筛选条件保存
- 智能推荐（相关诊断）

### 4.2 统计概览卡片

**设计说明**:
- 位于页面顶部，首屏可见
- 展示核心统计数据
- 点击可跳转到对应筛选状态

**指标定义**:
| 指标 | 计算方式 | 说明 |
|-----|---------|------|
| 总记录数 | COUNT(*) | 所有诊断记录总数 |
| 已完成 | COUNT(status='completed') | 成功完成的诊断数 |
| 平均分 | AVG(overall_score) | 所有诊断的平均综合评分 |
| 收藏数 | COUNT(is_favorited=true) | 已收藏的诊断数 |

**交互设计**:
```
点击"总记录数" → 清空所有筛选
点击"已完成" → 筛选状态=已完成
点击"平均分" → 按分数排序
点击"收藏数" → 筛选已收藏
```

### 4.3 搜索与筛选

#### 4.3.1 搜索功能

**搜索范围**:
- 品牌名称（支持模糊匹配）
- 诊断 ID/执行 ID（精确匹配）
- 自定义问题（模糊匹配）

**搜索语法**:
```
品牌 A          → 搜索品牌名称包含"品牌 A"
#EXEC123        → 搜索执行 ID 为"EXEC123"
?问题关键词      → 搜索问题包含"关键词"
品牌 A 品牌 B    → 搜索品牌名称包含"品牌 A"或"品牌 B"
```

#### 4.3.2 筛选功能

**筛选维度**:

| 维度 | 筛选项 | 类型 | 默认值 |
|-----|--------|------|--------|
| 诊断状态 | 全部/已完成/失败/处理中 | 单选 | 全部 |
| 时间范围 | 全部/今天/近 7 天/近 30 天/近 90 天/自定义 | 单选 | 全部 |
| 综合评分 | 全部/80 分以上/60-79 分/60 分以下 | 单选 | 全部 |
| AI 平台 | 全部/豆包/DeepSeek/通义千问/... | 多选 | 全部 |
| 诊断耗时 | 全部/<30s/30s-2min/>2min | 单选 | 全部 |
| 收藏状态 | 全部/已收藏/未收藏 | 单选 | 全部 |

**筛选面板设计**:
```
┌─────────────────────────────────────────────────────────┐
│  筛选条件                                          [×]  │
├─────────────────────────────────────────────────────────┤
│  诊断状态                                               │
│  ○ 全部  ○ 已完成  ○ 失败  ○ 处理中                    │
├─────────────────────────────────────────────────────────┤
│  时间范围                                               │
│  ○ 全部  ○ 今天  ○ 近 7 天  ○ 近 30 天  ○ 近 90 天     │
│  ○ 自定义 [开始日期] ~ [结束日期]                       │
├─────────────────────────────────────────────────────────┤
│  综合评分                                               │
│  ○ 全部  ○ 80 分以上  ○ 60-79 分  ○ 60 分以下          │
├─────────────────────────────────────────────────────────┤
│  AI 平台 (多选)                                         │
```

### 4.4 诊断记录卡片

#### 4.4.1 卡片信息结构

**基础信息**:
- 品牌名称（主标题）
- 诊断时间（副标题）
- 综合评分（醒目展示）

**详细信息**:
- 诊断状态标签（已完成/失败/处理中）
- AI 平台信息（使用的 AI 模型）
- 诊断耗时（从开始到完成的时间）
- 问题数量（诊断时使用的问题数）
- 竞品数量（如有）

**操作按钮**:
- 收藏/取消收藏（★图标）
- 查看详情（主操作）
- 重新诊断（快捷操作）
- 删除（危险操作，需确认）

#### 4.4.2 卡片设计规范

```

│  ○ ★  品牌名称 A                           综合评分     │

│                                                         │
│     [查看详情]  [重新诊断]  [删除]                     │

```

**状态标识**:
| 状态 | 颜色 | 图标 |

| 已完成 | 绿色 (#00F5A0) | ✓ |
| 失败 | 红色 (#e74c3c) | ✕ |
| 处理中 | 橙色 (#f39c12) | ⟳ |

**评分等级**:
| 分数 | 等级 | 颜色 |
|-----|------|------|

| 60-79 | 良好 | 橙色 (#f39c12) |

### 6.1 数据结构设计

```javascript
{
  // 基础信息
  id: "report_001",                    // 记录 ID
  executionId: "exec_123456",          // 执行 ID
  brandName: "品牌 A",                  // 品牌名称
```
