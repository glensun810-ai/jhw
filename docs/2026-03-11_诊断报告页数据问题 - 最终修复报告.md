## 一、问题总结

### 1.1 用户反馈

页面显示全空：
- 报告摘要：0 总样本数、0 关键词、0 涉及品牌
- 品牌分布：空
- 情感分布：空
- 关键词云：空

### 1.2 根本原因

**三个关键问题**:

| 问题 | 描述 | 影响 |
|-----|------|------|
| **问题 1** | Storage key 不匹配 | Storage 有数据但读取不到 |
| **问题 2** | 全局变量未检查 | 诊断完成后数据未传递到报告页 |
| **问题 3** | WebSocket 失败 | 实时推送不可用，降级逻辑缺失 |

---

## 二、修复方案

### 2.1 修复 1: Storage key 更正 ✅

**文件**: `miniprogram/pages/report-v2/report-v2.js`  
**位置**: `loadReportData()` 方法，第 656-688 行

**修复前**:
```javascript
const storageData = wx.getStorageSync('last_diagnostic_results');
```

**修复后**:
```javascript
const storageKey = 'diagnosis_result_' + id;
const storageData = wx.getStorageSync(storageKey);
```

**效果**: 正确读取 Storage 中保存的诊断数据

---

### 2.2 修复 2: 优先检查全局变量 ✅

**文件**: `miniprogram/pages/report-v2/report-v2.js`  
**位置**: `initPage()` 方法，第 104-185 行

**新增逻辑**:
```javascript
// 【P0 关键修复】优先检查全局变量（诊断刚完成）
const app = getApp();
if (app && app.globalData && app.globalData.pendingReport) {
  const pendingReport = app.globalData.pendingReport;
  
  if (pendingReport.executionId === options.executionId) {
    console.log('✅ executionId 匹配，直接使用全局变量数据');
    
    // 立即设置数据
    this.setData({
      brandDistribution: pendingReport.dashboardData?.brandDistribution || {},
      sentimentDistribution: pendingReport.dashboardData?.sentimentDistribution || {},
      keywords: pendingReport.dashboardData?.keywords || [],
      brandScores: pendingReport.dashboardData?.brandScores || {},
      isLoading: false,
      hasError: false,
      dataSource: 'globalData'
    });
    
    return;  // 不再执行后续逻辑
  }
}
```

**效果**: 诊断完成后立即使用全局变量数据，无需等待 WebSocket/轮询

---

### 2.3 修复 3: 诊断完成后主动加载 ✅

**文件**: `miniprogram/pages/report-v2/report-v2.js`  
**位置**: `handleComplete()` 方法，第 529-577 行

**新增逻辑**:
```javascript
handleComplete(result) {
  console.log('🎉 诊断完成');
  
  // 【P0 关键修复】立即从全局变量或 Storage 加载数据
  setTimeout(async () => {
    // 先检查全局变量
    const app = getApp();
    if (app && app.globalData && app.globalData.pendingReport) {
      console.log('✅ 全局变量有数据，直接使用');
      const pendingReport = app.globalData.pendingReport;
      this.setData({
        brandDistribution: pendingReport.dashboardData?.brandDistribution || {},
        sentimentDistribution: pendingReport.dashboardData?.sentimentDistribution || {},
        keywords: pendingReport.dashboardData?.keywords || [],
        brandScores: pendingReport.dashboardData?.brandScores || {}
      });
    } else {
      console.log('⚠️ 全局变量无数据，从 Storage 加载');
      await this.loadReportData(this.data.executionId);
    }
  }, 1500);
}
```

**效果**: 诊断完成后主动加载数据，不依赖 WebSocket/轮询

## 九、技术亮点

### 9.1 架构优化

**三层数据源策略**:
```
1. 全局变量 (优先) - 诊断刚完成时的数据
   ↓
2. Storage (备用) - 本地缓存数据
   ↓
3. 后端 API (降级) - 云端数据
```

**关键决策**:
- ✅ 优先使用全局变量，避免网络依赖
- ✅ Storage key 统一命名，避免不匹配
- ✅ 诊断完成主动推送，避免被动等待
