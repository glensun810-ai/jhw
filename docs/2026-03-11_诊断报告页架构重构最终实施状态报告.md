### 1.2 技术架构

#### 分层加载架构

```
0ms    → 第 1 层：核心信息
         - brandName ✅
         - overallScore ✅
         - overallGrade ✅
         - loading: false ✅

100ms  → 第 2 层：分析数据
         - dimension_scores ✅ (后端预计算或前端降级)
         - sovShare ✅ (后端数据)
         - sentimentScore ✅ (后端数据)
         - highRisks ✅ (后端预计算或前端降级)
         - sourceList ✅ (后端数据)

200ms  → 第 3 层：详细结果
         - detailedResults ✅ (简化，前 5 条)
         - aiModels ✅ (直接获取)
         - competitors ✅ (直接获取)
         - questions ✅ (直接获取)
```

#### 数据流

```
后端 API
    ↓
/test/status/{task_id}
    ↓
返回数据（包含预计算字段）
    ↓
前端 processHistoryDataOptimized
    ↓
分层 setData
    ↓
页面渲染
```
