## 1. 产品概述

### 1.1 产品简介

**云程企航**（进化湾 GEO）是一款基于人工智能技术的品牌诊断与分析系统，通过调用多个主流 AI 平台（豆包、DeepSeek、通义千问、ChatGPT、Gemini、智谱等），为企业提供全方位的品牌健康度评估、竞争情报分析和战略建议。

### 1.2 核心价值

- 🤖 **多 AI 引擎驱动**: 支持 7+ 主流 AI 平台，自动故障转移，确保服务高可用
- 📊 **深度品牌洞察**: 12 维度品牌评分体系，语义分析 + 情感分析双重引擎
- 🔍 **竞争情报分析**: 竞品对比、拦截风险分析、市场机会识别
- 📈 **实时进度追踪**: 诊断过程可视化，实时查看 AI 分析进度
- 📱 **便捷小程序**: 微信小程序即用即走，支持历史记录云端同步

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 品牌健康诊断 | 定期评估品牌市场表现和消费者认知 |
| 竞品分析 | 对比多个品牌，识别竞争优势和劣势 |
| 市场机会发现 | 发现潜在的市场机会和威胁 |
| 战略决策支持 | 为品牌战略调整提供数据支持 |

## 3. 核心功能详解

### 3.1 品牌诊断

#### 3.1.1 功能说明

品牌诊断是系统的核心功能，通过 AI 模型对指定品牌进行多维度分析，生成包含以下内容的诊断报告：

- **品牌综合评分**: 0-100 分的综合评分
- **12 维度分析**: 品牌力、传播力、影响力等维度评分
- **趋势分析**: 品牌发展趋势预测（上升/持平/下降）
- **竞争分析**: 与竞品的对比分析
- **信源分析**: 正面/负面信源统计
- **优化建议**: 针对性的品牌优化建议

#### 3.1.2 使用流程

```
输入品牌名称 → 选择 AI 模型 → 启动诊断 → 实时查看进度 → 查看报告
```

**详细步骤**:

1. **输入品牌信息**
   - 主品牌名称（必填）
   - 竞品品牌名称（可选，最多 5 个）

2. **选择 AI 模型**
   - 支持单选或多选
   - 推荐选择 2-3 个模型以获得更全面分析
   - 系统会自动进行故障转移

3. **自定义问题**（可选）
   - 输入特定关注的问题
   - 例如："品牌在年轻群体中的认知度如何？"

4. **启动诊断**
   - 点击"开始诊断"按钮
   - 系统生成诊断任务 ID

5. **查看进度**
   - 实时显示当前进度百分比
   - 显示当前执行阶段
   - 预计剩余时间

6. **查看报告**
   - 综合评分概览
   - 各维度详细分析
   - 竞品对比图表
   - 优化建议列表

#### 3.1.3 诊断报告解读

**综合评分构成**:
```
综合评分 = 品牌力 (30%) + 传播力 (25%) + 影响力 (20%) + 创新力 (15%) + 忠诚度 (10%)
```

**趋势判断标准**:
| 趋势 | 说明 |
|------|------|
| 📈 上升 | 品牌各项指标呈上升趋势，建议加大投入 |
| ➡️ 持平 | 品牌发展稳定，建议保持现状 |
| 📉 下降 | 品牌面临挑战，建议及时调整策略 |

**信源分析**:
- **正面信源**: 媒体报道、用户好评、行业认可等
- **负面信源**: 负面新闻、用户投诉、舆情风险等

### 3.2 多 AI 模型支持

#### 3.2.1 支持的 AI 平台

| 平台 | 模型 ID | 特点 | 适用场景 |
|------|--------|------|---------|
| **豆包** | doubao-seed-1-8 | 中文理解强，响应快 | 通用品牌分析 |
| **DeepSeek** | deepseek-chat | 推理能力强 | 深度分析 |
| **通义千问** | qwen-turbo | 多语言支持 | 国际化品牌 |
| **ChatGPT** | gpt-3.5-turbo | 全球知识覆盖 | 海外市场分析 |
| **Gemini** | gemini-pro | 多模态理解 | 综合评估 |
| **智谱 AI** | glm-4 | 中文专业领域强 | 垂直行业分析 |
| **文心一言** | ernie-bot | 百度生态整合 | 国内市场分析 |

#### 3.2.2 模型优先级配置

系统支持配置模型优先级，自动选择最高优先级的可用模型：

```bash
# .env 配置示例
DOUBAO_MODEL_PRIORITY_1=doubao-seed-1-8-251228
DOUBAO_MODEL_PRIORITY_2=doubao-seed-2-0-mini-260215
DOUBAO_AUTO_SELECT_MODEL=true
```

#### 3.2.3 故障自动转移

当首选模型不可用时，系统会自动：
1. 检测模型故障（超时/错误响应）
2. 自动切换到下一个优先级模型
3. 记录故障日志
4. 继续执行诊断任务

### 3.3 历史记录

#### 3.3.1 本地历史

- 自动保存最近 50 次诊断记录
- 支持本地搜索和筛选
- 支持批量删除

**操作路径**: 首页 → 底部导航"历史"

#### 3.3.2 云端同步

- 登录用户自动同步历史记录
- 支持跨设备访问
- 永久保存（除非手动删除）

**操作路径**: 我的 → 个人中心 → 云端历史

#### 3.3.3 历史记录管理

**查看历史**:
1. 进入"历史"页面
2. 按时间/品牌名称排序
3. 点击任意记录查看详情

**删除历史**:
1. 左滑单条记录 → 删除
2. 或点击右上角"编辑" → 批量选择 → 删除

**收藏历史**:
1. 点击记录右侧"⭐"图标
2. 收藏的记录可在"收藏夹"中查看

#### 3.4.1 支持的格式

| 格式 | 说明 | 适用场景 |
|------|------|---------|
| **PDF** | 完整报告，包含图表 | 正式汇报、存档 |
| **图片** | 关键指标截图 | 快速分享 |
| **Excel** | 原始数据表格 | 二次分析 |

#### 3.4.2 导出步骤

1. 打开诊断报告详情页
2. 点击右上角"导出"按钮
3. 选择导出格式
4. 选择保存位置
5. 等待生成完成

#### 3.4.3 PDF 报告内容

- 封面（品牌名称、诊断日期）
- 执行摘要
- 综合评分
- 维度分析
- 竞品对比
- 趋势预测
- 优化建议
- 附录（数据来源、方法论）

#### 3.5.1 图表类型

- **分享**: 长按保存图片

### 3.6 缓存与性能

#### 3.6.1 智能缓存

系统会自动缓存：
- 相同品牌的诊断结果（24 小时内）
- AI 模型响应结果
- 用户输入草稿

#### 3.6.2 缓存管理

**查看缓存**:
- 设置 → 存储管理 → 查看缓存大小

**清理缓存**:
- 设置 → 存储管理 → 一键清理

#### 3.6.3 性能优化

| 优化项 | 说明 |
|--------|------|
| 动态轮询间隔 | 根据进度自动调整轮询频率 |
| 增量聚合 | 只处理新增数据，减少重复计算 |
| 数据库索引 | 优化查询性能 |
| 连接池 | 复用数据库连接 |

## 7. 常见问题

### 7.1 使用问题

#### Q1: 诊断任务一直显示"等待中"怎么办？

**可能原因**:
- 网络连接不稳定
- AI 平台服务繁忙
- 后端服务未启动

**解决方案**:
1. 检查网络连接
2. 等待 1-2 分钟后刷新页面
3. 如超过 5 分钟未完成，尝试重新发起诊断
4. 联系管理员检查后端服务状态

#### Q2: 诊断结果为 0 分或空数据？

**可能原因**:
- AI 平台返回异常
- 数据解析失败
- 缓存数据过期

**解决方案**:
1. 清除缓存后重试
2. 更换 AI 模型重新诊断
3. 检查后端日志
4. 联系技术支持

#### Q3: 如何查看历史诊断记录？

**操作步骤**:
1. 打开小程序
2. 点击底部导航"历史"
3. 浏览或搜索历史记录
4. 点击任意记录查看详情

#### Q4: 导出的 PDF 无法打开？

**可能原因**:
- 文件下载不完整
- PDF 阅读器不兼容

**解决方案**:
1. 重新下载 PDF 文件
2. 使用 Adobe Acrobat 或系统自带 PDF 阅读器
3. 尝试导出为其他格式

### 7.2 开发者问题

#### Q5: 后端服务启动失败？

**常见错误及解决**:

**错误 1**: `ModuleNotFoundError`
```bash
# 解决方案：安装依赖
cd backend_python
pip3 install -r requirements.txt
```

**错误 2**: `Address already in use`
```bash
# 解决方案：更换端口
export PORT=5001
python3 run.py
```

**错误 3**: `.env file not found`
```bash
# 解决方案：创建配置文件
cp .env.example .env
# 编辑 .env 填入配置
```

#### Q6: API 调用返回 401 错误？

**可能原因**:
- API Key 配置错误
- Token 过期

**解决方案**:
1. 检查 `.env` 中 API Key 配置
2. 确认 API Key 未过期
3. 重启后端服务

#### Q7: 数据库连接失败？

**解决方案**:
```bash
# 检查数据库文件
ls -la backend_python/data/

# 重建数据库
cd backend_python
python3 -c "from wechat_backend.database import init_db; init_db()"
```

#### Q8: 如何查看系统日志？

**查看实时日志**:
```bash
tail -f backend_python/logs/app.log
```

**查看错误日志**:
```bash
grep ERROR backend_python/logs/app.log
```

**清理旧日志**:
```bash
find backend_python/logs -name "*.log" -mtime +7 -delete
```

### 7.3 管理员问题

#### Q9: 如何添加管理员账号？

**操作步骤**:
1. 登录数据库管理工具
2. 执行 SQL:
```sql
UPDATE users SET role = 'admin' WHERE username = 'username';
```
3. 或使用管理后台（超级管理员）

#### Q10: 如何查看用户诊断记录？

**操作路径**:
管理后台 → 诊断管理 → 搜索用户 → 查看记录
