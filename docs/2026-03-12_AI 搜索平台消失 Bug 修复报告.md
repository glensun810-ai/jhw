## 二、根因分析

### 2.1 代码分析

**WXML 结构** (`pages/index/index.wxml`):
```xml
<!-- 国内 AI 模型 -->
<view class="ai-category {{selectedMarketTab !== 'domestic' ? 'hidden' : ''}}">
  <view class="category-header">
    <text class="category-title">国内 AI 平台</text>
    <button class="select-all-btn" bindtap="selectAllModels" data-type="domestic">全选</button>
  </view>
  <view class="model-grid">
    <view class="model-chip-pro"
          wx:for="{{domesticAiModels}}"
          wx:key="id"
          ...>
      <!-- 平台卡片内容 -->
    </view>
  </view>
</view>
```

**JavaScript 数据** (`pages/index/index.js`):
```javascript
data: {
  selectedMarketTab: 'domestic',
  domesticAiModels: [
    { name: 'DeepSeek', id: 'deepseek', checked: true, logo: 'DS', tags: ['综合', '代码'] },
    { name: '豆包', id: 'doubao', checked: true, logo: 'DB', tags: ['综合', '创意'] },
    // ...
  ],
  overseasAiModels: [...]
}
```

### 2.2 可能原因

**原因 1: 数据被清空** 🔴
```javascript
// 可能在某个地方执行了
this.setData({
  domesticAiModels: [],
  overseasAiModels: []
});
```

**原因 2: 样式覆盖** 🟡
```css
/* 诊断过程中可能应用了某个样式 */
.isTesting .ai-category {
  display: none;  /* ❌ 错误 */
}
```

**原因 3: 数据绑定失效** 🟡
```
wx:for="{{domesticAiModels}}"
当 domesticAiModels 为 undefined 或 null 时，不渲染
```

### 2.3 根本原因

**核心问题**: 在诊断过程中，`domesticAiModels` 和 `overseasAiModels` 数据可能被意外清空或覆盖

**触发场景**:
1. 用户点击"诊断中"按钮
2. 页面进入 `appState: 'testing'` 状态
3. 某个地方修改了 AI 平台数据
4. 平台卡片消失

## 三、修复方案

### 3.1 方案 A: 数据保护（推荐）✅

**修复文件**: `pages/index/index.js`

**修复内容**:
1. 在 `setData` 时保护 AI 平台数据
2. 添加数据验证
3. 添加自动恢复机制

**修复代码**:
```javascript
// 在诊断开始时保护数据
startBrandTest: function() {
  // 【Bug 修复 - 2026-03-12】保存 AI 平台数据快照
  this._aiModelsSnapshot = {
    domesticAiModels: JSON.parse(JSON.stringify(this.data.domesticAiModels)),
    overseasAiModels: JSON.parse(JSON.stringify(this.data.overseasAiModels))
  };
  
  // 诊断中...
  this.setData({
    isTesting: true,
    appState: 'testing',
    // ... 其他数据
  });
}

// 在诊断完成后恢复数据
handleDiagnosisComplete: function(parsedStatus, executionId) {
  // 【Bug 修复 - 2026-03-12】恢复 AI 平台数据
  if (this._aiModelsSnapshot) {
    this.setData({
      domesticAiModels: this._aiModelsSnapshot.domesticAiModels,
      overseasAiModels: this._aiModelsSnapshot.overseasAiModels
    });
    this._aiModelsSnapshot = null;
  }
  
  // ... 原有逻辑
}
```

### 3.2 方案 B: 样式保护（辅助）✅

**修复文件**: `pages/index/index.wxss`

**修复内容**:
```css
/* 【Bug 修复 - 2026-03-12】确保 AI 平台区域在诊断过程中可见 */
.ai-category {
  display: block !important;  /* 强制显示 */
}

/* 只在必要时隐藏 */
.ai-category.hidden {
  display: none !important;
}
```

### 3.3 方案 C: WXML 保护（辅助）✅

**修复文件**: `pages/index/index.wxml`

**修复内容**:
```xml
<!-- 添加数据检查 -->
<view class="ai-category {{selectedMarketTab !== 'domestic' ? 'hidden' : ''}}"
      wx:if="{{domesticAiModels && domesticAiModels.length > 0}}">
  <!-- ... -->
</view>

<!-- 空状态提示 -->
<view class="ai-category-empty" wx:else>
  <text>AI 平台加载中...</text>
</view>
```
