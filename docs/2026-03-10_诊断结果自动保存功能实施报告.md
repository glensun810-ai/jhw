## 一、需求背景

### 1.1 问题描述

**原问题**: 每次诊断完成后，详细结果没有自动保存到诊断记录中，导致：
- 用户无法在诊断记录页面查看历史诊断
- 诊断数据丢失，无法追溯
- 无法进行历史数据对比分析

### 1.2 用户需求

作为品牌经理，我希望：
- 每次诊断完成后自动保存详细结果
- 在诊断记录页面可以查看所有历史诊断
- 可以快速定位和对比特定诊断

**自动添加到记录列表**:
```javascript
// utils/storage-manager.js - saveDiagnosisResult
function saveDiagnosisResult(executionId, data) {
  // 1. 保存完整数据
  const key = StorageKey.DIAGNOSIS_RESULT + executionId;
  wx.setStorageSync(key, storageData);
  
  // 2. 自动添加到诊断记录列表
  addToDiagnosisHistory({
    executionId: executionId,
    brandName: storageData.brandName,
    createdAt: storageData.completedAt,
    status: 'completed',
    overallScore: storageData.data.overallScore || 0,
    aiModels: storageData.data.selectedModels || [],
    questionCount: (storageData.data.customQuestions || []).length,
    competitorCount: (storageData.data.competitorBrands || []).length,
    timestamp: storageData.timestamp
  });
}
```

### 4.2 诊断记录列表管理

**添加到列表**:
```javascript
function addToDiagnosisHistory(record) {
  // 获取现有历史记录
  let history = wx.getStorageSync(DIAGNOSIS_HISTORY_KEY) || [];
  
  // 检查是否已存在（避免重复添加）
  const exists = history.some(item => item.executionId === record.executionId);
  if (exists) {
    console.log('记录已存在，跳过添加');
    return;
  }
  
  // 添加到列表开头（最新的在前）
  history.unshift(record);
  
  // 限制最多保存 200 条记录
  if (history.length > 200) {
    history = history.slice(0, 200);
  }
  
  // 保存回 Storage
  wx.setStorageSync(DIAGNOSIS_HISTORY_KEY, history);
}
```

**数据过期策略**:
- 诊断记录列表：保留最多 200 条
- 完整诊断数据：90 天过期
- 最后一次诊断：90 天过期

## 六、使用说明

### 6.1 用户操作流程

**诊断并保存**:
```
1. 在 AI 搜索页面输入品牌信息
2. 点击"开始诊断"
3. 等待诊断完成
4. ✅ 自动保存到诊断记录
5. 自动跳转到报告详情页

**查看历史诊断**:
```
1. 点击底部"诊断记录"Tab

3. 点击任意记录查看详情
```

**筛选历史诊断**:
```
1. 点击"筛选"按钮
2. 选择筛选条件（状态、时间、分数等）
