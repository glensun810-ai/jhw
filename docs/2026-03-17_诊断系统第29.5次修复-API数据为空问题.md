# 诊断系统第 29.5 次修复 - API 数据为空问题

**问题**: 前端调用后端 API 获取报告时返回 `{"uniqueId":17737523875363016}`，数据为空

**时间**: 2026-03-17 21:30

---

## 问题分析

### 错误现象

前端日志：
```javascript
[DiagnosisService] ❌ res.data 为空，完整 res: {"uniqueId":17737523875363016}
[DiagnosisService] HTTP 获取报告失败：Error: 后端返回数据为空，请检查后端服务
```

### 根本原因

1. **前端请求的 executionId 在数据库中不存在**
   - 后端 API `/api/diagnosis/report/{executionId}` 返回"报告不存在"错误
   - 返回的数据包含错误信息，但没有实际的报告内容

2. **`uniqueId` 来源**
   - 这个 ID 看起来是微信小程序云开发生成的
   - 可能是前端从云函数调用历史中获取的 ID，而不是后端数据库中的 `execution_id`

3. **数据不一致**
   - 前端历史列表可能从云函数获取
   - 后端报告存储在 SQLite 数据库中
   - 两者的 ID 可能不匹配

---

## 修复方案

### 方案 1：前端使用正确的 executionId

**检查点**:
1. 确认前端从何处获取 `executionId`
2. 确保使用的是后端返回的 `execution_id`，而非云函数的 `uniqueId`

**修复位置**: `brand_ai-seach/miniprogram/pages/diagnosis/diagnosis.js`

```javascript
// ❌ 错误：使用云函数的 uniqueId
const executionId = res.result.uniqueId;

// ✅ 正确：使用后端返回的 execution_id
const executionId = res.result.execution_id || res.result.executionId;
```

### 方案 2：后端 API 返回更友好的错误信息

**文件**: `backend_python/wechat_backend/views/diagnosis_api.py`

当前后端已经返回了友好的错误信息：
```json
{
  "error": {
    "message": "报告不存在",
    "status": "not_found",
    "fallbackInfo": {...}
  },
  "validation": {
    "isEmptyData": true,
    "errors": ["报告不存在"]
  }
}
```

但前端没有正确处理这个错误。

### 方案 3：前端正确处理空数据错误

**文件**: `brand_ai-seach/miniprogram/services/diagnosisService.js`

**修复**: 检测后端返回的错误信息并显示友好提示

```javascript
// 在 _getFullReportViaHttp 方法中
if (result.validation?.isEmptyData || result.error?.status === 'not_found') {
  console.warn('[DiagnosisService] ⚠️ 报告不存在:', executionId);
  throw new Error('报告不存在，请确认执行 ID 是否正确');
}
```
