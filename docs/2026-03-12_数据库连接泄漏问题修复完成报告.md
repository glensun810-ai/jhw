## 一、问题回顾

### 1.1 泄漏现象

**日志证据**:
```
2026-03-12 00:08:06,042 - wechat_backend.database - WARNING - 
[连接泄漏] 连接超时未归还：id=6313272528, 年龄=36.8 秒，
thread_id=8387939328, 池中剩余=0, 使用中=1
```

### 1.2 根因分析

**问题文件**: `wechat_backend/database/transaction.py`

**问题代码**:
```python
# ❌ 修复前：直接创建连接，绕过连接池
conn = sqlite3.connect(DB_PATH)
# ... 使用连接
conn.close()
```

**影响**:
- 连接绕过连接池监控
- 高并发下连接未及时关闭
- 连接池检测到"泄漏"

## 二、修复内容

### 2.1 修复文件

**文件**: `wechat_backend/database/transaction.py`

**修复行数**: 约 100 行

### 2.2 修复要点

**1. `database_transaction()` 函数**:
```python
# ✅ 修复后：从连接池获取连接
pool = get_db_pool()
conn = pool.get_connection(timeout=15.0)
# ... 使用连接
pool.return_connection(conn)  # 归还到连接池
```

**2. `database_readonly_transaction()` 函数**:
```python
# ✅ 修复后：从连接池获取只读连接
pool = get_db_pool()
conn = pool.get_connection(timeout=15.0)
conn.execute('PRAGMA query_only=ON')  # 设置只读
pool.return_connection(conn)  # 归还到连接池
```

**3. `get_connection()` 函数**:
```python
# ✅ 修复后：统一从连接池获取
pool = get_db_pool()
return pool.get_connection(timeout=15.0)
```

### 7.2 经验总结

**教训**:
- 直接 `sqlite3.connect()` 绕过连接池监控
- 事务上下文管理器必须使用连接池
- 连接必须归还到连接池，而不是关闭

**最佳实践**:
```python
# ✅ 推荐：使用连接池
pool = get_db_pool()
conn = pool.get_connection()
# ... 使用
pool.return_connection(conn)

# ❌ 不推荐：直接创建连接
conn = sqlite3.connect(DB_PATH)
```
