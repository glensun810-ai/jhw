## 一、问题现象

### 1.1 用户反馈

**用户疑问**: 
> "为什么用户没有选择豆包 AI 平台，但诊断结果是因为豆包 API 配置问题导致的失败？"

**期望行为**:
- 用户选择哪个平台，就调用哪个平台
- 未选择的平台不应该被调用

**实际行为**:
- 即使用户未选择豆包，后端仍然调用豆包 API
- 导致豆包 API 配置错误时，诊断失败

### 2.2 核心问题

**问题 1: 配置开关优先级过高** 🔴

**代码**:
```python
# Config.is_doubao_auto_select() 默认返回 True
DOUBAO_AUTO_SELECT_MODEL = os.environ.get('DOUBAO_AUTO_SELECT_MODEL', 'true').lower() == 'true'
```

**影响**:
- 默认启用自动选择
- 即使用户未选择豆包，也会调用豆包 API
- 豆包 API 配置错误时，诊断失败

**问题 2: 未尊重用户选择** 🔴

**期望逻辑**:
```python
# 应该检查用户是否选择了豆包
if model_name == 'doubao' and Config.is_doubao_auto_select():
    # 只有用户选择了豆包，才使用优先级适配器
    adapter = DoubaoPriorityAdapter()
```

**实际逻辑**:
```python
# 实际代码未检查用户选择
if Config.is_doubao_auto_select():
    # 直接强制使用豆包优先级适配器
    adapter = DoubaoPriorityAdapter()
```

## 三、问题定位

### 3.1 错误日志

```
2026-03-11 00:36:01,695 - ERROR - diagnosis_orchestrator.py:1567 - 
[Orchestrator] ❌ 阶段 8 完成：失败处理 - 6867b1da-3463-491b-9da9-b4f7d6803c53, 
错误=AI 调用失败：AI 调用返回空结果
```

**错误链路**:
```
用户选择 DeepSeek
  ↓
后端接收 selected_models = ['deepseek']
  ↓
AI 适配器工厂创建 DeepSeek 适配器
  ↓
但 DoubaoPriorityAdapter 被意外初始化
  ↓
DoubaoPriorityAdapter 尝试调用豆包 API
  ↓
豆包 API Key 配置错误
  ↓
AI 调用返回空结果
  ↓
诊断失败
```

### 3.2 为什么会调用豆包？

**原因**: `DoubaoPriorityAdapter` 在初始化时，会**自动尝试调用豆包 API**

**代码**:
```python
# doubao_priority_adapter.py:176
def _init_adapter(self) -> bool:
    api_logger.info(f"[DoubaoPriority] 尝试初始化适配器，优先级模型列表：{self.priority_models}")
    
    for i, model_id in enumerate(self.priority_models):
        # ❌ 问题：这里会尝试调用豆包 API 进行健康检查
        adapter = DoubaoAdapter(api_key=self.api_key, model_name=model_id)
        adapter._health_check()  # ❌ 调用豆包 API
```

**影响**:
- 即使前端未选择豆包
- `DoubaoPriorityAdapter` 初始化时会调用豆包 API
- 豆包 API 配置错误导致诊断失败

## 四、修复方案

### 4.1 紧急修复：关闭自动选择

**方案**: 修改 `.env` 配置

**操作**:
```bash
# 编辑 .env 文件
vim .env

# 添加或修改
DOUBAO_AUTO_SELECT_MODEL=false
```

**效果**:
- ✅ 立即生效
- ✅ 不再强制调用豆包 API
- ✅ 尊重用户选择

**副作用**:
- ⚠️ 豆包多模型优先级功能失效
- ⚠️ 需要手动配置豆包模型

### 4.2 中期修复：修复适配器工厂

**文件**: `backend_python/wechat_backend/ai_adapters/factory.py`

**修复代码**:
```python
@staticmethod
def create(model_name: str, **kwargs) -> AIClient:
    """
    创建 AI 适配器

    【P0 修复 - 2026-03-11】尊重用户选择
    只有用户选择了豆包，才使用豆包优先级适配器
    """
    if model_name == 'doubao':
        # ✅ 只有用户选择了豆包，才使用优先级适配器
        from wechat_backend.ai_adapters.doubao_priority_adapter import DoubaoPriorityAdapter
        
        # 检查是否启用自动选择

            return DoubaoPriorityAdapter()
        else:
            # 不启用自动选择，使用普通豆包适配器
            from wechat_backend.ai_adapters.doubao_adapter import DoubaoAdapter
            return DoubaoAdapter(model_name=model_name)
    
    # 其他平台使用对应适配器
    # ...
```

**效果**:
- ✅ 尊重用户选择
- ✅ 保留豆包多模型优先级功能
- ✅ 不影响其他平台

### 4.3 长期修复：架构重构

**问题**: 为什么 `DoubaoPriorityAdapter` 会在初始化时调用 API？

**修复**: 延迟初始化

**代码**:
```python
class DoubaoPriorityAdapter(AIClient):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        # ❌ 修复前：初始化时立即尝试调用 API
        # self._init_adapter()
        
        # ✅ 修复后：延迟初始化，首次调用时才初始化
        self._initialized = False
        self._adapter = None
    
    def _ensure_initialized(self):
        """确保适配器已初始化"""
        if not self._initialized:
            self._init_adapter()
            self._initialized = True
    
    async def chat_completion(self, messages, **kwargs):
        # ✅ 首次调用时才初始化
        self._ensure_initialized()
        
        if not self._adapter:
            raise ValueError("豆包适配器初始化失败")
        
        return await self._adapter.chat_completion(messages, **kwargs)
```

**效果**:
- ✅ 避免无用的 API 调用
- ✅ 提升初始化速度
- ✅ 减少错误日志
