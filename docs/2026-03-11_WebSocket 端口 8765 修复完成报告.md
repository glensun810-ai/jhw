## 二、根因分析

**技术根因**: `websockets 16.0` 版本中 `serve()` 需要在已有 event loop 的线程中调用，不能使用 `asyncio.run()`。

**错误信息**:
```
RuntimeError: no running event loop
```

**问题代码**:
```python
# ❌ 错误方式
def run_server():
    asyncio.run(
        serve(handle_websocket_connection, "0.0.0.0", 8765)
    )
```

---

## 三、修复方案

**修复代码** (`wechat_backend/app.py`):

```python
def start_websocket_server():
    import asyncio
    import threading
    
    try:
        from websockets.asyncio.server import serve
        from wechat_backend.websocket_route import handle_websocket_connection

        def run_server():
            # ✅ 创建新的事件循环
            loop = asyncio.new_event_loop()
            asyncio.set_event_loop(loop)
            
            async def start():
                async with serve(
                    handle_websocket_connection,
                    "0.0.0.0",
                    8765,
                    ping_interval=20,
                    ping_timeout=10
                ):
                    app_logger.info("✅ [WebSocket] 服务器已启动在端口 8765")
                    await asyncio.Future()  # 保持运行
            
            loop.run_until_complete(start())

        thread = threading.Thread(target=run_server, daemon=True)
        thread.start()
        
        # 等待确认线程启动
        import time
        time.sleep(2)
        
        # 检查线程是否还在运行
        if thread.is_alive():
            return True
        else:
            raise RuntimeError("WebSocket 服务器线程意外退出")
            
    except Exception as e:
        app_logger.error(f"❌ [WebSocket] 服务器启动失败：{e}")
        return False
```

**关键点**:
1. ✅ 创建新的事件循环：`asyncio.new_event_loop()`
2. ✅ 设置事件循环：`asyncio.set_event_loop(loop)`
3. ✅ 使用 `loop.run_until_complete()` 而不是 `asyncio.run()`
4. ✅ 使用 `await asyncio.Future()` 保持运行

### 7.2 技术要点

**关键修复**:
1. 使用 `asyncio.new_event_loop()` 创建事件循环
2. 使用 `asyncio.set_event_loop()` 设置事件循环

### 7.3 经验总结

**教训**:
- `asyncio.run()` 不能在已有 loop 的线程中调用
- 在后台线程运行 asyncio 需要手动管理事件循环
- `websockets 16.0` 版本 API 有变化

**最佳实践**:
- 在后台线程中使用 `asyncio.new_event_loop()`
- 使用 `async with` 管理 WebSocket 服务器
- 添加启动确认和错误处理
