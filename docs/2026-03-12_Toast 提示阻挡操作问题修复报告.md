## 二、根因分析

### 2.1 问题代码

**文件 1**: `miniprogram/pages/report-v2/report-v2.js` (第 353-357 行)

```javascript
handleWebSocketConnected() {
  // ...
  
  // ❌ 问题：显示 Toast 提示
  showToast({
    title: '实时推送已连接',
    icon: 'success',
    duration: 1500
  });
}
```

**文件 2**: `miniprogram/utils/uiHelper.js` (第 12-21 行)

```javascript
function showToast(options) {
  const defaultOptions = {
    icon: 'success',
    duration: 2000,
    mask: true  // ❌ 问题：显示遮罩层，阻挡用户操作
  };
  
  wx.showToast({
    ...defaultOptions,
    ...options
  });
}
```

### 2.2 根本原因

**两个关键问题**:

1. **Toast 设置了 `mask: true`**:
   - 显示半透明遮罩层
   - 阻挡所有用户交互
   - 即使用户点击也无法关闭

2. **WebSocket 连接提示不必要**:
   - 技术细节，用户无需知道
   - 连接成功是正常流程，无需提示
   - 只在连接失败时才需要提示

### 2.3 微信小程序 Toast 行为

**`mask: true` 的效果**:
```
显示 Toast 时:
├─ 显示半透明遮罩层
├─ 阻挡所有点击事件
├─ 无法操作其他界面元素
└─ Toast 消失后遮罩才消失
```

**问题场景**:
```
WebSocket 连接成功 → 显示 Toast (mask: true)
                      ↓
                 遮罩层覆盖界面
                      ↓
                 用户无法操作
                      ↓
                 Toast 未自动消失 (Bug)
                      ↓
                 界面永久被阻挡
```

## 三、修复方案

### 3.1 修复 1: 移除不必要的 Toast 提示 ✅

**文件**: `miniprogram/pages/report-v2/report-v2.js`

**修复前**:
```javascript
handleWebSocketConnected() {
  this.setData({
    isWebSocketConnected: true,
    connectionMode: 'websocket',
    connectionState: ConnectionState.CONNECTED
  });

  showToast({
    title: '实时推送已连接',
    icon: 'success',
    duration: 1500
  });
}
```

**修复后**:
```javascript
handleWebSocketConnected() {
  this.setData({
    isWebSocketConnected: true,
    connectionMode: 'websocket',
    connectionState: ConnectionState.CONNECTED
  });

  // 【修复 - 2026-03-12】移除 Toast 提示，避免阻挡用户操作
  // WebSocket 连接是正常流程，无需提示用户
}
```

### 3.2 修复 2: 修改 Toast 默认配置 ✅

**文件**: `miniprogram/utils/uiHelper.js`

**修复前**:
```javascript
function showToast(options) {
  const defaultOptions = {
    icon: 'success',
    duration: 2000,
    mask: true  // ❌ 遮罩层阻挡操作
  };

```

**修复后**:
```javascript

    icon: 'success',

  };
  // ...
}
```

## 五、其他 Toast 使用检查

### 5.1 检查清单

| 位置 | 用途 | 是否需要 | 状态 |
|-----|------|---------|------|
| `handleWebSocketConnected` | 连接成功 | ❌ 不需要 | ✅ 已移除 |
| `handleWebSocketDisconnected` | 连接断开 | ✅ 需要 | ⚠️ 待检查 |
| `handlePollingError` | 轮询错误 | ✅ 需要 | ⚠️ 待检查 |
| `handleComplete` | 诊断完成 | ✅ 需要 | ⚠️ 待检查 |
| `handleTimeout` | 超时提示 | ✅ 需要 | ⚠️ 待检查 |

### 5.2 修复建议

**原则**:
1. 成功提示可省略（正常流程）
2. 错误/异常必须提示（异常情况）
3. 所有 Toast 不应使用 `mask: true`

**示例**:
```javascript
// ✅ 推荐：错误提示（保留）
showToast({
  title: '实时推送已断开，尝试重连...',
  icon: 'none',
  duration: 2000,
  mask: false  // 不阻挡操作
});

// ❌ 不推荐：成功提示（移除）
showToast({
  title: '实时推送已连接',
  icon: 'success',
  duration: 1500,
  mask: true
});
```

### 6.2 经验总结

**教训**:
- Toast 默认不应使用 `mask: true`
- 成功提示可以省略，减少干扰
- 只在必要时显示提示（错误、异常）

**最佳实践**:
```javascript
// ✅ 推荐：不阻挡操作的 Toast
showToast({
  title: '操作成功',
  icon: 'success',
  duration: 1500,
  mask: false
});

// ❌ 不推荐：阻挡操作的 Toast

  title: '操作成功',
  icon: 'success',
  duration: 2000,
  mask: true  // 会阻挡用户操作
});
```
