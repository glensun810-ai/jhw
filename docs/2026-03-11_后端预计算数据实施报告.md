## 一、实施概述

### 1.1 问题背景

**前端需求**: 分层加载需要后端预计算数据

**后端现状**: 未返回预计算字段
- ❌ `dimension_scores` - 评分维度
- ❌ `recommendation_data` - 问题诊断
- ❌ `quality_score` - 质量评分

**影响**: 前端需要现场计算，导致性能下降

### 1.2 解决方案

**方案**: 后端添加预计算服务，在返回数据时自动计算

**策略**: 
- 已有数据：直接使用
- 缺失数据：现场预计算
- 渐进增强：不影响现有功能

## 二、实施详情

### 2.1 新增文件

**文件**: `backend_python/wechat_backend/services/precompute_service.py`

**功能**:
- `calculate_dimension_scores()` - 计算评分维度
- `generate_recommendation_data()` - 生成问题诊断
- `calculate_quality_score()` - 计算质量评分
- `precompute_all()` - 预计算所有数据

**代码量**: ~250 行

### 2.2 修改文件

**文件**: `backend_python/wechat_backend/views.py`

**修改位置**: `get_task_status_api()` 函数

**修改内容**:
```python
# 【P0 架构优化 - 2026-03-11】添加预计算数据（前端分层加载支持）
detailed_results_list = summary.get('detailed_results', [])
source_intelligence = summary.get('source_intelligence') or summary.get('source_intelligence_map')

# 如果后端未返回预计算数据，现场计算
if detailed_results_list and not summary.get('dimension_scores'):
    # 导入预计算服务
    from wechat_backend.services.precompute_service import precompute_diagnosis_data
    precomputed = precompute_diagnosis_data(detailed_results_list, source_intelligence)
    response_data['dimension_scores'] = precomputed['dimension_scores']
    # 如果 recommendation_data 为空，使用预计算
    if not response_data['recommendation_data']:
        response_data['recommendation_data'] = precomputed['recommendation_data']
    response_data['quality_score'] = precomputed['quality_score']
```

## 三、预计算服务详解

### 3.1 评分维度计算

**方法**: `calculate_dimension_scores()`

**输入**: `detailed_results: List[Dict]`

**输出**:
```python
{
    'authority': 80,      # 权威度
    'visibility': 75,     # 可见度
    'purity': 90,         # 纯净度
    'consistency': 85     # 一致性
}
```

**计算逻辑**:
```python
# 限制计算数据量，最多 50 条（性能优化）
limited_results = detailed_results[:50]

# 计算平均分
authority = round(sum(r.authority_score for r in limited_results) / len(limited_results))
```

**性能**: ~5-10ms（50 条数据）

### 3.2 问题诊断生成

**方法**: `generate_recommendation_data()`

**输入**: 
- `detailed_results: List[Dict]`
- `source_intelligence: Dict`（可选）

**输出**:
```python
{
    'risk_levels': {
        'high': [
            {
                'id': 'risk_1',
                'text': '品牌 A - 问题描述',
                'source': '豆包',
                'score': 45,
                'question': '品牌知名度如何',
                'response': '...'
            }
        ],
        'medium': [...]
    },
    'priority_recommendations': [
        {
            'id': 'rec_1',
            'text': '提升 AI 认知质量',
            'priority': '高',
            'description': '发现 5 个低分评价，需要重点关注'
        }
    ]
}
```

**生成逻辑**:
1. 识别低分结果（<60 分）→ 高风险
2. 识别中等分数（60-75 分）→ 中风险
3. 生成优化建议

**性能**: ~10-20ms（50 条数据）

### 3.3 质量评分计算

**方法**: `calculate_quality_score()`

**输入**: 
- `dimension_scores: Dict`
- `detailed_results: List[Dict]`

**输出**:
```python
{
    'overall_score': 85,
    'dimension_scores': {...}
}
```

**计算逻辑**:
```python
overall_score = round(
    (authority + visibility + purity + consistency) / 4
)
```

**性能**: ~1ms
