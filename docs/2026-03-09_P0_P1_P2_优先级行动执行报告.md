## 关键问题汇总

### 🔴 问题 1: 豆包 API Key 配置格式不正确

**严重级别**: 🔴 致命（阻断所有诊断功能）

**症状**:
```
HTTP 401 Unauthorized
{"error":{
  "code":"AuthenticationError",
  "message":"The API key format is incorrect",
  "request id":"0217730613981484d6fef69588cf5fe521a1e6494fd0573d3171b"
}}
```

**可能原因**:
1. API Key 值本身格式错误（缺少前缀/后缀）
2. API Key 包含不可见字符（空格、换行）
3. API Key 已过期或被吊销
4. .env 文件读取时解析错误

**验证方法**:
```bash
# 1. 检查.env 文件中的 API Key
grep "^ARK_API_KEY=" .env

# 2. 检查 API Key 长度（正常应该是 32-64 字符）
grep "^ARK_API_KEY=" .env | cut -d'=' -f2 | wc -c

# 3. 检查是否有不可见字符
grep "^ARK_API_KEY=" .env | cut -d'=' -f2 | od -c | head
```

**解决方案**:
1. 登录豆包开放平台重新生成 API Key
2. 更新 .env 文件中的 `ARK_API_KEY` 配置
3. 重启 Flask 服务
4. 重新执行端到端测试

**操作步骤**:
```bash
# 1. 编辑.env 文件
vim .env

# 2. 更新 API Key（确保格式正确）
ARK_API_KEY=your-new-correct-api-key

# 3. 重启服务
cd backend_python
python3 run.py

# 4. 验证
python3 scripts/test_doubao.py
```

### 🟡 问题 2: 错误码处理逻辑 Bug

**严重级别**: 🟡 高（影响错误信息展示）

**位置**: `diagnosis_orchestrator.py:474`

**症状**:
```python
AttributeError: 'tuple' object has no attribute 'code'
```

**原因**: `_determine_error_code()` 方法返回 tuple 而非 ErrorCode 对象

**修复方案**:
```python
# 修复前
error_code = self._determine_error_code(str(e))
return {
    'error_code': error_code.value.code if hasattr(error_code, 'value') else error_code.code,
    ...
}

# 修复后
error_code_or_tuple = self._determine_error_code(str(e))
if isinstance(error_code_or_tuple, tuple):
    # 从元组中提取错误码
    error_code = error_code_or_tuple[0] if error_code_or_tuple else None
    error_code_str = error_code.code if hasattr(error_code, 'code') else str(error_code)
else:
    error_code_str = error_code_or_tuple.value.code if hasattr(error_code_or_tuple, 'value') else error_code_or_tuple.code

return {
    'error_code': error_code_str,
    ...
}
```
