## 📊 问题描述

### 用户报告

从诊断记录中，点击查看历史诊断报告详情页时，无法打开，提示：
> "模拟器长时间没有响应，请确认你的业务逻辑中是否有复杂运算，或者死循环。"

### 问题根因分析

经过深入代码审查，发现**三处**死循环风险点：

| 编号 | 文件 | 位置 | 问题 |
|------|------|------|------|
| 1 | `diagnosis.js` | Line 600 | 错误处理递归调用 |
| 2 | `report-v2.js` | Line 923 | 错误处理递归调用 |
| 3 | `reportService.js` | Line 185 | API 重试递归调用 |

---

## 🔍 详细根因分析

### 死循环点 1 & 2: 前端页面递归调用

**问题代码** (diagnosis.js 和 report-v2.js):
```javascript
wx.showModal({
  success: (res) => {
    if (res.confirm) {
      this.loadHistoryReport(reportId);  // ❌ 递归调用
    }
  }
});
```

**问题**: 用户点击"重试"时递归调用原方法。

---

### 死循环点 3: reportService.js API 重试

**问题代码** (reportService.js Line 185):
```javascript
async getFullReport(executionId, options = {}) {
  const retryCount = options.retryCount || 0;
  
  try {
    // ... API 调用
  } catch (error) {
    const errorType = this._classifyError(error);
    const canRetry = retryCount < this.maxRetryCount &&
                     errorType !== ErrorTypes.DATA_NOT_FOUND;
    
    if (canRetry) {
      await new Promise(resolve => setTimeout(resolve, this.retryDelay * (retryCount + 1)));
      return this.getFullReport(executionId, { retryCount: retryCount + 1 });  // ❌ 递归重试
    }
  }
}
```

**问题**: 
1. `maxRetryCount` 未定义（应该使用 `this.maxRetryCount` 但实际未定义）
2. 每次重试都递归调用自己
3. 如果 API 持续失败，会无限重试

## 🔧 最终修复方案

### 修复 1: diagnosis.js

**修复内容**:
1. 添加 `_isLoadingHistory` 标志防止重复加载
2. 重试时使用 `currentPage.onLoad()` 而非递归调用

**修复代码**:
```javascript
async loadHistoryReport(reportId) {
  // 【死循环修复 - 2026-03-14】防止重复加载
  if (this._isLoadingHistory) {
    console.warn('[DiagnosisPage] ⚠️ 正在加载中，跳过重复请求');
    return;
  }
  
  showLoading('加载中...');
  this._isLoadingHistory = true;

  try {
    // ... 加载逻辑 ...
  } catch (error) {
    hideLoading();
    this._isLoadingHistory = false;

    // 【死循环修复 - 2026-03-14】使用页面导航而非递归调用
    wx.showModal({
      success: (res) => {
        if (res.confirm) {
          // 【修复】使用页面重新加载而非递归调用
          const pages = getCurrentPages();
          const currentPage = pages[pages.length - 1];
          currentPage.onLoad(currentPage.options);
        } else {
          wx.navigateBack();
        }
      }
    });
  }
}
```

### 修复 3: reportService.js ⭐ 关键修复

**修复内容**:
1. 明确定义 `maxRetryCount = 3`
2. 限制重试条件（DATA_NOT_FOUND 和 DATA_INVALID 不重试）
3. 指数退避（1s, 2s, 3s）
4. 超时时间从 30 秒优化到 15 秒

**修复代码**:
```javascript
async getFullReport(executionId, options = {}) {
  const retryCount = options.retryCount || 0;
  
  // 【死循环修复 - 2026-03-14】限制最大重试次数为 3 次
  const maxRetryCount = 3;

  try {
    console.log('[ReportService] Getting full report:', executionId, `retry: ${retryCount}/${maxRetryCount}`);

    // ... 缓存检查 ...

    // 生产环境：使用云函数
    const callFunctionPromise = wx.cloud.callFunction({
      name: 'getDiagnosisReport',
      data: { executionId },
      timeout: 15000  // 【优化】15 秒超时，避免长时间等待
    });

    const timeoutPromise = new Promise((_, reject) => {
      setTimeout(() => reject(new Error('请求超时')), 15000);
    });

    const res = await Promise.race([callFunctionPromise, timeoutPromise]);

    // ... 数据处理 ...

  } catch (error) {
    logError(error, { context: 'getFullReport', executionId, retryCount });

    // 【死循环修复 - 2026-03-14】重试逻辑优化
    const errorType = this._classifyError(error);
    const canRetry = retryCount < maxRetryCount &&
                     errorType !== ErrorTypes.DATA_NOT_FOUND &&
                     errorType !== ErrorTypes.DATA_INVALID;

    if (canRetry) {
      const retryDelay = 1000 * (retryCount + 1); // 指数退避：1s, 2s, 3s
      console.log(`[ReportService] ${retryDelay}ms 后重试：${retryCount + 1}/${maxRetryCount}`);
      await new Promise(resolve => setTimeout(resolve, retryDelay));
      return this.getFullReport(executionId, { retryCount: retryCount + 1 });
    }

    // 达到最大重试次数或不可重试错误
    const errorMessage = this._getErrorMessage(error, errorType);
    console.error('[ReportService] 最终失败:', errorMessage);

    // 【修复】返回错误报告而非抛错，让前端决定如何展示
    return this._createEmptyReportWithSuggestion(
      errorMessage,
      this._getErrorTypeString(errorType)
    );
  }
}
```

## 🎓 经验总结

### 问题根因

**三处死循环风险**：
1. diagnosis.js 错误处理递归
2. report-v2.js 错误处理递归
3. reportService.js API 重试递归

**根本原因**：在错误处理和重试逻辑中使用递归调用，未限制最大次数。

### 修复原则

1. **防止重复加载**: 使用标志位 `_isLoadingHistory`
2. **避免递归调用**: 使用 `currentPage.onLoad()` 而非 `this.method()`
3. **限制重试次数**: 明确定义 `maxRetryCount = 3`
4. **指数退避**: 1s, 2s, 3s
5. **清理状态**: 在 catch 块中清理标志位

### 最佳实践

```javascript
// ❌ 错误：递归重试
if (canRetry) {
  return this.getFullReport(executionId, { retryCount + 1 });
}

// ✅ 正确：限制重试次数
const maxRetryCount = 3;
if (retryCount < maxRetryCount && canRetry) {
  const retryDelay = 1000 * (retryCount + 1);
  await new Promise(resolve => setTimeout(resolve, retryDelay));
  return this.getFullReport(executionId, { retryCount: retryCount + 1 });
}

// ✅ 正确：错误处理避免递归
wx.showModal({
  success: (res) => {
    if (res.confirm) {
      const pages = getCurrentPages();
      const currentPage = pages[pages.length - 1];
      currentPage.onLoad(currentPage.options);  // 页面重新加载
    }
  }
});
```
