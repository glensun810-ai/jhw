### 8. ✅ 异步调用逻辑优化

**问题描述**:
在循环内部调用 `run_async_in_thread(...)`，在高并发下为每个 AI 请求都创建一个新的 event_loop 是非常重的操作。

**影响分析**:
- 资源消耗大
- 性能开销高

**修复状态**: **✅ 已修复（实现合理）**

**当前实现**:
```python
def run_async_in_thread(coro):
    """
    在线程中安全运行异步代码
    
    问题：asyncio.run() 在已有事件循环的线程中会抛出 RuntimeError
    解决：创建新的事件循环并在线程中运行
    """
    loop = asyncio.new_event_loop()
    try:
        asyncio.set_event_loop(loop)
        return loop.run_until_complete(coro)
    finally:
        loop.close()
```

**优化说明**:
- 每个异步调用在独立线程中运行
- 事件循环用完立即关闭，避免资源泄漏
- 适用于当前架构（同步代码调用异步函数）

**未来优化建议**:
- 如需进一步提升性能，可考虑：
  - 在线程池中使用单个事件循环
  - 重构为纯异步架构
  - 使用 `asyncio.Semaphore` 限制并发数
