**ResultRepository.add_batch** (`diagnosis_report_repository.py:493-548`):

```python
def add_batch(
    self,
    report_id: int,
    execution_id: str,
    results: List[Dict[str, Any]],
    batch_size: int = 10,
    commit: bool = True
) -> List[int]:
    """批量添加诊断结果"""
    result_ids = []

    # 【P0 修复】使用上下文管理器正确获取连接
    with self.get_connection() as conn:  # ✅ 使用上下文管理器
        cursor = conn.cursor()

        # 分批处理
        for i in range(0, len(results), batch_size):
            batch = results[i:i + batch_size]
            
            # 插入当前批次
            for result in batch:
                result_id = self._insert_result(cursor, report_id, execution_id, result)
                result_ids.append(result_id)

        # 提交事务
        if commit:
            db_logger.debug(
                f"[ResultRepository] 批量添加完成：count={len(result_ids)}, "
                f"batches={total_batches}"
            )
        # ✅ commit=True 时，get_connection() 会在 yield 后自动 conn.commit()

        return result_ids
```

**ResultRepository.get_connection** (`diagnosis_report_repository.py:96-116`):

```python
@contextmanager
def get_connection(self):
    """获取数据库连接"""
    conn = get_db_pool().get_connection()
    try:
        yield conn  # ✅ 先 yield 连接
        conn.commit()  # ✅ 成功后提交
    except Exception as e:
        conn.rollback()  # ✅ 失败时回滚
        db_logger.error(f"数据库操作失败：{e}")
        raise
    finally:
        # ✅ 无论成功还是失败，都归还连接
        try:
            get_db_pool().return_connection(conn)
        except Exception as return_err:
            db_logger.error(f"归还连接失败：{return_err}")
```

## 根因分析

### 真正根因

**不是** "诊断结果未保存"，而是：

1. **代码注释误导**: `DiagnosisTransaction.__exit__` 的注释说"事务会在 with 块结束时自动提交"，但实际**没有数据库提交操作**
   - 真正的提交在 `ResultRepository.get_connection()` 中
   - 这是**正确的设计**（每个数据库操作独立提交）
   - 但注释容易让人误解

2. **数据验证过滤**: 部分 AI 返回结果可能因缺少必要字段被过滤
   - 导致保存数量 < AI 返回数量
   - 这是**预期行为**（数据完整性保护）

3. **可能的真正问题**: 如果数据确实未保存，可能原因是：
   - **场景 A**: `get_connection()` 的 `conn.commit()` 未被调用（异常抛出）
   - **场景 B**: `DiagnosisTransaction.rollback()` 被调用（回滚所有操作）
   - **场景 C**: 数据验证过滤了太多结果

```python
def rollback(self):
    """回滚所有操作"""
    rollback_count = 0
    
    # 按相反顺序回滚
    for operation in reversed(self.operations):
        try:
            operation.rollback_func()  # 调用回滚函数
            rollback_count += 1
        except Exception as e:
            db_logger.error(f"回滚失败：{e}")
```
