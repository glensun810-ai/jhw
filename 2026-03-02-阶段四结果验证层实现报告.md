## 一、实现内容

### 1.1 新增文件

#### `wechat_backend/services/result_validator.py`

**核心类**:
- `ResultValidator` - 结果验证器主类
- `ValidationResult` - 验证结果数据类
- `ValidationLevel` - 验证级别枚举
- `ValidationStatus` - 验证状态枚举

**验证功能**:

| 验证类型 | 验证内容 | 失败条件 |
|---------|---------|---------|
| 数量验证 | 数据库保存数量 vs 预期数量 | 数量不匹配或为零 |
| 质量验证 | AI 响应内容有效性 | 空响应、过短响应 |
| 完整性验证 | 必要字段完整性 | 缺失 brand/question/model/response |

**验证级别**:

| 级别 | 行为 | 适用场景 |
|------|------|---------|
| STRICT | 任何无效响应都失败 | 高质量要求场景 |
| NORMAL | 允许<50% 无效响应（警告） | 默认场景 |
| LENIENT | 只要有有效响应即可 | 容错场景 |

## 二、验证流程

### 2.1 验证流程图

```
阶段 4: 结果验证
    ↓
更新状态为 results_validating (progress=70)
    ↓
创建 ResultValidator 实例
    ↓
执行验证:
├─ 数量验证
│  └─ 从数据库读取已保存结果
│  └─ 比较 expected_count vs actual_count
│  └─ 失败：CRITICAL (0 条) / FAILED (不匹配)
│
├─ 质量验证
│  └─ 检查每个响应的 content
│  └─ 识别空响应、过短响应
│  └─ 计算有效率
│  └─ 根据验证级别判断：CRITICAL/FAILED/WARNING/PASSED
│
└─ 完整性验证
   └─ 检查必要字段 (brand/question/model/response)
   └─ 统计缺失字段
   └─ 失败：FAILED (response 缺失) / WARNING (其他字段)
    ↓
聚合所有验证结果
    ↓
返回 ValidationResult
    ↓
验证通过 → 进入阶段 5
验证失败 → 触发错误处理
```

## 三、关键设计决策

### 3.1 早期失败原则

验证层采用"早期失败"策略，发现问题立即返回，避免无效数据继续流转：

```python
if quantity_result.status == ValidationStatus.CRITICAL:
    return quantity_result  # 立即返回，不继续验证

if quality_result.status == ValidationStatus.CRITICAL:
    return quality_result  # 立即返回，不继续验证
```

**优势**:
- 快速失败，减少资源浪费
- 清晰的错误定位
- 便于问题排查

---

### 3.2 可配置验证级别

支持三种验证级别，适应不同场景需求：

```python
# 严格模式 - 任何无效都失败
validator = get_result_validator(validation_level=ValidationLevel.STRICT)

# 普通模式 - 默认
validator = get_result_validator()

# 宽松模式 - 容错
validator = get_result_validator(validation_level=ValidationLevel.LENIENT)
```

**应用场景**:
- STRICT: VIP 用户、付费诊断
- NORMAL: 普通用户、免费诊断
- LENIENT: 测试环境、调试模式

## 五、使用示例

### 5.1 基本使用

```python
from wechat_backend.services.result_validator import get_result_validator, ValidationLevel

# 创建验证器
validator = get_result_validator(validation_level=ValidationLevel.NORMAL)

# 执行验证
result = validator.validate(
    execution_id="exec_123",
    expected_results=ai_results,  # AI 调用结果
    validate_quality=True,
    validate_completeness=True
)

# 检查结果
if result.is_passed:
    print(f"验证通过：{result.message}")
    if result.warnings:
        print(f"警告：{result.warnings}")
else:
    print(f"验证失败：{result.message}")
    print(f"无效结果：{result.invalid_results}")
```

### 5.2 在 Orchestrator 中使用

```python
async def _phase_results_validating(self, results: List[Dict]) -> PhaseResult:
    """阶段 4: 结果验证"""
    
    # 使用 ResultValidator 进行完整验证
    from wechat_backend.services.result_validator import get_result_validator, ValidationLevel
    
    validator = get_result_validator(validation_level=ValidationLevel.NORMAL)
    validation_result = validator.validate(
        execution_id=self.execution_id,
        expected_results=results,
        validate_quality=True,
        validate_completeness=True
    )
    
    # 检查验证是否通过
    if not validation_result.is_passed:
        return PhaseResult(success=False, error=validation_result.message)
    
    # 验证通过，继续执行
    return PhaseResult(success=True, data=validation_result.to_dict())
```
