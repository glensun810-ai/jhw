## 五、已发现问题

### 5.1 P0 问题（已修复）

#### BUG-001: `taskStatusService.js` 变量只读错误
**发现时间**: 2026-03-02 02:44  
**影响范围**: 所有诊断任务  
**问题描述**: 
在 `parseTaskStatus` 函数中，试图重新赋值用 `const` 声明的变量 `backendShouldStopPolling`，导致运行时错误：
```
TypeError: "backendShouldStopPolling" is read-only
```

**复现步骤**:
1. 启动诊断任务
2. 前端解析后端返回的状态数据
3. 控制台报错，轮询异常中断

**根本原因**: 
代码修复时，错误地试图重新赋值 `const` 变量：
```javascript
// ❌ 错误代码
const backendShouldStopPolling = ...;

if (typeof statusData.should_stop_polling === 'boolean') {
  backendShouldStopPolling = statusData.should_stop_polling;  // 试图重新赋值
}
```

**修复方案**: 
直接更新 `parsed.should_stop_polling`，不重新赋值 `backendShouldStopPolling`：
```javascript
// ✅ 修复后
if (typeof statusData.should_stop_polling === 'boolean') {
  parsed.should_stop_polling = statusData.should_stop_polling;
} else if (typeof statusData.should_stop_polling === 'number') {
  parsed.should_stop_polling = statusData.should_stop_polling !== 0;
}

if (parsed.should_stop_polling) {  // 使用 parsed 对象的属性
  ...
}
```

**修复状态**: ✅ 已修复  
**验证结果**: ⏳ 待验证
