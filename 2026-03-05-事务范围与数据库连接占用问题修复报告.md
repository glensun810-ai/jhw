## 三、详细修复

### 3.1 步骤 1: 新增短事务执行方法 (P0)

**文件**: `diagnosis_orchestrator.py`

**新增方法**:
```python
async def _execute_in_transaction(
    self,
    operation_func: Callable,
    operation_name: str
) -> Any:
    """
    在独立短事务中执行操作

    【P0 修复 - 2026-03-05】缩小事务范围，每个操作独立提交
    """
    tx = DiagnosisTransaction(...)
    with tx:
        result = operation_func(tx)
        # 记录事务摘要
    return result
```

**效果**: 每个数据库操作独立事务，快速提交释放连接

### 3.2 步骤 2: 重构阶段 3 结果保存 (P0)

**文件**: `diagnosis_orchestrator.py`

**修改前**:
```python
async with self._transaction_context():
    tx.create_report(...)       # 持有连接
    tx.add_results_batch(...)   # 持有连接 500-800ms
```

**修改后**:
```python
# 子事务 1: 创建报告（独立事务）
report_id = await self._execute_in_transaction(
    lambda tx: tx.create_report(...),
    operation_name="create_report"
)

# 子事务 2: 分批保存结果（独立事务）
result_ids = await self._execute_in_transaction(
    lambda tx: tx.add_results_batch(..., batch_size=10),
    operation_name="add_results_batch"
)
```

**效果**: 事务范围从跨阶段缩小到单操作

### 3.3 步骤 3-5: 批量插入分批提交 (P0)

**文件**: `diagnosis_transaction.py`, `diagnosis_report_service.py`, `diagnosis_report_repository.py`

**修改前**:
```python
def add_results_batch(self, report_id, results):
    # 一次性插入所有结果
    for result in results:
        cursor.execute("INSERT ...")
    conn.commit()  # 一次性提交
```

**修改后**:
```python
def add_results_batch(self, report_id, results, batch_size=10, commit=True):
    # 分批处理
    for i in range(0, len(results), batch_size):
        batch = results[i:i + batch_size]
        for result in batch:
            cursor.execute("INSERT ...")
        conn.commit()  # 每批提交一次
```

**效果**: 30 条记录分为 3 批，每批持有连接时间从 800ms 降至 250ms

### 3.4 步骤 6-7: 连接泄漏检测 (P1)

**文件**: `database_connection_pool.py`

**新增功能**:
1. **连接使用跟踪**: 记录连接使用开始时间和占用线程
2. **定期检测线程**: 每 10 秒检查一次泄漏
3. **强制归还**: 超过 30 秒的连接强制归还

```python
def _leak_check_loop(self):
    """定期泄漏检测循环"""
    while not self._leak_check_stop.is_set():
        leaked_count = self._check_and_fix_leaks()
        if leaked_count > 0:
            db_logger.warning(f"检测并修复 {leaked_count} 个泄漏连接")
        self._leak_check_stop.wait(10.0)

def _check_and_fix_leaks(self) -> int:
    """检查并修复连接泄漏"""
    for conn_id, start_time in self._connection_use_time.items():
        age = time.time() - start_time
        if age > self.max_connection_age:  # 30 秒
            # 强制归还
            self._in_use.discard(conn_id)
```

**效果**: 连接泄漏自动检测并修复
