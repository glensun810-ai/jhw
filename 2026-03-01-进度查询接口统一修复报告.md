## 统一接口说明

### `/test/status/{id}` 接口特性

**请求方式**: GET

**路径参数**: 
- `id` - 执行 ID (executionId)

**查询参数** (可选):
- `since` - 增量轮询时间戳

**返回字段**:
```json
{
  "task_id": "execution-id",
  "progress": 0-100,
  "stage": "init|ai_fetching|analyzing|generating|completed",
  "status": "processing|completed|failed",
  "is_completed": true/false,
  "should_stop_polling": true/false,
  "results": [...],
  "detailed_results": [...],
  "analysis": {...},
  "created_at": "timestamp",
  "updated_at": "timestamp",
  "has_updates": true/false,
  "source": "database|cache",
  "cache_sync_status": "synced|out_of_sync|cache_miss"
}
```

**特性**:
1. ✅ 支持增量轮询（通过 `since` 参数）
2. ✅ 返回详细结果数据
3. ✅ 包含 `should_stop_polling` 标志
4. ✅ 支持缓存同步状态检查
5. ✅ 需要 JWT Token 或微信 OpenID 认证
6. ✅ 限流：30 次/分钟

## 删除的接口

### `/api/test-progress` (已删除)

**原请求方式**: GET

**原查询参数**: 
- `executionId` - 执行 ID

**删除原因**:
1. 与 `/test/status/{id}` 功能重复
2. 增加维护成本
3. 前端轮询逻辑分散

## 迁移指南

### 前端代码迁移

**旧代码**:
```javascript
// 方式 1: 使用 getTestProgressApi
const progress = await getTestProgressApi(executionId);

// 方式 2: 直接调用
wx.request({
  url: `${baseUrl}/api/test-progress?executionId=${executionId}`
})
```

**新代码**:
```javascript
// 统一使用 getTaskStatusApi
const status = await getTaskStatusApi(executionId);

// 或直接调用
wx.request({
  url: `${baseUrl}/test/status/${executionId}`
})
```

### 响应字段映射

| 旧字段 (`/api/test-progress`) | 新字段 (`/test/status/{id}`) | 说明 |
|------------------------------|-----------------------------|------|
| `execution_id` | `task_id` | 执行 ID |
| `progress` | `progress` | 进度百分比 ✅ |
| `status` | `status` | 状态 ✅ |
| `stage` | `stage` | 阶段 ✅ |
| `results` | `results` / `detailed_results` | 结果数据 ✅ |
| `should_stop_polling` | `should_stop_polling` | 停止轮询标志 ✅ |
| - | `has_updates` | 增量更新标志 (新增) |
| - | `source` | 数据来源 (新增) |
